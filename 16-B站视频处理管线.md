## 一、一句话

B站视频处理 = API优先 → API不通果断切浏览器。**不再重试API绕圈。**

| 平台 | API失败后 | URL来源 |
|:----|:---------|:-------|
| B站 | `sleep(10)`→重试1次→仍失败→**走浏览器** | INDEX.md取 `mid:数字` → `https://space.bilibili.com/{mid}` |

## 二、模块总览

| 模块 | 功能 | 输出 | 前置依赖 |
|:----|:-----|:-----|:---------|
| **A 拿cookie** | CDP提取B站cookie → 存文件 | `~/桌面/凭证/bilibili_com_cookie_最新.txt` | Chrome 9222端口连通 |
| **B 拉UP主列表** | API调space/arc/search → 或浏览器直读 | 视频列表JSON/浏览器提取（30条） | 模块A的cookie / Chrome CDP |
| **C 取字幕** | API直接取字幕文本 → 失败走浏览器读 | 字幕文字/NO_SUBTITLE | 模块A的cookie / Chrome CDP |
| **D 视频信息** | API取标题/描述/时长 | 基本信息 | 模块A的cookie |
| **E 下载音频** | yt-dlp下载（备用方案，极少用） | `/tmp/bili_video.m4a` | 无 |
| **F ASR转文字** | VideoCaptioner Docker | 全文txt+SRT | Docker容器运行中 |
| **G 分析提炼** | LLM分析→存events/ | 观点+持仓关联 | 模块C或F的输出 |
| **H hindsight留存** | 管道完成后调hindsight_retain | 进度存入向量库 | 全部模块完成 |

## 三、调用流程

### 两步排查法（优先）

收到B站新内容通知时，不走full pipeline，先快速确认有无必要走完：

```
Step 1: 直接浏览器导航到UP主主页
  → 最近几条视频标题扫一眼
  → 判断：跟持仓/宏观有关系吗？
  → 无关 → 跳过，不浪费ASR
  
Step 2: 有关系的 → API拉列表 → 看具体内容
  → 如果API挂了（code:-799/403）→ sleep(10)→重试1次
  → 还不行 → 浏览器直读（不用ASR，直接读简介+BV号取字幕）
```

### 场景1：用户说"XX UP主最近有什么新视频"

```
模块A（拿cookie）→ 模块B（拉列表）→ 筛选关键词 → 返回结果给用户
  ├─ API可通 → 快速直接
  └─ API不通 → sleep(10)→重试1次→不通就走浏览器
```

### 场景2：用户说"帮我总结这个B站视频"

```
模块D（取基本信息）+ 模块C（看有没有字幕）
   ├─ 有字幕 → 直接取 → 模块G（分析）
   └─ 没字幕 → 模块E（下载）→ 模块F（ASR）→ 模块G（分析）
```

### 场景3：搜索UP主（INDEX.md没有的）

```
不从INDEX.md取 → 直接在浏览器搜索用户
  → 搜到进主页
  → 问用户：加不加监控清单？
  → 加 → 更新INDEX.md
```

## 四、API→浏览器切换铁律

```
API调用失败（code:-799 / 403 / 空数据）
  ↓
第①步：sleep(10)延迟，再试一次
  ├─ 过了 → ✅ 继续用API
  └─ 还是不行 → 果断切浏览器
       ↓
第②步：从INDEX.md取该UP主mid
  → 拼URL：https://space.bilibili.com/{mid}
  → browser_navigate(url)
  ↓
第③步：等JS渲染 → 取内容
  ✅ 直接URL进主页，不在搜索框搜
```

**URL格式（写死，不搜）：**
| 平台 | URL格式 | 说明 |
|:----|:-------|:-----|
| B站 | `https://space.bilibili.com/{mid}` | mid从INDEX.md取 |

## 六、故障排除

| 问题 | 原因 | 解决 |
|:----|:-----|:-----|
| API返回code:-799 | 没带cookie被限流 | 跑模块A拿cookie |
| 脚本报错/返回0UP主 | INDEX.md路径不对 | 检查脚本中index_file路径 |
| cookie失效 | 7-30天 | 重跑CDP提取（`~/桌面/脚本/启动Chrome调试模式.sh`） |
| Docker容器不在 | 重启过 | `sg docker -c "docker start videocaptioner"` |
| ASR报DNS错误 | 必剪API偶发 | 重试1-2次 |
| pipeline跑完忘写hindsight | 违反铁律 | 立即调hindsight_retain存进度 |

## 七、与现有系统的耦合

```
B站管线 → 分析完成
      │
      ├─→ events/{日期}-B站-{视频名}.md
      ├─→ thinkers/{UP主}.md（新UP主首次处理时）
      ├─→ positions/{资产}.md（贝叶斯更新）
      ├─→ 23-贝叶斯更新日志.md（如有P值变化）
      └─→ hindsight_retain（存进度到向量库）
```

B站管线 = 信息获取方式不同（API/浏览器 vs 抖音监控），但**获取后的处理逻辑与抖音完全一致**。
