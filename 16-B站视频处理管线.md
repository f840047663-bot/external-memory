# 16-B站视频处理管线

> 独立模块组合：拿cookie→拉UP主视频列表→取视频信息→下载音频→ASR转文字→分析提炼。
> 任何AI读到本文档+对应skill，即可完整复现B站视频处理能力。
> 最后更新：2026-05-21

---

## 一、一句话

B站视频处理 = 6个独立模块的管线。每步可单独调用、单独调试、单独替换。**能API就API，不行才走下载+ASR。**

---

## 二、模块总览

| 模块 | 功能 | 输出 | 前置依赖 |
|------|------|------|---------|
| **A 拿cookie** | **→ 调用 skill:cdp-cookie-extractor** | `/tmp/bili_cookies.txt` | Chrome 9222端口 |
| **B 拉UP主列表** | API调space/arc/search | 视频列表JSON（30条） | 模块A的cookie |
| **C 取字幕** | API直接取字幕文本 | 字幕文字/NO_SUBTITLE | 无（API公开） |
| **D 视频信息** | API取标题/描述/时长 | 基本信息 | 无（API公开） |
| **E 下载音频** | yt-dlp下载 | `/tmp/bili_video.m4a` | 无 |
| **F ASR转文字** | VideoCaptioner Docker | 全文txt+SRT | Docker容器运行中 |
| **G 分析提炼** | LLM分析→存events/ | 观点+持仓关联 | 模块C或F的输出 |

---

## 三、调用方式

### 场景1：用户说"XX UP主最近有什么新视频"

```
模块A（拿cookie）→ 模块B（拉列表）→ 筛选关键词 → 返回结果给用户
```

### 场景2：用户说"帮我总结这个B站视频"

```
模块D（取基本信息）+ 模块C（看有没有字幕）
   ├─ 有字幕 → 直接取 → 模块G（分析）
   └─ 没字幕 → 模块E（下载）→ 模块F（ASR）→ 模块G（分析）
```

### 场景3：用户说"付鹏最近说了什么和我持仓相关的"

```
模块A→B（拉财经大咖录列表）→ 筛选付鹏视频
  → 逐个走场景2 → 汇总所有观点
  → 对应更新 positions/*.md 的贝叶斯概率
```

## 五、故障排除

| 问题 | 原因 | 解决 |
|------|------|------|
| API返回code:-799 | 没带cookie被限流 | 跑模块A |
| cookie失效 | 7-30天 | 重跑模块A |
| Docker容器不在 | 重启过 | `sg docker -c "docker start videocaptioner"` |
| ASR报DNS错误 | 必剪API偶发 | 重试1-2次 |
| ASR字带空格 | 引擎特性 | `tr -d ' '` 去空格 |

## 六、与现有系统的耦合

```
B站管线（模块A→F）→ 模块G分析
                    │
                    ├─→ events/{日期}-B站-{视频名}.md  （与抖音事件并列）
                    ├─→ thinkers/{UP主}.md  （与抖音观点合并）
                    ├─→ positions/{资产}.md  （贝叶斯更新，走thinker-info-to-belief）
                    └─→ 23-贝叶斯更新日志.md  （如有P值变化）
```

B站管线 = 信息获取方式不同（API vs 抖音监控），但**获取后的处理逻辑与抖音完全一致**。
