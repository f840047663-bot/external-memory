## 一、核心流程（一句话）

```bash
每天13:00 → bilibili_monitor.py (no_agent)
  ↓
读INDEX.md第四列（mid:数字）
  ↓
读 ~/桌面/凭证/bilibili_com_cookie_最新.txt
  ↓
调B站API /x/space/arc/search?mid={mid}&ps=30
  ↓
对比状态文件 → 找出新视频 → 输出到 ~/桌面/每日视频总结/
```

## 二、各模块细节

| 模块 | 实现 | 路径 |
|:----|:-----|:-----|
| **博主列表** | 读INDEX.md，兜底~/桌面/配置清单/B站监控清单.txt | `bilibili_monitor.py`第70-90行 |

| **输出** | `~/桌面/每日视频总结/{日期}_B站报告.md` | markdown格式 |
| **主题过滤** | `video_topic_filter.py` 过滤持仓/宏观相关 | 自动 |

## 三、API挂了怎么办

**典型故障：** `code=-799 请求过于频繁`

**修复：**
1. 等10-30分钟让IP解除限流
2. 如果急用 → 走浏览器直接拼URL：
```python
browser_navigate(url="https://space.bilibili.com/{mid}")
```
3. 如果浏览器也封（偶发）→ 过30分钟再试

⚠️ **不要换IP、不要换UA重试、不要搜替代API。**

## 四、ASR完整管线（手动处理阶段）

当需要把B站视频转成文字时（不常用，只在用户要求详细分析时）：

```bash
# 步骤1：获取cookie
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain bilibili.com --output ~/桌面/凭证/

# 步骤2：下载音频（仅当API字幕不可用时）
yt-dlp --cookies ~/桌面/凭证/bilibili_com_cookie_最新.txt \
  -x --audio-format m4a -o "/tmp/bili_video.%(ext)s" {BV号}

# 步骤3：ASR转文字（VideoCaptioner Docker）
# 详见 skill:bilibili-video-pipeline

# 步骤4：LLM分析 → 存入 events/{日期}.md
```
