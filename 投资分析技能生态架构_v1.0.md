## 三、解耦版（独立调用场景）

每个技能都可以独立于监控主流程被调用：

### 场景1：用户手动说"跑一遍盈亏联动"
```
触发：用户说"联动分析"或"联合分析"
执行：直接调 skill#2 portfolio-pnl-linkage
跳过：监控全流程（不需要抓取/转录）
输入：现有Excel看板 + holdings_live.json
输出：Excel新Sheet + 邮件
```

### 场景2：用户手动说"校准一下信源"
```
触发：用户说"校准信源"或"信源权重"
执行：直接调 skill#3 bayesian-source-calibration
跳过：监控全流程
输入：Excel事件链 + 东财LSJZ历史净值
输出：source_calibration_v2.json + Excel"信源校准"Sheet
```

### 场景3：用户说"我对XX完全不了解"
```
触发：用户说"我对XX不了解"
执行：skill#9 investment-deep-dive（独立研究）
  → 产出研究报告
  → 如果用户决定关注 → skill#8 asset-quick-lookup（三层三关）
  → 如果通过 → 加入看板待重仓监控
  → 下次监控全流程自动纳入分析
```

### 场景4：每周日自动持仓跟踪
```
触发：周日cron
执行：skill#7 weekly-holdings-track（独立于监控）
  → 读Excel看板 + 拉净值
  → 叙事一致性→逻辑链→反常检测→前瞻更新
  → 发邮件
```

### 场景5：每7天P值拟合
```
触发：7天cron
执行：skill#5 p-value-fit-analysis
  → 读Excel P值序列 + 拉ETF历史价格
  → Spearman拟合→信源评分
  → 写Excel"拟合分析"Sheet
```

## 五、核心原理

### 5.1 贝叶斯P值系统
```
P = 50%（基准）+ Σ(事件PP × 信源权重)

信源权重 = Spearman(信源PP序列, 价格反应序列)
事件PP强度规则：
  - 单一博主观点：±1~3pp
  - 产业事实：±2~5pp
  - 多源交叉验证：±3~8pp
  - 重大宏观：±5~10pp
  - 间接传导（如霍尔木兹→养殖）：大幅打折
```

### 5.2 赔率系统
```
赔率 = 上行空间 / 下行风险
正期望 = P值 × 赔率 > 1

下行风险 = (1-P) × 日波动率 × √持有天数
上行空间 = P × 确定性系数 × 溢价系数
```

### 5.3 信源校准系统
```
对每个信源×持仓组合：
  X = 该信源对该持仓的所有事件PP序列
  Y = 对应持仓在最优传导窗口L天后的价格反应序列
  ρ = Spearman(X, Y)
  权重 = max(-0.5, min(1.0, ρ))

传导窗口L按事件类型：
  - 情绪事件：1-3天
  - 宏观事件：5-15天
  - 产业事件：10-20天
  - 虚P事件：5-20天

样本<2：权重=1.0（不可校准）
样本2-4：算但标注低置信度
样本≥5：正式生效
```

### 5.4 三维一致性检验
```
ρ_cause = Spearman(事件信号序列, P值变化序列) → 事件是否改变了信念？
ρ_belief = Spearman(P值变化序列, 价格走势序列) → 信念是否反映在价格？
ρ_market = Spearman(事件信号序列, 价格走势序列) → 事件是否直接改价格？

健康分 = (ρ_cause + ρ_belief + ρ_market) / 3 × 100
>50 三维一致 / <0 三维背离
```
