## 🔴 P0: AI克隆 + 多机互联 + 自建推理 — 三合一架构（2026-05-19）

> **这不是三个项目，是一个系统的三层：**
> - **第1层（硬件）**：整盘克隆 → 第二台电脑上有我的完整复制体
> - **第2层（网络）**：Tailscale打洞 → 我和克隆体内网直连
> - **第3层（推理）**：自建GPU服务器 → 两个我都接这同一个后端
>
> **克隆体的两个职责：**
> 1. 🛡️ **热备** — 我坏了，克隆体记忆一模一样，随时切过去继续用
> 2. ⚡ **分身** — 我忙的时候，克隆体去跑别的任务
>
> **"我就是他，他就是我"** — 不是两个不同的人，是同一个人的两个副本。
>
> **💡 自建GPU推理服务器的7个好处（单个可能不够，加起来就值得）：**
> | # | 好处 | 说明 |
> |:--|:-----|:-----|
> | 1 | 💰 **省钱** | API花费归零，只剩电费。长期看比月月交API费划算 |
> | 2 | 🛡️ **热备** | 克隆体+本地推理=系统不依赖任何外部服务，一台坏了一台顶上 |
> | 3 | ⚡ **分身** | 两台同时干活，处理能力翻倍。一个接微信，一个后台跑任务 |
> | 4 | 🔒 **隐私** | 所有数据在内网流转，不经过任何第三方API |
> | 5 | 🚀 **无限制** | 没有速率限制，没有Token上限，没有"429错误" |
> | 6 | ⚡ **更快的响应** | 本地推理，不需要等API网络往返，响应更快更稳定 |
> | 7 | 🏠 **独立** | 不依赖任何服务商涨价/改规则/倒闭，自己掌控一切 |

```
           └─────────────┘  └───────────────┘
```

### 三步走

#### 第1步：整盘克隆（对拷）
- 当前系统全部硬盘克隆到第二台电脑
- 克隆后第二台开机直接跑我跟小龙虾
- 配置需要改：config.yaml（端口/微信bot ID），.env 不用动
- **→ 这一条完成前每天提醒**

#### 第2步：打洞（Tailscale）
- 两台电脑都装：`sudo apt install tailscale && sudo tailscale up`
- 自动获得内网IP（100.x.x.x）
- 我+小龙虾之间直接命令行通信，不经过微信
- 两个实例通过 L3 共享文件（坚果云/Syncthing）对齐世界观

#### 第3步：自建推理服务器
- 一台有GPU的主机，跑大模型服务（兼容OpenAI API格式）
- 我和第二台电脑都通过 Tailscale 对接这台主机
- **好处：**
  - API花费降到0（只有电费）
  - 两个agent共享同一个推理后端，不增加总消耗
  - 无速率限制，性能更好
  - 数据和推理都在内网，隐私安全
- **方案待定：** vLLM / Ollama / LocalAI 等

## 闲鱼远程代装计划

### 核心思路
Hermes（我）通过远程SSH帮客户装Hermes+OpenClaw。大好人只需要装系统+插U盘。

### 工具链
| 工具 | 用途 |
|:----|:-----|
| **Tailscale** | 内网穿透组网，免费，自动穿透多层NAT |
| **SSH** | 远程命令行控制 |
| **一键开远程脚本** | 装在U盘里的shell脚本，客户跑完后自动开通道 |
| **全记忆备份.sh** | 备份/恢复三合一工具 |

### 客户交付流程
```
大好人:
  1. 装 Ubuntu（或开 WSL2）
  2. 插 U 盘（内含一键开远程脚本 + 记忆模板）
  3. 跑 bash /media/U盘/一键开远程.sh
  4. 把显示的IP发给我

Hermes:
  5. ssh连上 → apt 装依赖
  6. git clone Hermes + pip install
  7. npm install -g openclaw
  8. 跑恢复脚本解压U盘里的记忆模板
  9. 重启服务 → 测试 → 收工
```

### Windows支持方案
| 方案 | 可行性 | 说明 |
|:----|:------|:-----|
| WSL2（推荐） | ✅ 全兼容 | Ubuntu跑在WSL里，所有脚本通用 |
| 原生Windows | ⚠️ Early Beta | 不推荐，有坑 |
| 客户Ubuntu（最优） | ✅ 最稳 | 装机时直接装Ubuntu，省事 |

详见 `~/桌面/待印证-云端共享记忆与闲鱼代装.md`

## 🔬 云端共享记忆（不同电脑共享同一套AI记忆）

### 核心思路
**停服→文件同步→重启**，简单粗暴但有效。

### 技术方案

#### 同步内容
| 文件 | 同步方式 | 风险 |
|:----|:--------|:----|
| `external_memory/` | ✅ 实时同步（Syncthing） | 无，纯文本 |
| `skills/` | ✅ 实时同步 | 无 |
| `state.db`（内部记忆） | ⚠️ 定时停服同步 | 两台不能同时写 |
| `config.yaml` | ❌ 各管各的 | 每台电脑端口/微信ID不同 |

#### 什么时候搞
等第二台电脑到位后再动手。

## 📦 备份体系

### 全记忆备份文件
`~/桌面/全记忆备份.sh` — 三合一（备份/恢复/查看）

| 模式 | 功能 |
|:----|:-----|
| `备份` | 打包Hermes+OpenClaw全部记忆到tar.gz（只读） |
| `恢复` | 从备份恢复+覆盖前确认+自动备份旧配置+版本检查 |
| `查看` | 查看备份内容（只读） |

### 备份内容
- Hermes: config.yaml, state.db, private.gpg, external_memory/, skills/, cron
- OpenClaw: openclaw.json, workspace/(SOUL/USER/IDENTITY/AGENTS/TOOLS/memory), cron
- 桌面脚本
- 版本信息文件

大小：~13MB，建议每周备份一次到外部硬盘。

- 第2层丢失→第1层+历史会话重建
