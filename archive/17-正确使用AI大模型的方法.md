# 17 - 正确使用AI大模型的方法

> 大好人 2026-05-18 总结。**任何 AI 换上来，先读这个文件。**

---

## 核心理念

大模型不是万能的。它不知道你的需求、不知道你的环境、不知道你的前置条件。

**正确用法 = 用编程思维玩大模型。**

---

## 三条铁律

### ① 告诉他功能怎么实现

不要指望 AI 自己猜。直接告诉它路径：

```
❌ "去把知乎那个帖子读一下"
✅ "先用 cookie-extractor 提取 Chrome cookie → 再用 waf-bypass 绕过 WAF → 再用 zhihu-reader 读帖"
```

### ② 告诉他有哪些库/技能可以用

就像编程时告诉程序员"你有 requests 库、有 pandas"一样，告诉 AI 你有什么工具：

```
可用技能：
  cookie-extractor   — 从 Chrome 偷 cookie
  waf-bypass         — 手机 UA 绕过 WAF
  zhihu-reader       — 读知乎帖子
  topic-monitor      — 组合上面三个做话题监控
```

### ③ 告诉他架构和步骤

架构你定、步骤你定，AI 只负责执行。不要让它自己设计架构：

```
❌ "帮我搞一套知乎监控"
✅ "三层架构：通用积木层(cookie-extractor+waf-bypass) → 平台层(zhihu-reader) → 编排层(topic-monitor)。按这个跑"
```

---

## 类比：AI = 编译器

```
你写的规则/步骤       → 源代码
AI                    → 编译器
技能(cookie-extractor等) → 标准库
输出正确结果           → 编译通过
你纠偏                 → debug
```

编译器不懂业务逻辑，但它能精准执行你写的代码。AI 同理——**不给创造力，给执行力。**

---

## 为什么这样更反脆弱？

1. **AI 本身不可靠** — 今天 DeepSeek 明天 Claude，同一个模型不同版本也可能行为不同
2. **用户知道环境** — 你知道你 Chrome 什么版本、知乎什么状态、IP 有没有被封
3. **步骤写死了就不依赖 AI 判断** — "先 A 再 B 再 C"是确定性流程，AI 执行即可
4. **任何 AI 都能接盘** — 纯文本技能备份在 L3 外部记忆（`99-技能完整备份.md`），不依赖特定框架

---

## 总结

> **把大模型当执行力，不当创造力。架构你定、步骤你定、工具你定，AI 只负责跑通。**
