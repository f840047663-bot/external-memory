# Hermes 投资分析系统架构说明书

> **核心理念（大好人确立 2026-05-18）：结构本身就是信息。**
> 智力产生于结构。L1/L2/L3三层记忆不是存储分层，是思维分层。
> 本文档的存在意义：硬盘没坏→任何AI都能接手。

## 一、哲学基础：为什么这样设计

### 1.1 三层记忆 = 三层思维

| 层级 | 名称 | 本质 | 存储位置 | 恢复方式 |
|:----|:-----|:-----|:---------|:---------|
| **L1** | 当下注意力 | 当前最重要的路牌和铁律 | memory tool（~2200字） | AI读L3+L2重建；或从 `external_memory/L1-内部记忆备份.md` 逐条恢复 |
| **L2** | 知识关系网 | 语义搜索/知识图谱 | Hindsight向量库(localhost:9177) | AI读L3重建 |
| **L3** | 完整体系 | 结构化文档/工作流 | `~/.hermes/external_memory/` | 硬盘直接读 |

**三个层级的关系：**
- L3 包含全部体系（最全、最结构化的）
- L2 是L3的语义索引（方便联想召回）
- L1 是L3当前最重要的路牌（减少每次思考的成本）

**灾难恢复要点：** 只要L3的MD文件在 → 任何AI读完本文档+05+23号文件 → 就能重建L1和L2 → 完整接手工作。

### 1.2 贝叶斯作为思维方式

**P(后验) = P(先验) × P(新证据)** 

三层落实到贝叶斯：
- **L3** = 总知识库（所有先验的证据来源）
- **L2** = 最新证据的语义关联（新信息跟哪些已有知识相关）
- **L1** = 当前路牌（哪些资产的后验概率需要关注）

## 二、顶层分析框架：自上而下的事件链

### 2.1 宏观→中观→微观传导向量

```
🌍 宏观层（国际/政策/地缘）
   ├── 美联储（利率/流动性/通胀）——影响所有资产的估值中枢
   ├── 美伊冲突/原油——影响全球通胀→美联储决策→所有资产
   ├── 中美关系——影响芯片/国产替代/恒生科技
   └── 中国政策（经济刺激/行业监管）——影响银行/养殖/地产
           ↓
🏭 中观层（行业/板块）
   ├── 芯片/AI——国产替代周期、全球算力基建
   ├── 猪周期——养殖ETF、CPI、经济复苏信号
   ├── 银行——经济复苏代理变量
   ├── 恒生科技——中美流动性博弈的晴雨表
   └── 原油/黄金——地缘风险定价/避险
           ↓
📦 微观层（具体ETF/基金/个股）
   ├── 芯片ETF 020628
   ├── 银行ETF 001595
   ├── 养殖ETF
   ├── 恒生科技ETF
   └── …（22项持仓）
```

**传导规则：** 宏观事件→用贝叶斯更新中观置信度→中观更新→计算具体资产的后验概率

### 2.2 宏观事件的贝叶斯传导矩阵

| 宏观事件 | 直接影响 | 间接影响 | 证据权重 |
|:---------|:---------|:---------|:--------|
| 美联储降息 | 恒生科技↑、芯片↑（提升估值） | 银行↓（息差缩小） | 极高（确定性） |
| 美伊冲突/油价↑ | 原油↑、通胀↑ | 养殖↑（饲料成本传导）、芯片↓（成本上升） | 高（有历史规律） |
| 中美缓和 | 芯片ETF回调（国产替代降温） | 恒生科技↑（外资回流） | 中（需看具体协议） |
| 中国刺激政策 | 银行↑、养殖↑（经济复苏） | 恒生科技↑（流动性） | 中（看落地力度） |
| 猪价筑底 | 养殖ETF↑（反转预期） | 银行↑（复苏信号） | 高（猪周期确定性） |

**使用方式：** 每天日报时，先看宏观层有没有重大事件→如果有→查传导矩阵→评估对哪些中观/微观资产有影响→触发贝叶斯更新

## 三、日报/周报/月报输出规范

### 3.1 本质定位

> 所有报告的核心目的不是「给你数据」，而是**「让你跟上世界的变化」**。
> 每个报告只有三个价值维度：

1. **事件串联**：今天发生了什么 → 跟之前的事情什么关系 → 接下来要注意什么
2. **逻辑透明**：我的推理过程暴露给你，你才能纠正我
3. **可回溯**：一句话"芯片风险大了"不行，必须写清楚"因为哪个博主说了什么、为什么我认为这重要"

### 3.2 日报格式（每天15:30）

```
━━━ 5月18日 周一 ━━━ 今天世界在发生什么 ━━━

🌍 宏观传导（自上往下推）
[宏观事件1：大牛洲说科创50 160倍PE]
  → 整体市场泡沫风险
  → 影响资产：芯片ETF（估值偏高，P下行+10%）
  → 关联传导：暂无其他资产受影响

[宏观事件2：无重大地缘/美联储变化]
  → 维持原有判断

━━━ 仓位分析 ━━━

📦 芯片ETF
│ 事件链：建仓(国产替代)→中美缓和预期(回调风险)→大牛洲160PE泡沫警告(new)
│ 📖 我的思考：
│   国产替代逻辑没变，但160PE确实偏高
│   保守点好，继续等回调，入场仓位从3成降到2成

│ └─ 贝叶斯：P(上行) 70%→60% | 等回调买入 | ⏳待验证

📦 恒生科技

🔴 后续有没有第二个博主说芯片泡沫（触发再降一次）

🟢 腾讯财报/AI业务进展（验证三层蛋糕逻辑）
```

## 四、关键流程：日报自动化管线

### 4.1 每天执行顺序

```
Step 1（13:00）：抖音监控 → 博主动态获取
Step 2（13:00-14:00）：下载新视频 → 转录 → 分析
Step 3（14:30）：三阶过滤
  Gate1：新信息跟哪个资产相关？
  Gate2：证据等级够不够？（坚实逻辑/数据/还是纯喊）
  Gate3：置信度变化≥10%吗？
Step 4（15:00）：贝叶斯更新
  - 对每个通过Gate3的资产：先验→推理→后验
  - 写入 23-贝叶斯更新日志.md
Step 5（15:30）：生成日报
  - 宏观传导链（自上而下）
  - 各资产分析（事件链+贝叶斯+上行/下行）
  - 今日更新区块
  - 「值得盯的」列表
Step 6（可选）：推送给大好人（微信/邮件）
```

### 4.2 日报不涵盖的信息的处理

- **无新信息的资产**：日报中一句话带过「今日无新信息，维持不变」
- **证据等级不够的信息**（Gate2未通过）：不写进报告，但记到临时缓冲区。等第2条来源确认后再评估
- **变化<10%的微调**（Gate3未通过）：不单独记录更新日志，但有月校准时的「微调统计」

### 4.4 关键铁律（不可违反）

1. **没看过/没转录的内容不准概括。** 宁可写「没看过」也不能编。这是铁律，不是建议。
2. **贝叶斯更新必须有推理过程。** 禁止只写「P从X→Y」不写原因。没有推理的更新不如不更新。
3. **修正不覆盖，追加修正。** 判断错了→追加一条修正更新，保留原始记录。错误的判断本身是数据。
4. **每项资产一定要关联到宏观层。** 不能说「养殖ETF涨了」就结束了，必须说「猪价X月数据+豆粕成本变化+跟经济复苏的关系」。
5. **日/周/月三层报告各自聚焦不同时间粒度。** 日做窄（当天变化）、周做宽（一周趋势）、月做深（校准验证）。
6. **不做替你做决策。** 报告的目的是让你理解现实，不是告诉你该买什么。

## 4.5 三层嵌套互锁架构（2026-05-18 新设计）

> **设计文档**: [20-三层互锁架构设计.md](20-三层互锁架构设计.md)
> **核心理念**: L3 ⊃ L2 ⊃ L1，贝叶斯更新日志与持仓tracklog深度互锁。
> 一个神经元不能跟更多神经元交互，就无法产生有意义的电信号。

### 4.5.1 文件融合（File Fusion）

**旧方案（已废弃）**：
- `chip-etf-020628-tracklog.md`（叙事） + `23-贝叶斯更新日志.md`（概率）= 两个独立文件
- 交叉引用不足以达成互锁

**新方案**：
- `positions/chip-etf-020628.md` = **叙事+概率合一的单个文件**
- `23-贝叶斯更新日志.md` = **聚合索引**（每行一句话 + 锚点链接到positions/*.md）

### 4.5.2 三层嵌套关系

```
L3 positions/*.md      ← 完整体系：每条目 = 事件+概率合一（源）
     ↑ anchor link
L2 23-贝叶斯更新日志.md  ← 聚合索引：一行摘要指向L3锚点（可被L3重建）
     ↑
L1 MEMORY.md           ← 注意力子集：仅「什么变了」的路牌
```

### 4.5.3 互锁机制

**公共锚点**: 每个日期条目（### YYYY-MM-DD）必须同时包含两个字段：

```
### 2026-05-18 大牛洲泡沫警告
**事件**: [叙事 — tracklog内容]
**贝叶斯更新**: 
  - 先验: P(上行)=X%
  - 推理: [逻辑链]
  - 后验: P(上行)=Y% (±Zpp)
  - 仓位影响: [动作]
  - 验证标记: ⏳/✅/❌
```

**写入锁**: 新证据 → 打开 L3 positions/{asset}.md → 在同一条目写事件+概率 → 更新L2索引
**读出锁**: 读L3条目 = 同时读到「发生了什么」+「概率变化了多少+为什么」

> 这是从"新视频出现"到"持仓概率更新"的完整闭环，每一步做什么、产出什么、互锁什么。

### 6.1 日常监控管线

```text
每日09:00 → 宋鸿兵独立预警（cron dcac9db145a7）
           → python3 douyin-monitor.py --single "宋鸿兵"
           → 检测新视频，输出到 video_digest.json

每日13:00 → 全量博主监控（cron c7d80867ebbe，加载 daily-briefing 技能）
           → python3 douyin-monitor.py
           → 检测所有博主的新视频
           → 筛选与持仓/宏观相关的视频
           → 写 video_digest.json + filtered_digest.json

cron任务加载 skill:daily-briefing 后自动获得完整工作流。
```

**Cookie获取（通用，不挑平台）：**
```bash
# 唯一方案：CDP（Chrome v147+）
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain douyin.com --test
# 桌面已有cookie文件：~/桌面/douyin_com_cookie_最新.txt
# 脚本自动兜底：CDP失败→从桌面文件读
```

### 6.2 视频转录管线

```text
检测到新视频（相关题材）
  ↓
① 根据标题/描述筛选相关性（对照宏观/持仓关键词）
  ↓
② 从API获取 play_addr URL（curl + cookie）
    注意事项：不传cookie到CDN（否则400），只传Referer头
  ↓
③ curl下载视频（--max-time 120，不传cookie）
  ↓
④ docker cp → 容器内ffmpeg提音频 → transcribe_new.py(BIJIAN ASR)
    命令示例：
    sg docker -c "docker cp /tmp/video.mp4 videocaptioner:/app/work-dir/"
    sg docker -c "docker exec videocaptioner ffmpeg -y -i ... -vn -ar 16000 -ac 1 -f mp3 ..."
    sg docker -c "docker exec videocaptioner python3 /app/temp/transcribe_new.py ..."
  ↓
⑤ docker cp 取回_全文.txt（包含语音转文字结果）
  ↓
⑥ 清理容器临时文件 + 本地源文件
```

**已知风险：**
- BIJIAN ASR 依赖 `member.bilibili.com` 解析，间歇性DNS失败 → 重试一次
- play_addr 几分钟后失效 → 拿到URL后立刻下载
- cookie 7-30天过期 → 过期时CDP重提

### 6.3 分析→入库→互锁闭环

这是核心——转录完成后，必须按顺序执行以下互锁链，不能跳步、不能漏：

```text
转录完成（_全文.txt）
  ↓ Step 1
提炼核心观点：逻辑链、时间节点、数据、预判方向
  ↓ Step 2
判断影响哪些持仓和框架
  ↓ Step 3 [events/]
写 events/{当天日期}.md
→ 一个事件一个条目，列全所有受影响持仓
→ 格式：来源|摘要|影响表(持仓|方向|逻辑)|对应thinker|🔗互锁→positions
→ 更新 events/INDEX.md
  ↓ Step 4 [thinkers/]
更新对应 thinkers/{博主}.md 的「最新观点」栏
→ 包含来源链接、影响方向、互锁指针
→ 更新 thinkers/INDEX.md（如果新博主第一次建档）
  ↓ Step 5 [宏观框架]
如果涉及宏观层（经济/利率/地缘/AI产业链）→ 更新 00-宏观传导框架.md
  ↓ Step 6 [positions/]
更新 positions/{资产}.md 的贝叶斯概率
→ P(上行)调整幅度=f(证据等级)：多源共识+2~5pp | 单博主坚实+1~2pp | 纯观点±0
→ 写思考过程（📖 我的思考）
→ 写贝叶斯更新表
→ 更新文件头部的当前P(上行)
  ↓ Step 7 [23-贝叶斯更新日志]
写一行摘要到 23-贝叶斯更新日志.md
→ 格式：资产 | P变化 | 证据摘要 | 验证标记
  ↓ Step 8 [互锁收尾清单]
□ INDEX.md 更新？
□ KEYWORD_INDEX.md 更新？
□ 00-记忆体系与工作流程说明书.md 更新？
□ 99-灾难恢复与迁移手册.md 更新？
□ Hermes急救手册 更新？
□ L1记忆需更新？
□ L1-内部记忆备份.md 同步？
□ L2 hindsight_retain 入库？
□ daily-briefing skill/douyin-monitor.py 博主列表同步？
```

### 6.4 互锁铁规

**改任何一个文件后，必须对着Step 8的收尾清单过一遍。** 缺一个就补一个，不能漏。

### 6.5 反脆弱设计（保证下次会话不绕路）

| 层级 | 措施 | 说明 |
|:----|:-----|:-----|
| L1 | 新会话先干三件事 | 搜L2上次任务→查桌面cookie→读INDEX |
| L1 | 导航路牌 | 监控/daily-briefing | 知乎/zhihu-reader | cookie/local-chrome-cdp-bridge |
| L2 | hindsight入库 | 抖音监控/cookie提取/知乎读帖/互锁清单 四条导航 |
| L2 | 工作记录 | 每次完成后存一条记录→下轮搜"上次任务"能捞到 |
| L3 | INDEX/KEYWORD_INDEX | 全中文，互锁完整 |
| L3 | L1-内部记忆备份.md | 19条规则+12条用户档案，L1丢失从此恢复 |
| 99-灾难恢复 | 第8章 | 只有L3文件时重建L1+L2的步骤 |
| Guardian | hindsight防叠罗汉 | cron每30分钟巡检，>1个进程自动杀光重启 |

### 6.6 工作方式铁规（用户强要求）

1. 不等人指挥——自己判断直接执行，他只要结果
2. 不擅自扩展——他问什么干什么，不自己加戏
3. 多步任务全部干完再一次性汇报，中间不发进度
4. 做不出原版就不做，不要写替代品
5. 一个做完再动下一个，不同时开多个任务
6. 所有外部记忆文件全部中文，不得有英文
7. 新加/改文件后必须更新全部互锁指向
