## 🔄 完整流水线

### Step 1: 运行监控脚本

```bash
cd /home/fw/.hermes/scripts && python3 douyin-monitor.py
```

**这个脚本是纯API方式：**
- 不走浏览器，不启动Chrome，不碰CDP
- 直接调抖音API：`/aweme/v1/web/aweme/post/?sec_user_id={ID}&count=30&aid=1128`
- Cookie来自 `/tmp/douyin_cookies.txt`（已有，有效）
- 所有博主信息硬编码在脚本内，不需要读外部文件

### Step 2: Cookie管理（唯一可能出问题的地方）

Cookie文件 `/tmp/douyin_cookies.txt` **已经有sessionid，目前有效**。

**如果哪天API返回412（cookie过期）：**
```bash
yt-dlp --cookies-from-browser chrome --cookies /tmp/douyin_cookies.txt --skip-download "https://www.douyin.com/video/1"
```
注意：yt-dlp 提取失败是常见情况（报 `Failed to parse JSON`），但**已有cookie一般还能用几天**。Cookie过期周期通常是3-7天。

**手动验证cookie是否有效：**
```python
curl -s --max-time 15 "https://www.douyin.com/aweme/v1/web/aweme/post/?sec_user_id=MS4wLjABAAAAzaye_V0qtP4d7m77UywUBRq7xB9CRiLaeGPfg79hLtQ&count=5&aid=1128" \
  -H "User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/147.0.0.0 Safari/537.36" \
  -H "Referer: https://www.douyin.com/user/MS4wLjABAAAAzaye_V0qtP4d7m77UywUBRq7xB9CRiLaeGPfg79hLtQ" \
  -H "Cookie: $(cat /tmp/douyin_cookies.txt | tr '\n' '; ')"
```
返回200且有 `aweme_list` = 有效。返回412 = cookie过期了，需要手动打开浏览器登录抖音后重新导出。

### Step 3: 新视频检测

脚本自动对比 `~/.hermes/douyin_state/{博主名}.json` 中的已处理视频ID。
- 新视频 → 输出到 `~/.hermes/output/video_digest.json`
- 无新视频 → 脚本正常退出，logger.info("无新视频")

### Step 4: 下载视频 → 转文字 → 分析

当有值得下载的新视频时：
1. **获取视频直链**：从API返回的 `video.play_addr.url_list[0]` 取CDN URL
2. **下载**：用curl，**只传Referer不传Cookie**（否则400）
   ```bash
   curl -L -o /tmp/{vid}.mp4 -H 'Referer: https://www.douyin.com/' -H 'User-Agent: Mozilla/5.0 ...' "{play_url}"
   ```
3. **转文字**：Docker VideoCaptioner（参考06-环境备忘.md）
4. **分析**：读转录文本，LLM提炼核心观点

### Step 5: 写入 video_digest.json → 通知小龙虾

```json
{
  "date": "2026-05-14",
  "items": [
    {
      "title": "视频标题",
      "source": "博主名",
      "analysis": "核心观点",
      "relevance": "关联持仓说明",
      "signal": "bullish/bearish/neutral",
      "confidence": "high/medium/low",
      "asset_impacted": ["芯片ETF", "AI联接C"]
    }
  ]
}
```

通知小龙虾：
```bash
openclaw agent --session-id main --message "Hermes今日视频监控完成，video_digest.json已更新" --timeout 120
```

### Step 6: 清理

```bash
rm -f /tmp/{vid}.mp4 /tmp/{vid}_transcript.txt
pkill -f "google-chrome.*--remote-debugging" 2>/dev/null || true
```

## 🚫 已废弃的路线（不要再用）

以下方法在历史流程中反复尝试、反复失败，**已确认不可靠**，不要再走：

| 已废弃方法 | 失败原因 | 替代 |
|:-----------|:---------|:-----|
| CDP WebSocket 开临时Chrome搜抖音 | 抖音弹验证码/headless检测 | API直调 |
| 复制用户Chrome配置到临时目录 | Chrome147 cookie加密(portal/libsecret)不可解密 | API直调 |
| Playwright + 系统Chrome（非headless） | 依赖用户桌面会话，会话关闭就不行 | API直调 |
| yt-dlp 下载抖音视频 | 频繁报Failed to parse JSON | API直链 + curl |
| 浏览器访问用户主页URL | CAPTCHA拦截 | API `/aweme/v1/web/aweme/post/` |

## ⚠️ 已知坑（不要再踩了）

### Cookie类
| 坑 | 表现 | 正确做法 |
|:--|:-----|:---------|
| yt-dlp提取报错 | `Failed to parse JSON` | 忽略，直接用已有cookie文件 |
| 下载传Cookie | 400 Bad Request | 下载时只传Referer，不传Cookie |
| cookie过期 | API返回412 | 手动打开Chrome登录抖音后重新运行yt-dlp |

### API类
| 坑 | 表现 | 正确做法 |
|:--|:-----|:---------|
| 用户主页URL访问 | CAPTCHA | 用API接口，不用浏览器访问主页 |
| 视频URL有时效性 | 几小时后play_addr返回400 | API拿到后立即curl下载 |
| 置顶视频干扰 | create_time不按最新排序 | 脚本已做 `sort(key=create_time, reverse=True)` |

### 定时任务
| 坑 | 表现 | 正确做法 |
|:--|:-----|:---------|

如果cookie没过期，这两个命令永远不需要修改。cookie过期了就去浏览器登录抖音后跑一次：
```bash
yt-dlp --cookies-from-browser chrome --cookies /tmp/douyin_cookies.txt --skip-download "https://www.douyin.com/video/1"
```

> **⚠️ 历史坑记录（2026-05-14）：**
> 1. **digest只写一次不更新** — 旧逻辑只在发现新视频时写入digest，导致digest永远卡在第一次跑的数据上。**修复：** digest改为每次从state取最新5条重建，不管有没有新视频。
> 2. **state只存ID不存标题** — 旧state格式 `{"last_video_ids": [...]}`，导致重建digest时没有标题和描述。**修复：** state格式改为 `{"videos": [{"aweme_id","title","desc","create_time"}, ...]}`。
