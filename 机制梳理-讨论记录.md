## 第五轮：子agent注意力锚机制设计

**时间：** 2026-06-02 12:00
**用户提议：** 引入子agent辅助主agent，防止注意力漂移。前提：必须反脆弱，不能承包节点。

### V4 Pro设计方案
**核心定位：** 子agent做"探针"而非"控制器"
- 只检查L1.5是否超时未更新、是否有循环/偏离
- 检测到异常→写ATTENTION_FLAG到L1.5→主agent在下个阶段入口读到flag后处理
- 通信通过L1.5文件接力（最反脆弱）
- 子agent失败→主agent降级到手动自查L1.5
- 融入统筹框架：加"哨兵阶段"，约15行
- L1.5是"被动锚点"，子agent是"主动哨兵"

# ~/.hermes/scripts/watchdog.sh
FLAG="/tmp/hermes_watchdog_flag"
L15="/home/fw/.hermes/external_memory/L1.5_工作记忆.md"
PARENT_PID=$1

while true; do
  # 检查父进程是否存活
  if ! kill -0 $PARENT_PID 2>/dev/null; then
    exit 0  # 父进程死了，自动退出
  fi

  if [ -f "$L15" ]; then
    heartbeat=$(grep "heartbeat:" "$L15" | tail -1 | awk '{print $2}')
    if [ -n "$heartbeat" ]; then

      if (( now - heartbeat > 120 )); then
        echo "TIMEOUT at $(date -Iseconds)" >> "$FLAG"
      fi
    fi
  fi
  sleep 30
done

## 看门狗原理解释

**为什么看门狗能解决注意力漂移？**

1. **外部独立监控**：看门狗是独立bash进程，不依赖主agent的上下文/记忆/注意力
2. **心跳机制**：主agent每1-2分钟更新L1.5的心跳时间戳，看门狗检查心跳是否超时
3. **超时检测**：如果主agent漂移/卡死/陷入循环，心跳停止更新→看门狗2分钟后写flag
4. **强制恢复**：主agent在下个检查点读到flag→立即中断→回读L1.5恢复进度
5. **反脆弱**：看门狗挂了→主agent降级到手动自查；主agent挂了→看门狗自动退出

**本质：** 用外部独立进程监控主进程的心跳，类似云计算里的health check。主agent自己可能"忘了抬头"，但看门狗永远在盯着。
