# 流程日志规范

> 路径：~/桌面/监控运行日志.md
> 最后更新：2026-06-01

## 格式

```
| 时间 | 平台 | 博主 | 视频ID | 标题 | 状态 | 错误信息 |
```

**示例：**
```
| 2026-05-31 23:04:38 | douyin | 付鹏的财经世界 | 764562033057205 | 通胀不一定是坏事 | downloaded |  |
| 2026-05-31 23:12:16 | douyin | 宋鸿兵观天下 | 764486926784594 | 美国金融市场对油价极其乐观 | failed | no play_url |
```

## 状态值（只许用这些）

| 状态 | 含义 |
|:----|:-----|
| `downloaded` | 视频/音频文件已下载到pipeline_downloads/ |
| `transcribed` | 转录完成，md文件已归档到events/ |
| `analyzed` | AI分析完成，结论已写入 |
| `archived` | 完整管道走完（下载→转录→分析→归档） |
| `failed` | 失败，错误信息列写明原因 |
| `skipped` | 跳过（太旧/已处理过） |

## 🚨 写日志铁律

1. **干完才写，不许预写** — 文件没下载到磁盘就不能写downloaded，转录md没生成就不能写transcribed
2. **每处理完一条就append一行** — 不要攒着一堆再写
3. **时间戳必须真实** — 用当前实际时间，不许伪造
4. **任务完成后追加完成标记**：`## 任务完成 2026-06-01 00:00`
5. **日志不清空** — 它是审计痕迹，不清空。新任务在同文件追加新日期标题

## 🚨 读日志铁律

1. **新会话/中断恢复第一步：先读日志最后20行**
2. **看时间间隔** — 最后一条如果超过2小时前，说明任务中断了，不能盲目继续
3. **验证最后状态** — 日志说downloaded，去ls验证文件真的存在。日志不可信就重跑
4. **从最后一个failed或downloaded但没transcribed的开始**

## 日志不清空的原因

- 中断恢复需要知道之前干了啥
- 用户问"昨天付鹏的视频处理了没"可以直接查
- 审计追踪：哪个视频失败了、为什么失败、重试了几次
