### 场景B：系统还在，但我（Hermes）坏了

指：Hermes能启动，但内部记忆丢失（失忆了、乱回答、不记得你是谁）

```
症状：Hermes不记得持仓、不记得改配置铁律、像刚装好一样
原因：state.db 损坏或丢失（内部记忆存在这里）

修复步骤：
1. 先别慌——外部记忆文件还在就没事
2. 删掉损坏的 state.db（Hermes会自动重建）
   rm ~/.hermes/state.db
   rm ~/.hermes/state.db-wal   # （如果有）
   rm ~/.hermes/state.db-shm   # （如果有）

3. 重启Hermes
   hermes start
   或 hk kill && hk

4. 我重启后会读外部记忆重建内部记忆
   → 读 INDEX.md → 知道有三层记忆架构
   → 读 08.md → 知道备份盲区
   → 读 05.md → 知道持仓和投资框架
   → 读 00.md → 知道改配置铁律
   → 几分钟内恢复所有能力

5. 如果还不能恢复 → 用备份恢复 state.db
   全记忆备份包里包含 state.db
   解压单独恢复：
   tar -xzf 备份文件.tar.gz \
       -C ~/ .hermes/state.db
   然后重启
```

**为什么外部记忆比内部记忆更可靠：**
```
内部记忆（state.db）  → SQLite数据库，格式随版本变，易损坏
外部记忆（MD文件）     → 纯文本Markdown，任何版本都能读
所以：外部记忆在，我就不会真正失忆
```

### 恢复失败排查表

| 现象 | 原因 | 解决办法 |
|:----|:----|:---------|
| `hermes start` 报错 | Hermes版本不匹配 | 确认装的是最新版 |
| 恢复脚本说"版本不兼容" | 备份时的Hermes版本和现在装的不同 | 继续恢复，如果state.db不兼容就删掉让它重建 |
| 模型连不上 | `.env` 没拷过去 | 检查 `~/.hermes/.env` 是否存在 |
| 抖音监控不运行 | `scripts/` 没拷或 `douyin_state.json` 丢了 | 重新拷scripts和state文件 |
| OpenClaw启动不了 | 版本不对或配置丢了 | 重新 `npm install -g openclaw@latest` |
| Hermes不记得持仓 | state.db是旧的 | 读外部记忆会自动重建 |
| 备份文件打不开 | .tar.gz损坏 | 试另一个备份，或从U盘原始文件恢复 |

## 五、安全提醒

1. **`config.yaml` 第14行有明文的硅基流动API Key** — 应改为 `${SILICONFLOW_API_KEY}` 并搬到 `.env`
