## 二、各平台详细流程

### B站（bilibili_monitor.py）

**启动方式：** cron每日13:00自动跑（job: c964de258fbe, no_agent）

**博主列表读取：**
1. 优先读 `INDEX.md` 第四列（B站列），格式 `mid:数字`
2. 兜底读 `~/桌面/配置清单/B站监控清单.txt`

**Cookie加载：**
1. 读 `~/桌面/凭证/bilibili_com_cookie_最新.txt`（primary）
2. 兜底：`yt-dlp --cookies-from-browser chrome --cookies /tmp/bili_latest_cookies.txt`

**API调用：**
```bash
curl "https://api.bilibili.com/x/space/arc/search?mid={mid}&ps=30" \
  -H "Cookie: $(cat ~/桌面/凭证/bilibili_com_cookie_最新.txt)"
```

**速度：** ~5秒完成。偶发-799（请求过频），等10秒重试1次即可。

**⚠️ 请求频率铁律：** 不同博主API请求之间必须加**sleep(5-8秒)**。连续快速请求会触发-799限流。手工跑时尤其注意——cron no_agent模式单线程自带间隔，但手工逐个查时容易太快。

**输出：** `~/桌面/每日视频总结/2026-05-25_B站报告.md`

**如果API全挂（全部-799）：**
```bash
browser_navigate(url="https://space.bilibili.com/{mid}")
```

### 抖音（douyin-monitor.py）

**启动方式：** cron每日13:00自动跑（job: c7d80867ebbe, no_agent, 脚本365行）

**博主列表读取：**
1. 读 `INDEX.md` 第三列（抖音列），匹配反引号包裹的 `MS4w...` sec_uid
2. 不从 `~/桌面/配置清单/抖音主页.txt` 读（已废弃）

**Cookie提取（Chrome v147+专属流程）：**
1. Chrome调试模式必须提前启动在9222端口（`~/桌面/系统工具/启动Chrome调试模式.sh`，双击或在终端用`.desktop`启动）
2. 脚本自动走CDP连接Chrome → 创建标签页 → 导航抖音首页 → 等6秒 → Network.getAllCookies
3. 提取成功的cookie含`sessionid` → 存入 `/tmp/douyin_cookies.txt`
4. 兜底读 `~/桌面/凭证/douyin_com_cookie_最新.txt`

**API调用：**
```bash
curl "https://www.douyin.com/aweme/v1/web/aweme/post/?sec_user_id={sec_uid}&count=30" \
  -H "Cookie: $(cat /tmp/douyin_cookies.txt)"
```

**输出：** 直接stdout打印（no_agent模式→cron捕获）

**已知问题：**
- 部分博主（付鹏）偶尔API返回空 `status_code:5` → 跳过
- CDP导航页面需等6秒，扫描12人约需60秒

### 知乎（zhihu_monitor.py）

**启动方式：** cron每日13:10自动跑（job: 595e1a1dd15e, no_agent）

**用户列表读取：**
1. 读 `INDEX.md` 第五列（知乎列），格式 `people/xxx`
2. 兜底读 `~/桌面/配置清单/知乎监控清单.txt`

**Cookie加载：**
1. 读 `~/桌面/凭证/zhihu_com_cookie_最新.txt`（primary）
2. 兜底：`yt-dlp --cookies-from-browser chrome --cookies /tmp/zhihu_latest_cookies.txt`

**API调用：**
```bash
# 文章列表
curl "https://www.zhihu.com/api/v4/members/{url_token}/articles?limit=10" \
  -H "Cookie: $(cat ~/桌面/凭证/zhihu_com_cookie_最新.txt)" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)"

# 回答列表
curl "https://www.zhihu.com/api/v4/members/{url_token}/answers?limit=10\&order_by=created" \
  -H "Cookie: $(cat ~/桌面/凭证/zhihu_com_cookie_最新.txt)" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)"
```

**速度：** ~10秒完成（需iPhone UA绕过WAF）

**⚠️ 请求频率铁律：** 不同用户API请求之间必须加**sleep(5-8秒)**。知乎WAF敏感，连续请求容易触发403拦截。

**输出：** `~/桌面/每日视频总结/2026-05-25_知乎报告.md`

**如果API挂了：**
```bash
# 不用搜索，直接拼URL
browser_navigate(url="https://www.zhihu.com/people/{url_token}/articles")
```

## 三、Cookie提取（统一流程）

**Chrome v147+只能用CDP，任何python/yt-dlp/playwright方案都废了：**

### 启动Chrome调试模式
**只有一个脚本：** `~/桌面/系统工具/启动Chrome调试模式.sh`（其他Chrome脚本已删除）
**启动方式：** 在系统工具文件夹里**双击「启动Chrome调试模式.sh」**
**验证：** 终端运行 `curl http://127.0.0.1:9222/json/version` 看返回
**注意：** terminal里直接跑会因X权限问题失败，必须从桌面图形环境双击启动
**🤷 如果Hermes自己搞不定：** 直接喊用户「请双击桌面→系统工具→启动Chrome调试模式.sh」，等8秒再连

### 如果没有图形界面可用：
```bash
# 用xauth获取认证
xauth merge /home/fw/.Xauthority
export DISPLAY=:0
# 或直接用headless模式（不需要显示器）
google-chrome --headless --remote-debugging-port=9222 --no-sandbox --user-data-dir=$HOME/.config/google-chrome-debug --disable-gpu &
```

### 提取cookie
```bash
# 1. 先确保Chrome调试模式运行中
curl http://127.0.0.1:9222/json/version

# 2. 提取cookie
python3 ~/.hermes/scripts/cdp-get-cookies.py \
  --domain douyin.com \
  --output ~/桌面/凭证/

# 3. 验证文件存在
ls -la ~/桌面/凭证/{bilibili,zhihu,douyin}_com_cookie_最新.txt
```

**三个cookie文件位置（写死，不动路径）：**
```
~/桌面/凭证/bilibili_com_cookie_最新.txt  ← 972B（2026-05-25）
~/桌面/凭证/douyin_com_cookie_最新.txt   ← 6.2KB（2026-05-25）
~/桌面/凭证/zhihu_com_cookie_最新.txt    ← 1.6KB（2026-05-25）
```

**有效期：** 7-30天，过期后需用户重新登录Chrome一次，再跑cdp-get-cookies.py

## 四、三铁律（不遵守会出bug且不告诉你哪错了）

### 铁律1：博主来源统一从INDEX.md读

**所有脚本只能从一个地方读博主列表：`~/.hermes/external_memory/thinkers/INDEX.md`**

| 脚本 | 读哪列 | 格式 |
|:----|:------|:----|
| bilibili_monitor.py | 第四列（B站） | `mid:数字` |
| douyin-monitor.py | 第三列（抖音） | `MS4w...` |
| zhihu_monitor.py | 第五列（知乎） | `people/xxx` |

**禁止行为：** ❌ 从桌面txt读 ❌ 从单独的配置清单读 ❌ 在脚本里写死

### 铁律2：API不行就切浏览器，不绕圈不重试

```
API挂了 → sleep(10) → 重试1次
  ├─ 过了 → 继续
  └─ 不行 → 浏览器拼URL直进主页
             URL从INDEX.md取（mid/sec_uid/people），不搜
```

### 铁律3：说"无更新"必须带证据

```
✅ 付鹏 → 最新: 聊A股近期走势 | 05-22 → 已处理
✅ 宋鸿兵 → API返回空（code=-799过频），浏览器确认最新是05-22
❌ 无新增（没有来源说明）
```
