## 二、三件套积木

### 积木A：cookie-extractor（Chrome < v147）
- 位置：`skills/devops/cookie-extractor`
- 轻量：yt-dlp + curl，秒级完成
- 脚本：`skills/.../scripts/get_cookie.py --domain xxx`
- **限制：** Chrome v147+ 不可用（v11加密不可破解）

### 积木B：waf-bypass（被反爬拦截时）
- 位置：`skills/devops/waf-bypass`
- 方案：切换手机UA / 延时请求 / 切换IP

### 积木C：local-chrome-cdp-bridge（Chrome v147+）
- 位置：`skills/devops/local-chrome-cdp-bridge`
- 脚本：`~/.hermes/scripts/cdp-get-cookies.py --domain xxx`
- **新增2026-05-19：** 全自动流程（crontab重启Chrome → CDP提取cookie → 验证）

## 三、连接三件套：自动重启Chrome（调试模式）

当cookie过期时，**不需要用户操作**，AI自动完成：

### Step 0：判断用哪个积木
```
Chrome v147+  → CDP Bridge（本方案）
Chrome <147   → cookie-extractor（yt-dlp）
不确定版本    → google-chrome --version → 判断
```

### Step 1：重启Chrome（调试模式）
```bash
# 1a. 杀旧Chrome
kill -9 $(ps aux | grep "[c]hrome" | awk '{print $2}') 2>/dev/null

# 1b. 准备临时profile（链接真实用户数据）
rm -rf /tmp/chrome-debug
mkdir -p /tmp/chrome-debug
ln -sf ~/.config/google-chrome/Default /tmp/chrome-debug/Default

# 1c. 通过crontab启动（唯一可靠的Wayland方案）
crontab -l | grep -v "remote-debugging\|chrome-debug\|One-shot" > /tmp/cron_new
echo "# Chrome CDP" >> /tmp/cron_new
echo "$(date -d '+1 minute' '+%M %H %d %m *') WAYLAND_DISPLAY=wayland-0 XDG_RUNTIME_DIR=/run/user/1000 DBUS_SESSION_BUS_ADDRESS=unix:path=/run/user/1000/bus google-chrome --remote-debugging-port=9222 --remote-allow-origins=* --user-data-dir=/tmp/chrome-debug --no-first-run --ozone-platform=wayland > /tmp/chrome_cdp.log 2>&1" >> /tmp/cron_new
crontab /tmp/cron_new
rm /tmp/cron_new
```

### Step 2：等待端口就绪
```bash
for i in $(seq 1 30); do
  curl -s --connect-timeout 2 http://127.0.0.1:9222/json/version && break
  sleep 4
done
```

### Step 3：提取cookie
```bash
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain zhihu.com --test --output ~/桌面/
```

### Step 4：用cookie调API
```bash
curl -H "Cookie: $(cat ~/桌面/zhihu_com_cookie_最新.txt)" \
  "https://www.zhihu.com/api/v4/questions/{question_id}/answers?limit=5"
```

## 四、场景：抓知乎帖子

### 总流程
```
用户说"帮我看看XXX这个知乎话题"
  │
  ▼
① 检查cookie是否有效 → curl测试zhihu API
  ├─ HTTP 200 → 直接用cookie抓数据
  └─ HTTP 401/403 → 执行"三、连接三件套"提取新cookie
  │
  ▼
② 调知乎API拿数据
  └─ GET https://www.zhihu.com/api/v4/questions/{id}/answers
     └─ GET https://www.zhihu.com/api/v4/answers/{id}/comments
  │
  ▼
③ 解析结果 → 提炼观点 → 存入外部记忆/回复用户
```

### 已有技能
```
└── skills/devops/zhihu-reader/（专门读知乎内容）
    └── cookie-extractor / cdp-bridge（提供cookie）
```

### 常用知乎API端点
| 用途 | API | 需要cookie |
|------|-----|-----------|
| 问题答案 | `GET /api/v4/questions/{id}/answers?limit=5&order=default` | ❌ 不强制 |
| 推荐文章 | `GET /api/v3/feed/topstory/recommend?limit=5` | ✅ 必须 |
| 评论 | `GET /api/v4/answers/{id}/comments?limit=10` | ❌ 不强制 |
| 搜索结果 | `GET /api/v4/search_v3?q={keyword}&limit=10` | ✅ 必须（否则频繁报429） |
| 用户时间线 | `GET /api/v4/members/{url_token}/activities?limit=10` | ✅ 必须 |

## 五、推广到任意网站

对于任何需要登录态才能读内容的网站，流程通用：

```
① 判断Chrome版本 → 选积木
② 提取cookie（chrome v147+ → CDP / chrome <147 → yt-dlp）
③ 用cookie调目标网站API
④ 解析返回数据
⑤ 存入外部记忆/汇报用户
```

### 添加新网站步骤
```bash
# 1. 提取cookie（积木C）
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain new-site.com --test

# 2. 找API端点（浏览器F12 → Network → 找需要登录的请求）
# 3. 用curl测试
curl -H "Cookie: $(cat ~/.hermes/cdp_cookies/new_site_com_cookie.txt)" \
  -H "User-Agent: Mozilla/5.0" \
  "https://api.new-site.com/v1/user/me"

# 4. 写成专用reader脚本，存在skills/下
# 5. 更新本总纲
```

## 六、已知坑（铁律）

1. **Chrome v147+ 的 v11 cookie加密不可破解** — 不要试任何Python解密方案，直接走CDP
2. **`--user-data-dir` 必须用非默认目录** — 否则Chrome拒绝开调试端口
3. **`--remote-allow-origins=*` 必须加** — 否则Python WebSocket连不上
4. **Wayland下不能设DISPLAY=:0** — 那是X11变量，Chrome会报"Missing X server"
5. **不要动用户Chrome进程** — 除非确定他在睡觉/不在用。重启前先报备
6. **Cookie会过期** — z_c0 约7-30天。过期了重新执行CDP流程即可
7. **crontab重启Chrome后标签页自动恢复** — Chrome有这个功能，无需额外操作
8. **桌面cookie备份文件命名** — `{域名}_cookie_最新.txt`，用户可以双击查看
