## 一、哲学：先API，不行就浏览器

**数据获取只有两步，没有第三步：**

```
第①步：API → 最快最轻
  ├─ cookie存 ~/桌面/凭证/{域名}_cookie_最新.txt
  └─ curl -H "Cookie: $(cat ~/桌面/凭证/xxx_cookie_最新.txt)" URL
       ↓
API挂了（403/WAF/验证码/空数据）→ sleep(10)→重试1次
       ↓
还不行 → 第②步
       ↓
第②步：浏览器直读（用CDP控制Chrome）
  从INDEX.md取ID → 拼URL → browser_navigate直进主页
  不搜！不搜！不搜！
```

## 三、Cookie提取（Chrome v147+ CDP方案）

Chrome v147+的cookie用v11加密，**任何Python/yt-dlp方案都是废的**，只能走CDP。

### 前置：启动Chrome调试模式

```bash
bash ~/桌面/脚本/启动Chrome调试模式.sh
# 等几秒 → 验证端口通：curl http://127.0.0.1:9222/json/version
```

### 提取cookie

```bash
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain zhihu.com --test --output ~/桌面/凭证/
# 输出：~/桌面/凭证/zhihu_com_cookie_最新.txt
```

### 已验证可用的cookie

| 平台 | cookie文件 | 有效期 |
|:----|:----------|:------|
| 抖音 | `douyin_com_cookie_最新.txt` | ~7-30天 |
| 知乎 | `zhihu_com_cookie_最新.txt` | ~7-30天（需用户手动登录一次） |
| B站 | `bilibili_com_cookie_最新.txt` | ~7-30天 |

## 四、三铁律（不遵守会被用户骂）

### 铁律1：API不行就切浏览器，不绕圈

```
API被封 → sleep(10)→重试1次
  ├─ 过了 → 继续
  └─ 不行 → 浏览器走起（URL从INDEX.md拼）
```

**禁止行为：** ❌ 换UA重试3次 ❌ 搜替代API ❌ 换IP ❌ 爬虫方案 ❌ 猜URL

### 铁律2：浏览器直拼URL，不在搜索框搜

```
✅ 正确：从INDEX.md取mid → https://space.bilibili.com/{mid}
❌ 错误：在B站搜索框里搜用户名
```

### 铁律3：CDP cookie存 ~/桌面/凭证/，不动路径

```
✅ 正确：~/桌面/凭证/zhihu_com_cookie_最新.txt
❌ 错误：~/桌面/zhihu_com_cookie_最新.txt
❌ 错误：/tmp/bili_cookies.txt
```

## 五、各平台特殊情况

### 抖音
- **监控流程：** 每天13:00 cron → douyin_monitor.py → 检测新视频 → 下载→ASR→分析
- **已监控博主：** 伊娃、宋鸿兵、付鹏、蒋宇飞、岩松笔记、拿幸等（见thinkers/INDEX.md）
- **浏览器方案：** 拼sec_uid URL直接进主页

### B站
- **监控流程：** 每天13:00 cron → bilibili_monitor.py → 检测新视频
- **API优先：** 带cookie调space/arc/search
- **浏览器后备：** 拼mid URL直接进空间
- **完整管线：** 详见16-B站视频处理管线.md

### 知乎
- **监控流程：** 每天13:00 cron → zhihu_monitor.py → 文章+回答
- **注意点：** 知乎WAF最严，必须用CDP提取cookie
- **阅读范围：** 只读最近10-20天，文章和回答都要看
- **浏览器方案：** 拼people/xxx URL（注意：知乎页面加载慢，要等JS渲染完）

## 铁律3：验证"无更新"必须有证据链（2026-05-25 用户锁死）

**当说一个博主「今日无新增」时，必须同时返回：最新内容标题 + 发布日期 + 是在哪查到的（API/浏览器）**

```
✅ 付鹏  → 最新: 聊A股近期走势 | 05-22 → 已处理（无新增）
❌ 付鹏  → 无新增（没有来源说明）
```

API坏了就马上说：「B站cookie过期了，全部-799，查不到任何人的最新更新」
浏览器也封了就说：「扫不了，XX平台可能封了」

## 六、已废弃/不要做的事

| 废弃方案 | 原因 |
|:--------|:-----|
| yt-dlp提取cookie | Chrome v147+ v11加密不可破 |
| source_credibility.py | 用户骂"当数据库整""越整越黑箱"，已删除 |

| 在外部记忆建JSON/数据库 | 用户禁令：所有数据用markdown |
| 爬虫/scrapy/selenium | 太重，CDP浏览器够用 |
