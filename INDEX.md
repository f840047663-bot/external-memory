> 本文档是**唯一入口**。任何AI读完本文档，即可完整接手全部工作。
> 
> L3（本文档+链接到的文件）= **信息完备层**，L2（语义关系）和 L1（注意力路牌）均可从L3推导。
> 
> 最后更新：2026-05-23 | 文件总数精简中

### ⚠️ 开机必读铁律：先定性，后定量
**处理任何信息、更新任何P值前，必须回答：这是基本面事件、系统性因子、还是噪声？**
- 基本面事件 → 正常pp加减
- 系统性因子 → 统一折价系数，不分资产各自打分
- 噪声 → 忽略
**跳步就是胡扯。** 见 workflow-info-processing.md 第0步 | 21-贝叶斯防偏差规则.md 第7章

## 二、核心工作流程

### 流程A：处理博主新信息（日常最多的工作）

```
收到博主新视频/文章
     │
Step 1 信息分层
     ├── L1 事实/数据（具体数字、公告、可验证事件）
     │   → 写进 events/{日期}.md + thinkers/{博主}.md
     ├── L2 洞察/逻辑（推理链、因果关系、时间窗口）
     │   → 写进 thinkers/{博主}.md + 作为贝叶斯推理依据
     └── L3 框架/原则（可复用投资原则，如"越靠近GPU越值钱"）
         → 还要嵌入 00-宏观传导框架.md
     │
Step 2 影响哪些持仓？
     ├── 在 events/{日期}.md 里列全（持仓 | 方向 | 逻辑）
     └── 如果来自博主 → 在 thinkers/{博主}.md 也加一列
     │
Step 3 贝叶斯更新
     ├── 打开 positions/{资产}.md
     ├── 写：以前P(上行)=X% 因为 A 理由
     ├── 写：新信息改变了什么 → 先验变化描述
     ├── 写：贝叶斯更新（因子 | 调整 | 依据）
     └── 写：合计 ±Xpp | P(上行) X%→Y%
     │
Step 4 更新聚合索引
     └── 23-贝叶斯更新日志.md 追加一行 + 更新当前概率总览
     │
Step 5 验证互锁（见第四节）
     └── 检查所有受影响文件是否已更新
```

### 流程B：每日日报

```
Step 1（13:00）抖音监控 → 博主动态
Step 2（13:00-14:00）下载→转录→分析
Step 3（14:30）三阶过滤
  Gate1：跟哪个资产相关？
  Gate2：证据等级够不够？
  Gate3：置信度变化≥10% 或 框架级原则？
Step 4（15:00）走流程A（信息→贝叶斯→仓位）
Step 5（15:30）生成日报 → 发邮件
```

### 流程C：新AI接手

```
1. 读本文档（INDEX.md）→ 完全理解架构
2. 读 skills/ → 知道有什么工具、怎么用
3. 读 23-贝叶斯更新日志.md → 知道所有持仓的当前状态
4. 读 00-宏观传导框架.md → 理解宏观视角
5. 按需读 positions/{资产}.md → 知道每个持仓的完整决策链
6. 按需读 events/ 和 thinkers/ → 知道最近发生了什么
7. 按流程A工作
```

### 流程F：问资产审核（用户说"看看XX"时 — 绝对不准只看价格）

每次问任何资产，必须完整走这套：

\`\`\`
Step 1: 🌍 宏观层
  美债位置？美联储态度？地缘？汇率？

Step 2: 🏭 中观层
  行业基本面？供需格局？护城河？政策？

Step 3: 🔬 微观层
  P值(最新贝叶斯) + 仓位(金额+占比+浮盈) + 三关结果

Step 4: 🎯 3选1动作
  ① 不加不减（时机未到/仓位已到）
  ② 加仓（P>60%+三关全过 或 价格跌+三关全过=打折促销）
  ③ 减仓/清仓（P<50%+三关不过 / P≤20%无条件清）

⚠️ 结论必须落脚到动作，不准出现"等等看""待跟进"
详见 27-投资原则-巴菲特审核框架.md
\`\`\`

### 流程D：前瞻扫描（每周日10:00自动）

```
1. 读 前瞻扫描框架.md → 确认扫描框架
2. 流动性（BTC）：财联社搜比特币，判断风险偏好
3. 黄金：全球资产占比+实际利率
4. 板块估值：按资产类型选温度计（PE/PB/PEG），查历史百分位
5. 宏观：利率/PMI/社融/美债
6. 综合判断：环境在变好/变差/模糊
7. 输出前瞻周报 → 写入 events/当天
```

### 流程E：决策回看（每两周一次14:00自动）

```
1. 读 decisions/ 过去14天的决策记录
2. 每条决策对比当下结果（涨跌/逻辑验证）
3. 给信息源打分（准了+分/错了-分，单次最多±8pp）
4. 更新 thinkers/{博主}.md 的信誉评分
5. 输出回看报告 → 写入 events/当天
```

## 六、铁律（不可违反）

1. **没看过/没转录的内容不准概括。** 宁可写「没看过」也不能编。
2. **贝叶斯更新必须有"以前怎么想→现在怎么想"的信念变化说明。** 禁止只写"利好+3pp"。
3. **修正不覆盖，追加修正。** 判断错了→追加一条修正记录，保留原始记录。
4. **每项资产一定要关联到宏观层。** 不能说"涨了"就结束，要说清楚为什么。
5. **不自作主张改配置。** 改配置前备份+通知小龙虾，改完10分钟互相确认。
6. **系统卡死第一反应**：`bash ~/桌面/修复卡死.sh`
7. **搞不定的事不硬磕。** 3次工具调用没结果→问用户或找更强模型。
8. **审核资产必用估值方法论。** 用户说"看看/审核XX"时，必须走流程F：三层(宏观→中观→微观)+三关(懂不懂/护城河/管理层)+3选1(不动/加/减)。不准只报价格涨跌。详见27-投资原则-巴菲特审核框架.md。
9. **图片不能看就用OCR。** 用户发照片时，用`image-ocr`技能（OCR.space免费API）提取文字，不准说"我看不了图"就放弃。

## 八、耦合解耦 + 反脆弱 设计说明

### 耦合解耦

| 应该耦合的 | 应该解耦的 |
|:----------|:-----------|
| events ←→ positions（事件↔持仓影响） | 一个博主视频→不耦合到其他博主 |
| positions ←→ 23-贝叶斯（每个持仓↔总览） | 用户配置→不耦合到分析流程 |
| **B站管线 → events/ → positions/** | **B站管线的7个模块之间互不耦合（可单独换）** |
| thinkers → events（谁说了什么） | 持仓的决策链→不依赖特定AI的记忆 |

### 反脆弱设计

1. **L3信息完备**：硬盘不坏→任何AI都能从INDEX.md+skills/+positions/重建整个系统
2. **对称恢复**：Hermes崩了→小龙虾读INDEX.md接手；小龙虾崩了→Hermes读同样文件接手
3. **错误不覆盖**：错误的判断是数据，追加修正不删除原始记录
4. **概率不固化**：P(上行)随新证据动态更新，月校准防偏差δ²ⁿ灾难
