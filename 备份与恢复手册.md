# 记忆恢复指南（从百度网盘）

> 如果 Hermes 崩了、换新AI了、或者新装系统需要恢复记忆，按此流程操作。

## 备份内容

L1+L2 打包上传到百度网盘 `/Hermes备份/`，每3天自动上传一次。

| 层 | 内容 | 文件 |
|:--|:-----|:-----|
| L1 | 系统铁律、用户画像 | `MEMORY.md` + `USER.md` |
| L2 | hindsight向量库（551条事实） | `~/.pg0/` 目录 |

## 恢复步骤

### 1. 从百度网盘下载备份

用 bypy 下载最新备份：

```bash
# 先授权（如果还没配）
bypy quota  # 能显示配额说明已授权

# 列出备份目录
bypy ls /Hermes备份/ -t 60

# 下载最新的完整备份目录
REMOTE_DIR="/Hermes备份/20260522_XXXXXX"  # 替换成最新的日期
LOCAL_DIR="$HOME/hermes-restore-$DATE"
mkdir -p "$LOCAL_DIR"

# 下载所有分卷
bypy downdir "$REMOTE_DIR" "$LOCAL_DIR" -t 120
```

### 2. 合并分卷

```bash
cd "$LOCAL_DIR"
cat part_* > backup.tar.gz
```

### 3. 恢复L1内部记忆

```bash
# 停 Hermes（防止文件锁）
hermes stop

# 恢复 L1
tar xzf backup.tar.gz \
  -C ~/.hermes memories/MEMORY.md memories/USER.md

# 重启 Hermes
hermes start
```

### 4. 恢复L2向量库

```bash
# 停 hindsight
kill -9 $(pgrep -f hindsight-api) 2>/dev/null || true
sleep 2

# 恢复 L2
tar xzf backup.tar.gz -C ~ .pg0

# 重启 hindsight
hindsight serve  # 或启动 Hermes（会自动拉起）
```

### 5. 验证恢复

```bash
# 验证 L1
diff ~/.hermes/memories/MEMORY.md <(echo "已恢复") && echo "L1 OK"

# 验证 L2（需要 hindsight 运行中）
curl localhost:9177/v1/default/banks/hermes/stats
# 应有 500+ 条记忆
```

## 注意事项

- 百度网盘 bypy 网络较慢，每个命令加 `-t 120`（120秒超时）
- L2 向量库约 300MB，下载可能较慢
- 如果只需要 L1（系统铁律），只需恢复 `memories/` 目录，无需等L2下载
- L1 也同时在 GitHub 上备份（`external_memory/memories/MEMORY.md`），可以直接从 GitHub 下载
