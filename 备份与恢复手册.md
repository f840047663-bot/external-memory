# 备份方案总览与使用说明

## 一、备份架构

```
┌─────────────────────────────────────────────────┐
│                  Hermes 备份体系                   │
├────────────┬──────────────┬──────────────────────┤
│    层      │   存储位置    │      同步方式        │
├────────────┼──────────────┼──────────────────────┤
│  L1 内部记忆 │ GitHub 仓库   │ 每30分钟自动同步     │
│  (MEMORY.md │              │ (crontab触发,       │
│   +USER.md) │              │ 先copy到external_   │
│            │              │ memory再commit+push) │
├────────────┼──────────────┼──────────────────────┤
│  L1 + L2   │ 百度网盘      │ 每3天自动上传        │
│  (携带向量库) │ /Hermes备份/  │ (Hermes cronjob,    │
│            │              │ 分卷20MB×N个上传)    │
├────────────┼──────────────┼──────────────────────┤
│  L3 外部记忆 │ GitHub 仓库   │ 每30分钟自动同步     │
│  (全部MD文件)│              │ (已配置多年)         │
└────────────┴──────────────┴──────────────────────┘
```

## 二、关键文件说明

### GitHub仓库（external_memory/）

> 地址：`github.com/f840047663-bot/external-memory`
> 本地路径：`~/.hermes/external_memory/`（桌面 `Hermes外部记忆` 是软链接）

| 文件/目录 | 说明 |
|:---------|:-----|
| `INDEX.md` | **总导航** — 新AI/新人第一份必读文件 |
| `memories/MEMORY.md` | L1 — 系统铁律、工作流、所有规则 |
| `memories/USER.md` | L1 — 用户画像、偏好、习惯 |
| `备份存档/内部记忆/` | L1按时间戳的历史版本 |
| `skills/*.md` | 全部62个技能备份（Hermes skills的平铺版本） |
| `备份与恢复手册.md` | 本文件 |
| `00-*.md` 至 `99-*.md` | L3外部知识库全部文档 |

### 百度网盘（/Hermes备份/）

> 路径：`/apps/bypy/Hermes备份/`（bypy默认前缀）
> 格式：`/Hermes备份/YYYYMMDD_HHMMSS/`

| 文件 | 说明 |
|:----|:-----|
| `part_00 ~ part_N` | 分卷压缩包（每卷20MB） |
| `合并.sh` | 合并脚本：`cat part_* > backup.tar.gz` |

备份文件 `backup.tar.gz` 内包含：
```
memories/MEMORY.md       ← L1 系统规则
memories/USER.md         ← L1 用户画像
pg0/...                  ← L2 向量数据库（~300MB）
```

## 三、恢复方法

### 场景A：只需要L1（系统规则）— 从GitHub

最简单，几秒钟的事：

```bash
# 克隆仓库（如果还没有）
git clone https://github.com/f840047663-bot/external-memory.git
# 或直接拉最新
cd external-memory && git pull

# 复制L1到Hermes
cp memories/MEMORY.md ~/.hermes/memories/MEMORY.md
cp memories/USER.md ~/.hermes/memories/USER.md
```

### 场景B：需要L1+L2完整恢复 — 从百度网盘

```bash
# 1. 确认 bypy 已授权
bypy quota -t 60

# 2. 列出备份目录，找到最新的
bypy ls /Hermes备份/ -t 60

# 3. 下载最新备份
DATE="20260522_XXXXXX"  # 替换为最新目录名
bypy downdir "/Hermes备份/$DATE" ~/hermes-restore -t 120

# 4. 合并分卷
cd ~/hermes-restore
bash 合并.sh   # 会生成 backup.tar.gz

# 5. 停服务 → 恢复 → 重启
hermes stop
kill -9 $(pgrep -f hindsight-api) 2>/dev/null; sleep 2

tar xzf backup.tar.gz \
  -C ~/.hermes memories/MEMORY.md memories/USER.md
tar xzf backup.tar.gz -C ~ .pg0

hermes start
```

### 场景C：换新AI

新AI读 Git仓库 → 按 INDEX.md 导航 → 加载 skills/ 全部技能 → 读 L1 记忆 → 开工。

需要L2历史记忆的话，从百度网盘下载恢复。

## 四、定时任务一览

```bash
# 系统 crontab — 每30分钟自动同步L1+L3到GitHub
*/30 * * * * cp ~/.hermes/memories/MEMORY.md ~/.hermes/external_memory/memories/ && cp ~/.hermes/memories/USER.md ~/.hermes/external_memory/memories/ && cd ~/.hermes/external_memory && git add -A && git commit -m "auto" && git push

# Hermes cron — 每3天上传L1+L2到百度网盘
# 名称: L1L2百度网盘备份
# 触发: every 3 days
# 脚本: ~/.hermes/scripts/bak-l1l2-baidu.sh
# 行为: 打包→分卷→逐个上传→清临时文件
```

## 五、首次设置（已经做完，仅作记录）

```bash
# 1. 装 bypy
pipx install bypy

# 2. 授权（只需要做一次）
bypy quota    # 按提示打开链接 → 授权 → 粘贴授权码

# 3. token 保存在 ~/.bypy/bypy.json
# 4. token 有效期30天，bypy自动refresh
```

## 六、注意事项

- 百度网盘 bypy 国内网络较慢，所有命令加 `-t 60`（60秒超时）
- 上传大文件时分卷（20MB/卷），小文件直传
- L1 + L3 全在 GitHub 上，随时可读，不需要百度网盘
- L2 向量库约300MB，只有百度网盘上有
- 百度网盘总共2TB，已用435GB，空间充裕
