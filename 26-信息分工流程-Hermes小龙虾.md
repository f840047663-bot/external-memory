# 26-信息分工流程：Hermes + 小龙虾

> 建档：2026-05-24 | 大好人确定的分工方案
> 目标：信息闭环，每步有人干，不浪费采集不重复打分

---

## 分工总图

```
小龙虾（收集层 + 输出层）
  │
  ├─ 定时任务拉视频/监控（抖音/B站/知乎）
  ├─ 写报告时搜集当日重要新闻
  │
  ▼ 原文推给 Hermes

Hermes（分析层 — 贝叶斯决策管道）
  │
  ├─ 收小龙虾推来的原文
  ├─ 走5道门：A(定性)→B(宏观折价)→C(读positions)→D(定性复核)→E(去重)
  ├─ 算出P值调整
  │
  ▼ 分数+分析推回给小龙虾

小龙虾（输出层）
  │
  ├─ 收 Hermes 的P值调整
  ├─ 写入报告（来源+分数+逻辑完整呈现）
  └─ 发邮件/推送到微信
```

## 核心原则

1. **P值只有一个来源** — Hermes 走5道门，小龙虾不自己打分
2. **双通道采集** — 小龙虾监控脚本 + Hermes监控管道，两边新闻都进贝叶斯管道
3. **小龙虾不背贝叶斯框架** — 他只做采集和输出
4. **Hermes不依赖小龙虾的采集** — 监控管道独立运行，两边可以互相补漏
5. **闭环校验** — 报告里每项P值调整必须有"来源→分析→分数"的完整链路

## 运行评估（2026-05-26前）

跑2天后评估小龙虾搜集的新闻质量：
- ✅ 有增量信息（新事件、新数据）→ 保留双通道
- ❌ 全是已定价背景信息（如"消费弱"重复）→ 取消小龙虾的采集任务，采集全归监控管道

## 铁律

- ❌ 小龙虾收到原文后不自己算分，原文推送即可
- ❌ Hermes 收到分数后不质疑自己的5道门结果（除非新信息进来）
- ✅ 报告里必须写清楚「来源→Hermes判断→pp调整」
- ✅ 如果分歧 → 大好人一票裁定
