# 监控统筹框架

> **最后更新：** 2026-06-01
> **设计原则：** 状态驱动。AI不需要"想怎么走"，只需要"读状态→执行→写状态"。
> **⚠️ 本文档是调度器，<100行。每个模块只写"调哪个子模块"，不重复内容。**

---

## 执行顺序

```
[环境准备] → [抓取/阶段一] → [转录/阶段二] → [分析/阶段三] → [收尾/阶段四]
                                                     ↓
                                           有API失败项且用户同意？
                                                     ↓ 是
                                            [浏览器/兜底]
                                                     ↓
                                            回到对应阶段重试
```

**遇到障碍？** → 调 [决策树/排错]，不要自己猜。

---

## 状态文件：`/tmp/monitor_state.json`

每个模块执行完**必须写**，下一个模块**必须读**。字段名统一如下：

```json
{
  "当前Step": "环境准备完成",
  "Cookie": {"抖音": "✅", "B站": "✅", "知乎": "❌"},
  "开关": {"调试": "要", "宏观": "不要", "待重仓": "不要", "三关": "不要"},
  "抓取结果": {},
  "失败项": [],
  "待转录": [],
  "转录结果": [],
  "待分析": [],
  "贝叶斯结果": [],
  "断点已更新": false
}
```

## 模块路由表（按顺序执行，不要跳）

| 模块 | 子模块文件 | 输入 | 输出 |
|:----|:----|:----|:----|
| **1.环境准备** | `工作流模块/通用/环境准备.md` | 无 | Cookie状态+开关→写状态文件 |
| **2.抓取/阶段一** | `工作流模块/抓取/阶段一.md` | 读状态文件 | 抓取结果+失败项+待转录→写状态文件 |
| ↳ 抖音子模块 | `工作流模块/抓取/抖音.md` | sec_uid+断点 | 新视频→videos/→写状态清单 |
| ↳ B站子模块 | `工作流模块/抓取/B站.md` | mid+断点 | 新视频→videos/→写状态清单 |
| ↳ 知乎子模块 | `工作流模块/抓取/知乎.md` | url_token+断点 | 新文章→texts/→写状态清单 |
| **3.转录/阶段二** | `工作流模块/转录/阶段二.md` | 待转录+失败项 | 转录结果+待分析→写状态文件 |
| **4.分析/阶段三** | `工作流模块/分析/阶段三.md` | 待分析列表 | 贝叶斯结果+归档→写状态文件 |
| **5.收尾/阶段四** | `工作流模块/收尾/阶段四.md` | 全部结果 | 断点+日报+汇报→写状态文件 |
| **6.浏览器/兜底** | `工作流模块/浏览器/兜底.md` | 失败项 | 成功项+仍失败项 |
| **7.决策树/排错** | `工作流模块/通用/决策树排错.md` | 错误信息 | 解决方案或"问用户" |

## 执行规则

1. **按顺序**，不跳步。每步输出结果，等用户说"继续"
2. **每阶段间必须更新 L1.5_工作记忆.md** — 追加一行：`[阶段X完成 HH:MM] 关键数据`
3. **调试模式**：每个模块print返回值，失败项单独标红
4. **阶段一失败项** → 问"是否浏览器兜底？" → 是→调模块6→回阶段一重试
5. **全部完成** → 模块5自检10项 → 汇报用户 → **清空L1.5**
6. **中断恢复** → 读L1.5_工作记忆.md + `/tmp/monitor_state.json` + `/tmp/监控运行日志.md`最后20行
7. **遇到问题** → 调模块7，不自己猜

**运行日志：** `~/桌面/监控运行日志.md` — 每处理完一条内容append一行
