## ⚠️ 执行约束（先读这条，再走流程）

1. **先读流程再执行** — 每步必须读对应模块，禁止凭记忆操作
2. **卡住→先查模块→查原文件→再问用户** — 不自己猜方案
3. **调试输出** — 每步输出：调用模块 + 返回值 + 决策结果
4. **Step之间停顿** — 等用户说"继续"再走下一步

## 🚨 遇到问题决策树

```
API返回空/JSON解析失败
  → 不是Cookie过期！先sleep(10s)重试
  → 还失败 → sleep(15s)再重试
  → 还失败 → 检查Cookie → 无效则刷新 / 有效则标记"失败-待重试"

API返回-799/403（限流）
  → sleep(15s)重试1次
  → 还限流 → 换下一个平台（三平台交替）
  → 阶段二回头重试

Cookie无效
  → 调用[通用/环境准备]刷新

转录报"Unsupported sound format: mp4"
  → 必须先ffmpeg提mp3，不能直接传mp4

以上都试了还不行
  → 汇报用户，不闷头试
```

### 🚫 禁止行为
- 看到空返回就跳过（必须sleep重试至少2次）
- 看到空返回就刷新Cookie（先重试，可能是间歇性问题）
- 连续重试同一个平台不换气（必须三平台交替）
- 遇到error自己猜方案（先查决策树→查模块→再问用户）
- Cookie无效还继续跑流程（必须先刷新）
- 转录时直接传mp4（必须先ffmpeg提mp3）

## 主线流程总览（四阶段流水线）

```
第-1步：内存检查+清理
    ↓
第〇步：甲乙丙丁戊己（五个开关 + Cookie检查）
    ↓
╔═══════════╗
║ 阶段一    ║  ①读断点 ②拉列表+比对+筛选 ③写抓取状态清单
║ 抓取      ║  三平台交替 + 失败自动重试
╚═══════════╝
    ↓
╔═══════════╗
║ 阶段二    ║  ④下载+转录 ⑤重试失败项
║ 转录      ║  转录天然限速
╚═══════════╝
    ↓
╔═══════════╗
║ 阶段三    ║  ⑥提炼观点 ⑦贝叶斯更新 ⑧归档
║ 分析      ║  ⛔⑨异动自动审核（if块，归档完必须判断）
╚═══════════╝
    ↓
╔═══════════╗
║ 阶段四    ║  ⑩更新断点+校验 ⑪出日报 ⑫强制汇报
║ 收尾      ║
╚═══════════╝
```

**抓取和分析严格分离：阶段一/二只负责拿到内容存文件，阶段三才做分析。**

## Step -1：内存检查

**调用模块：** `[通用/内存检查]`

**返回值：** `{状态: ✅/❌, available内存: X GB}`

**决策：**
- ✅ (>3GB) → 跳Step 〇
- ❌ (<3GB) → 执行清理 → 再检查 → 跳Step 〇

---

## Step 〇：环境准备 + 开关确认

**调用模块：** `[通用/环境准备]`

**返回值：**
```
{
  "CDP状态": "✅/❌",
  "Cookie": {"抖音": "✅/❌", "B站": "✅/❌", "知乎": "✅/❌"},
  "开关": {"调试": "要/不要", "宏观审核": "要/不要", "待重仓": "要/不要", "三关": "要/不要"}
}
```

**决策：**
- CDP ❌ → **终止**，提示用户启动Chrome
- Cookie部分❌ → 调用刷新 → 再检查 → 全部✅才继续
- Cookie全❌ → **终止**，提示用户登录
- 开关确认完毕 → 跳Step ①

**标准话术：**
```
第〇步 - 五个开关 + Cookie检查：
抖音：✅/❌（X字节）  B站：✅/❌（X字节）  知乎：✅/❌（X字节）
1.调试输出？ 2.宏观审核？ 3.待重仓？ 4.三关？
（"直接跑"=全不要，"全开"=全要）
```

## Step ①-③：阶段一（抓取）

**调用模块：** `[抓取/阶段一抓取]`

**输入：** 开关配置（调试模式是否开启）

**返回值：**
```
{
  "状态清单": "/tmp/monitor_status.md",
  "统计": {"成功": X, "失败待重试": Y, "0新": Z},
  "待转录列表": ["视频ID列表"],
  "待分析列表": ["知乎文本路径列表"]
}
```

**决策：**
- 待转录+待分析都为空 → **跳Step ⑩**（直接收尾汇报：今日无新内容）
- 有内容 → **停顿**，问用户"是否继续转录分析？" → 跳Step ④

**核心规则（在模块内）：**
- 三平台交替请求（抖音A→B站A→知乎A→...）
- 转录天然限速（有新视频就转录，0新sleep 12s）
- 失败自动重试2次（sleep 10s → sleep 15s）
- 每处理完一个博主立刻写`/tmp/monitor_status.md`

---

## Step ④-⑤：阶段二（转录+重试）

**调用模块：** `[转录/阶段二转录]`

**输入：** 待转录列表（来自Step ②）

**返回值：**
```
{
  "转录结果": [{"视频ID": "xxx", "状态": "✅/❌", "文本路径": "xxx"}],
  "重试结果": [{"博主": "xxx", "状态": "✅/❌", "原因": "xxx"}],
  "最终待分析列表": ["所有转录完成+知乎文本的路径"]
}
```

**决策：**
- 全部✅ → 跳Step ⑥
- 部分❌ → **停顿**，问用户"是否重试/跳过？"
- 全❌ → 跳Step ⑩（无内容可分析）

---

## Step ⑥-⑨：阶段三（分析）

**调用模块：** `[分析/阶段三分析]`

**输入：** 最终待分析列表 + 开关配置（宏观审核/三关是否开启）

**返回值：**
```
{
  "状态": "✅/❌",
  "影响持仓": [{"资产": "xxx", "P值变化": "51%→54%", "原因": "xxx"}],
  "异动审核": "已执行/未触发/用户跳过",
  "事件归档": "events/2026-05-31.md"
}
```

**决策：**
- ✅ → 跳Step ⑩
- ❌ → **停顿**，问用户"是否跳过贝叶斯更新？" → 跳Step ⑩

**⛔ 异动审核是if块（不是参考章节）：**
```
IF P值变动≥±3pp OR 重大事件 OR 左侧资产:
  → 问用户"要不要跑完整审核？"
  → 用户说跑 → 宏观前置 + 巴菲特三关
```

---

## Step ⑩-⑫：阶段四（收尾）

**调用模块：** `[收尾/阶段四收尾]`

**输入：** 全部阶段的结果

**返回值：**
```
{
  "断点更新": "✅/❌",
  "日报路径": "xxx",
  "邮件发送": "✅/❌",
  "自检清单": "10/10通过"
}
```

**决策：**
- 全部✅ → **全流程结束**
- 断点❌ → **不允许出日报**，必须先补断点
- 自检缺项 → **不允许发消息给用户**，补完再报

## 断点恢复机制

**每次启动前检查：** `~/监控处理断点.md` + `/tmp/monitor_status.md`

| 状态 | 行为 |
|:----|:-----|
| 无断点/无状态清单 | 从Step -1开始 |
| 有状态清单 | 读清单，从失败的Step继续 |
| 清单显示全部✅ | 从Step ⑩开始（收尾） |

## 🚨 强制自检清单（10项，缺任何一条不准发消息）

```
□ ① 主页ID从INDEX.md读了吗？（不是断点表）
□ ② 长期主题跟踪表≥7天的搜了新进展？
□ ③ P值链可追溯？（事件链累加=文件头P）
□ ④ P'虚值标了吗？（未落地事件用P'）
□ ⑤ events/当天.md写了吗？
□ ⑥ hindsight_retain存了吗？
□ ⑦ 抓取状态清单写了？（/tmp/monitor_status.md）
□ ⑧ 失败项重试了？
□ ⑨ 内容保存到暂存目录了？（不是只打印）
□ ⑩ 三平台交替了？（没有一口气跑完一个平台）
```

## 调试模式输出格式

**调试模式 = 不允许黑箱！每个博主必须输出：**
1. 断点ID + 断点时间 + 断点标题
2. API返回状态（成功/失败/限流）
3. API返回的视频列表（前5条：ID+时间+标题）
4. 比对过程（断点ID位置 + 新视频判定）
5. 最终结论：0新 / X新（列新视频ID+时间+标题）

**API失败额外输出：** 每次请求的原始错误 + sleep时间 + Cookie检查结果

**🚫 禁止：** 直接说"API失败"跳过 / 直接给结论不给原始数据 / 3个博主合并输出
