# 监控统筹框架

> **最后更新：** 2026-05-31
> **设计原则：** 状态驱动，不是判断驱动。AI不需要"想怎么走"，只需要"读状态→执行→写状态"。
> **⚠️ 本文档是调度器，不是执行手册。每个模块只写"去读哪个文件的哪一节"，不重复内容。**

---

## 执行顺序

```
[环境准备] → [抓取/阶段一] → [转录/阶段二] → [分析/阶段三] → [收尾/阶段四]
                                                     ↓
                                           有API失败项且用户同意？
                                                     ↓ 是
                                            [浏览器/兜底]
                                                     ↓
                                            回到对应阶段重试
```

**遇到障碍？** → 读 [决策树/排错] 模块，不要自己猜。

---

## 状态文件：`/tmp/monitor_state.json`

每个模块执行完**必须写**，下一个模块**必须读**。

- **写状态：** 抓取结果 + 失败项 + 待转录列表
- **铁律：** 三平台交替请求，每处理完一个博主立刻写 `/tmp/monitor_status.md`

### 模块3：[转录/阶段二]
- **读：** `监控全流程-刻下来.md` §④（下载+转录）+ §⑤（重试失败项）

- **铁律：** 断点更新前必须确认events/thinkers/positions/日志全部归档完成

### 模块6：[浏览器/兜底]（条件触发）

- **读：** `Chrome-CDP-操作手册.md`（CDP详细操作，仅当需要时）
- **完成后：** 回到对应阶段重试

### 模块7：[决策树/排错]（遇到问题时读）
- **读：** `监控全流程-刻下来.md` §决策树（第1-4步决策树 + 禁止行为）
- **读：** `00-网站内容抓取总纲.md` §七（排错档案8条）
- **原则：** 先查决策树→再查排错档案→还解决不了→问用户

## 执行规则

1. **按顺序执行**，不要跳步
2. **每步之间停顿**，输出结果，等用户说"继续"
3. **阶段一有失败项** → 问用户"是否用浏览器兜底？" → 用户说要 → 调[浏览器/兜底] → 回到阶段一重试
4. **全部完成** → 强制自检10项 → 汇报用户
5. **中断恢复** → 读状态文件 + 读 `/tmp/monitor_status.md`，从断点Step继续

## 强制自检清单（阶段四执行，缺任何一条不准发消息）

```
□ ①主页ID从INDEX.md读的  ②长期主题≥7天搜了  ③P值链可追溯
□ ④P'虚值标了  ⑤events写了  ⑥hindsight_retain存了
□ ⑦状态清单写了  ⑧失败项重试了  ⑨内容存暂存目录  ⑩三平台交替了
```
