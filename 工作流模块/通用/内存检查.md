# 内存检查

## 输入
- 无

## 输出（返回值）
| 字段 | 类型 | 说明 |
|:----|:----|:----|
| 状态 | ✅/❌ | available内存是否>3GB |
| available | 字符串 | 可用内存（如"5.2G"） |
| 已清理 | ✅/❌ | 是否执行了清理 |

## 执行步骤

### Step 1：检查内存
```bash
free -h | grep Mem
# 看available列：>3GB就不用管，<3GB就跑清理脚本
```

### Step 2：如果<3GB，跑清理脚本
```bash
bash ~/桌面/脚本/内存急救.sh
```

### Step 3：验证清理效果
```bash
free -h | grep Mem
```

### Step 4：如果清理脚本不管用（hindsight自动重生）
```bash
# 手动杀旧hindsight，保留最新的（最高PID）
PIDS=$(ps aux | grep 'hindsight_api.main' | grep -v grep | awk '{print $2, $6}' | sort -k2 -n)
echo "$PIDS" | head -n -1 | awk '{print $1}' | xargs kill 2>/dev/null
sleep 2
free -h
```

## 清理频率
- 流程开始前：必须检查一次
- 每处理5个博主后：检查一次，<3GB就清
- 跑CDP/浏览器操作前：检查一次（Chrome本身吃内存）

## 返回值示例
```
{"状态": "✅", "available": "5.2G", "已清理": "❌"}
```
