# 决策树与排错

> **遇到问题先读这里，不要自己猜方案。**
> **执行顺序：** 决策树 → 排错档案 → 还解决不了→问用户

## 决策树（按顺序走，不要跳）

### 第1步：API返回空/JSON解析失败
```
第1次→空/JSON失败 → sleep(10s) → 第2次→成功✅
                              →失败→sleep(15s)→第3次→成功✅
                                    →失败→检查Cookie(wc -c)
                                      ├─ <500B→调[环境准备]刷新→再试
                                      └─ >1000B→标记"失败-待重试"，继续下一个
```
**知乎特别注意：** 返回空**不是cookie过期**，第一反应是sleep→重试！

### 第2步：API返回-799/403/限流
```
收到-799/403 → sleep(15s) → 重试1次→成功✅
                   →还限流→换下一个平台（三平台交替）
                   →阶段二回头重试（冷却时间够了大概率成功）
```
**🚫 禁止：** 连续重试同一平台 / 限流了只sleep(5秒)

### 第3步：转录报"Unsupported sound format: mp4"
```
原因：transcribe_new.py只接受mp3/wav
解决：docker exec videocaptioner ffmpeg -i v.mp4 -vn -acodec mp3 -y v.mp3
```

### 第3.5步：抖音视频只有h264无aac音频
```
原因：抖音CDN音视频分流，CDP拦截只能拿到纯视频流
解决：调[浏览器/兜底]→Network拦截两个douyinvod URL→ffprobe区分video/audio→ffmpeg合并
```

### 第4步：以上都试了还不行
```
汇报格式：
❌ 失败项：{平台}-{博主}-{类型}
已尝试：1.sleep(10s)重试→失败 2.sleep(15s)再试→失败 3.Cookie检查→{有效/已刷新}
需要你回来处理。
```

## 🚫 禁止行为（违反就会被骂）
```
❌ 看到空返回就跳过（必须sleep重试至少2次）
❌ 看到空返回就刷新Cookie（可能是间歇性问题）
❌ 连续重试同一平台不换气（必须三平台交替）
❌ 遇到error自己猜方案（先查决策树→排错档案→问用户）
❌ Cookie无效还继续跑（必须先刷新）
❌ 转录直接传mp4（必须ffmpeg提mp3）
```

## 排错档案（来自00-网站内容抓取总纲.md §七）

| # | 问题 | 症状 | 修复 |
|:--|:----|:----|:----|
| 1 | 抖音count太小 | API返回空列表，误判私密账号 | 永远用count=30 |
| 2 | 抖音UA太笼统 | 空响应 | 用完整Chrome版本号UA |
| 3 | 抖音Referer不带user路径 | 空响应或条数少 | Referer精确到/user/{sec_uid} |
| 4 | 抖音API非时间倒序 | 误报错位 | 先搜断点ID位置，再比ID大小 |
| 5 | 抖音ID不严格单调 | 个别ID大但时间早 | ID+时间双验证 |
| 6 | JSON解析失败≠API挂了 | 批处理中超时 | 子进程timeout设120秒+ |
| 7 | 知乎文章vs回答 | 只追踪一个类型漏内容 | 拆两行各自独立追踪 |
| 8 | sleep不足→限流 | 跑几个后开始空/异常 | sleep(12秒)，不着急 |

## 崩溃恢复速查（来自00-网站内容抓取总纲.md §六）

| 故障现象 | 原因 | 修复 |
|:--------|:----|:----|
| B站全-799 | API限流 | 等10-30分钟或换平台 |
| 抖音全status_code=5 | cookie过期/CDP连不上 | 重跑cdp-get-cookies.py |
| 知乎403/空 | cookie过期 | 用户重新登录→cdp-get-cookies.py |
| 所有平台全挂 | Chrome调试模式没开 | systemd-run --user --pty启动 |
| 抖音单账号API空 | 限流 | 等5-10分钟重试即恢复 |
| cron no_agent报错 | 脚本路径不对 | 检查job的script字段 |
