# 环境准备（CDP启动 + Cookie检查+刷新 + 开关确认）

## 输入
- 无

## 输出（返回值）
```
{
  "CDP状态": "✅/❌",
  "Cookie": {"抖音": "✅/❌", "B站": "✅/❌", "知乎": "✅/❌"},
  "开关": {"调试": "要/不要", "宏观审核": "要/不要", "待重仓": "要/不要", "三关": "要/不要"}
}
```

## 执行步骤

### Step 1：检查Chrome CDP
```bash
curl -s http://127.0.0.1:9222/json/version
# 返回JSON含"Browser"→✅，否则→启动
```

**启动方式：**
```bash
# systemd服务
systemctl --user enable --now chrome-cdp.service
# 或手动
bash ~/桌面/系统工具/启动Chrome调试模式.sh
```

**⚠️ Chrome每次重启后必须运行：**
```bash
bash ~/系统工具/更新Chrome-CDP-ID.sh
```

### Step 2：检查Cookie有效性
```bash
wc -c ~/.hermes/cookies/douyin_netscape.txt ~/.hermes/cookies/bilibili_netscape.txt ~/.hermes/cookies/zhihu_netscape.txt
```
**判定标准：** >1000字节=✅，<500字节=❌

### Step 3：如果有❌，刷新Cookie
```bash
# 用cdp-get-cookies.py刷新
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain douyin.com --output ~/桌面/凭证/douyin_com_cookie_最新.txt
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain bilibili.com --output ~/桌面/凭证/bilibili_com_cookie_最新.txt
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain zhihu.com --output ~/桌面/凭证/zhihu_com_cookie_最新.txt

# 同步到/tmp（脚本读取位置）
cp ~/桌面/凭证/douyin_com_cookie_最新.txt /tmp/douyin_cookies.txt
cp ~/桌面/凭证/bilibili_com_cookie_最新.txt /tmp/bilibili_cookies.txt
cp ~/桌面/凭证/zhihu_com_cookie_最新.txt /tmp/zhihu_cookies.txt

# 持久化到~/.hermes/cookies/（Netscape格式）
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain douyin.com --output ~/.hermes/cookies/douyin_netscape.txt --format netscape
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain bilibili.com --output ~/.hermes/cookies/bilibili_netscape.txt --format netscape
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain zhihu.com --output ~/.hermes/cookies/zhihu_netscape.txt --format netscape
```

### Step 4：确认五个开关
```
第〇步 - 五个开关 + Cookie检查：
抖音：✅/❌（X字节）  B站：✅/❌（X字节）  知乎：✅/❌（X字节）
1.调试输出？ 2.宏观审核？ 3.待重仓？ 4.三关？
（"直接跑"=全不要，"全开"=全要）
```

**开关说明：**
- 甲：调试输出（要=每博主输出原始数据+停顿；不要=静默跑完）
- 乙：收网检查（读~/.hermes/收网提醒.txt）
- 丙：待重仓监控（读待重仓监控.md）
- 丁：宏观前置审核（读00-宏观传导框架.md）
- 戊：巴菲特三关（读27-投资原则-巴菲特审核框架.md）

**🚨 铁律：五个检查必须全部做完，得到用户回答后，才能进入阶段一。**

## 失败处理
| 故障 | 解决 |
|:----|:-----|
| Chrome启动失败 | 喊用户手动双击启动脚本 |
| Cookie刷新后仍<500字节 | 喊用户："需要你在Chrome登录{平台}" |
| 用户Chrome没开 | 喊用户开Chrome |
