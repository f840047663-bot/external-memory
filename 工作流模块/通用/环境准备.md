# 环境准备

> **输入：** 无
> **输出：** 写`/tmp/monitor_state.json` → 返回{Cookie状态, 开关配置}

## 执行步骤

### 1. 内存检查
```bash
free -h | grep Mem  # available>3GB→✅ | <3GB→bash ~/桌面/脚本/内存急救.sh
```

**清理频率：**
- 流程开始前：必须检查一次
- 每处理5个博主后：检查一次，<3GB就清
- 跑CDP/浏览器操作前：检查一次（Chrome本身吃内存）

**如果清理脚本不管用（hindsight自动重生）：**
```bash
PIDS=$(ps aux | grep 'hindsight_api.main' | grep -v grep | awk '{print $2, $6}' | sort -k2 -n)
echo "$PIDS" | head -n -1 | awk '{print $1}' | xargs kill 2>/dev/null
sleep 2 && free -h
```

### 2. CDP检查+启动
```bash
curl -s http://127.0.0.1:9222/json/version  # 有返回→✅
# 无返回→用systemd-run启动（这台Wayland机器systemd服务文件不可用！）
rm -rf /tmp/chrome-debug && mkdir -p /tmp/chrome-debug
systemd-run --user --pty --service-type=exec \
  --property=Environment="DISPLAY=:0" \
  bash -c 'google-chrome --remote-debugging-port=9222 --remote-allow-origins=* \
    --user-data-dir=/tmp/chrome-debug --no-first-run --no-sandbox --disable-gpu'
sleep 8 && curl -s http://127.0.0.1:9222/json/version
bash ~/系统工具/更新Chrome-CDP-ID.sh 2>/dev/null  # Chrome每次重启ID都变
```

### 3. Cookie检查
```bash
# 三个存储位置都要查（优先级从高到低）
wc -c ~/桌面/凭证/{bilibili,douyin,zhihu}_com_cookie_最新.txt
wc -c /tmp/{bilibili,douyin,zhihu}_cookies.txt 2>/dev/null
wc -c ~/.hermes/cookies/{bilibili,douyin,zhihu}_netscape.txt 2>/dev/null
# >1000B=✅ | <500B=❌需刷新 | 0B=❌需刷新
```

**刷新流程（有❌时）：**
```bash
# 第1步：cdp-get-cookies.py刷新（Chrome调试模式必须已在9222端口运行）
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain douyin.com --output ~/桌面/凭证/douyin_com_cookie_最新.txt
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain bilibili.com --output ~/桌面/凭证/bilibili_com_cookie_最新.txt
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain zhihu.com --output ~/桌面/凭证/zhihu_com_cookie_最新.txt

# 第2步：同步到/tmp（脚本读取位置）
cp ~/桌面/凭证/douyin_com_cookie_最新.txt /tmp/douyin_cookies.txt
cp ~/桌面/凭证/bilibili_com_cookie_最新.txt /tmp/bilibili_cookies.txt
cp ~/桌面/凭证/zhihu_com_cookie_最新.txt /tmp/zhihu_cookies.txt

# 第3步：持久化到~/.hermes/cookies/（Netscape格式，供CDP Chrome加载）
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain douyin.com --output ~/.hermes/cookies/douyin_netscape.txt --format netscape
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain bilibili.com --output ~/.hermes/cookies/bilibili_netscape.txt --format netscape
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain zhihu.com --output ~/.hermes/cookies/zhihu_netscape.txt --format netscape

# 第4步：验证刷新结果
wc -c /tmp/douyin_cookies.txt /tmp/bilibili_cookies.txt /tmp/zhihu_cookies.txt
# 如果还是<500字节 → Cookie刷新失败
```

**🚨 什么时候需要喊用户：**
- Chrome调试模式启动失败（systemd-run报错）→ "Chrome调试模式启动失败，需要你双击桌面→系统工具→启动Chrome调试模式.sh"
- cdp-get-cookies.py刷新后Cookie还是<500字节 → "Cookie刷新失败，需要你打开Chrome登录抖音/B站/知乎，然后我再刷新"
- 用户Chrome根本没开 → "需要你开Chrome才能提取Cookie"

**标准话术：** 检查完直接说"抖音✅(6.5K) B站❌(932B) 知乎✅(2.6K)"，不要绕弯子。

### 4. 收网检查（第〇步-乙）
```
读 ~/.hermes/收网提醒.txt
├─ 存在且有内容 → 问用户：要不要执行收网？
└─ 不存在或空 → 正常，跳过
然后读 events/宏观事件跟踪.md，如果没有 → 跳过
```

### 5. 问用户五个开关
```
抖音:✅/❌(XB) B站:✅/❌(XB) 知乎:✅/❌(XB)
1.调试输出？2.宏观审核？3.待重仓？4.三关？
("直接跑"=全不要 "全开"=全要)
```

**🚨 铁律：** 这五个检查必须在第〇步全部做完，得到用户回答后，才能进入阶段一。

### 6. 写状态文件
```python
import json
json.dump({
  "当前Step": "环境准备完成",
  "Cookie": {"抖音": "✅/❌", "B站": "✅/❌", "知乎": "✅/❌"},
  "开关": {"调试": "要/不要", "宏观": "要/不要", "待重仓": "要/不要", "三关": "要/不要"}
}, open("/tmp/monitor_state.json","w"), ensure_ascii=False, indent=2)
```

**返回值：** `{Cookie:{抖音:✅/❌,B站:✅/❌,知乎:✅/❌}, 开关:{调试,宏观,待重仓,三关}}`

**🚨 铁律：** Cookie全❌→终止。部分❌→刷新后再检查。五个开关必须全部确认才能进阶段一。
