# 环境准备

> **输入：** 无
> **输出：** 写`/tmp/monitor_state.json` → 返回{Cookie状态, 开关配置}

## 执行步骤

### 1. 内存检查
```bash
free -h | grep Mem  # available>3GB→✅ | <3GB→bash ~/桌面/脚本/内存急救.sh
```

### 2. CDP检查+启动
```bash
curl -s http://127.0.0.1:9222/json/version  # 有返回→✅
# 无返回→用systemd-run启动（这台Wayland机器systemd服务文件不可用！）
rm -rf /tmp/chrome-debug && mkdir -p /tmp/chrome-debug
systemd-run --user --pty --service-type=exec \
  --property=Environment="DISPLAY=:0" \
  bash -c 'google-chrome --remote-debugging-port=9222 --remote-allow-origins=* \
    --user-data-dir=/tmp/chrome-debug --no-first-run --no-sandbox --disable-gpu'
sleep 8 && curl -s http://127.0.0.1:9222/json/version
bash ~/系统工具/更新Chrome-CDP-ID.sh 2>/dev/null  # Chrome每次重启ID都变
```

### 3. Cookie检查
```bash
# 优先级：先看凭证目录原始文件，再看/tmp脚本读取文件，最后看Netscape持久化
wc -c ~/桌面/凭证/{bilibili,douyin,zhihu}_com_cookie_最新.txt
wc -c /tmp/{bilibili,douyin,zhihu}_cookies.txt 2>/dev/null
wc -c ~/.hermes/cookies/{bilibili,douyin,zhihu}_netscape.txt 2>/dev/null
# >1000B=✅ | <500B=❌需刷新 | 0B=❌需刷新
```

**刷新流程（有❌时）：**
```bash
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain {douyin|bilibili|zhihu}.com --output ~/桌面/凭证/{平台}_com_cookie_最新.txt
cp ~/桌面/凭证/{平台}_com_cookie_最新.txt /tmp/{平台}_cookies.txt
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain {平台}.com --output ~/.hermes/cookies/{平台}_netscape.txt --format netscape
```

### 4. 收网检查（第〇步-乙）
```
读 ~/.hermes/收网提醒.txt
├─ 存在且有内容 → 问用户：要不要执行收网？
└─ 不存在或空 → 正常，跳过
然后读 events/宏观事件跟踪.md，如果没有 → 跳过
```

### 5. 问用户五个开关
```
抖音:✅/❌(XB) B站:✅/❌(XB) 知乎:✅/❌(XB)
1.调试输出？2.宏观审核？3.待重仓？4.三关？
("直接跑"=全不要 "全开"=全要)
```

### 5. 写状态文件
```python
import json
json.dump({
  "当前Step": "环境准备完成",
  "Cookie": {"抖音": "✅/❌", "B站": "✅/❌", "知乎": "✅/❌"},
  "开关": {"调试": "要/不要", "宏观": "要/不要", "待重仓": "要/不要", "三关": "要/不要"}
}, open("/tmp/monitor_state.json","w"), ensure_ascii=False, indent=2)
```

**返回值：** `{Cookie:{抖音:✅/❌,B站:✅/❌,知乎:✅/❌}, 开关:{调试,宏观,待重仓,三关}}`

**🚨 铁律：** Cookie全❌→终止。部分❌→刷新后再检查。五个开关必须全部确认才能进阶段一。
