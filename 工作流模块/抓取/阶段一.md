# 阶段一抓取

> **输入：** 读`/tmp/monitor_state.json`（Cookie状态+开关配置）
> **输出：** 写`/tmp/monitor_state.json` + `/tmp/monitor_status.md` → 返回{统计, 待转录, 待分析, 失败项}

## 执行步骤

### 1. 初始化暂存目录
```bash
mkdir -p ~/.hermes/work/{videos,texts,transcripts}
echo "# 抓取状态清单 $(date +%Y-%m-%d_%H:%M)" > /tmp/monitor_status.md
echo "| 平台 | 博主 | 类型 | 结果 | 新内容数 | 暂存文件 | 备注 |" >> /tmp/monitor_status.md
echo "|:----|:----|:----|:----|:--------:|:--------|:----|" >> /tmp/monitor_status.md
```
**⚠️ /tmp重启就清！视频/文本用持久化目录 ~/.hermes/work/**

### 2. 读断点
打开`/home/fw/监控处理断点.md`，拿每个博主最后视频ID+时间。

**🚨 调试模式输出铁律（开关=要时）：**
```
每个博主必须输出（缺一不可）：
1.断点ID+时间+标题  2.API返回状态  3.视频列表前5条(ID+时间+标题)
4.断点ID在列表中位置(找到/没找到)  5.新视频判定过程
6.最终结论:0新/X新(列新视频ID+时间+标题)
API失败额外:每次请求原始错误+Cookie检查结果(wc -c)+最终判定
❌ 禁止:直接说"API失败"跳过/给结论不给原始数据/3人合并输出
```

**断点损坏/丢失时重建：**
```
✅ 正确数据源：~/.hermes/{douyin_state|zhihu_states|bilibili_states}/*.json
❌ 错误数据源：events/*.md（只有日期无ID）thinkers/*.md（博主档案非断点）
```
**为什么不能从events重建？** events记的是"分析结果"，格式不统一，大部分只记日期无视频ID和精确时间。state缓存是脚本每次运行后自动保存的，格式固定：`last_processed_id`+`last_processed_ts`(Unix秒)+`last_processed_title`。断点=state缓存的给人看的版本。

**断点文件正确格式：** `平台|博主|最后处理视频ID|发布时间(精确到时分)|最后视频标题`。不准只有日期/状态/备注！

### 3. 三平台交替抓取
```
正确：抖音A→B站A→知乎A→抖音B→B站B→知乎B→...
错误：抖音12人全跑完→B站3人→知乎4人（一口气30+请求必被限）
```

**对每个博主：** 调用对应平台模块 → 返回新内容 → **立刻写/tmp/monitor_status.md一行**

**🚨 铁律：每处理完一个博主必须立刻写状态清单！中断后读文件就知道哪到哪了。**

### 4. 请求频率控制
```
有新视频要转录 → 转录时间(1-2分钟)天然限速，不用sleep
0新不用转录    → 必须sleep(12秒)
知乎纯文字      → sleep(5秒)
```

### 5. 失败自动重试（内嵌，不调外部）
```
第1次→空/403/JSON失败 → sleep(10s) → 第2次→成功✅/失败→sleep(15s)→第3次→成功✅/失败→标记"失败-待重试"
```
**知乎特别注意：** 返回空**不是cookie过期**，第一反应是sleep→重试！

### 6. 汇总输出
```
✅ 成功抓取：X条  ❌ 失败待重试：Y条  ⏭️ 0新跳过：Z条
```

### 7. 写状态文件
```python
json.dump({
  "当前Step": "阶段一完成",
  "统计": {"成功":X,"失败":Y,"0新":Z},
  "失败项": ["平台-博主"],
  "待转录": ["~/.hermes/work/videos/xxx.mp4"],
  "待分析": ["~/.hermes/work/texts/xxx.json"]
}, open("/tmp/monitor_state.json","w"), ensure_ascii=False, indent=2)
```

**返回值：** `{统计:{成功X,失败Y,0新Z}, 失败项:[], 待转录:[], 待分析:[]}`

**标题筛选：** 持仓沾边/宏观中观沾边→进管道 | 鸡汤/社会/个人生活→跳过

## 🚨 抓取返回值铁律（2026-05-31 确立）

**每个博主抓取后必须返回最后一条视频的发布时间+发布日期+标题，拿去跟断点文档核对，才能确定有没有新发视频。**

```
抓到视频列表 → 返回最新视频(ID+时间+标题) → 跟断点对比 → 0新/X新 ✅
抓到空列表   → 说明脚本有问题或流程出问题了 ❌（不是"0新"！）
API返回-799  → sleep(15s)重试→还限流→浏览器兜底→还失败→标记"抓取失败" ❌
```

**禁止行为：**
- ❌ API返回-799/空，直接说"0新"（根本不知道有没有新视频）
- ❌ 啥都没抓到就标记"无更新"（没有证据）
- ❌ 不返回最新视频信息就更新断点（断点会漂移）

**正确行为：**
- ✅ 抓到列表→返回最新视频信息→跟断点对比→确定0新或X新
- ✅ -799→sleep(15s)重试→还限流→浏览器兜底→拿到至少最新视频信息
- ✅ 浏览器也失败→标记"抓取失败-无法确认是否有新内容"
