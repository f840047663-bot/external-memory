# 抖音抓取

## 输入
| 参数 | 类型 | 说明 |
|:----|:----|:----|
| sec_uid | 字符串 | 博主sec_uid（从thinkers/INDEX.md第三列读） |
| 断点ID | 字符串 | 最后处理视频ID |
| 断点时间 | 时间戳 | 最后处理视频时间 |

## 输出（返回值）
```
{
  "平台": "抖音",
  "博主": "xxx",
  "状态": "✅/❌",
  "新视频": ["aweme_id列表"],
  "失败原因": "xxx"
}
```

## 执行步骤

### Step 1：拉视频列表
```bash
COOKIE=$(cat /tmp/douyin_cookies.txt)
curl -s -H "Cookie: $COOKIE" \
  -H "User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/147.0.0.0 Safari/537.36" \
  -H "Referer: https://www.douyin.com/user/{sec_uid}" \
  "https://www.douyin.com/aweme/v1/web/aweme/post/?sec_user_id={sec_uid}&count=30&aid=1128"
```

**⚠️ &aid=1128参数必须带！** 抖音API必需参数。

### Step 2：ID锚点比对
```
第一级：搜断点ID在列表中的位置
  ├─ 找到 → 看它前面的条目，ID>断点ID 且 时间≥断点时间 → 算新
  └─ 没找到 → 第二级：时间降级（create_time > 断点时间 → 算新）
```

### Step 3：下载新视频
```bash
curl -L -o ~/.hermes/work/videos/{博主}_{视频ID}.mp4 \
  -H "Referer: https://www.douyin.com/" "{play_url}"
```

**⚠️ 下载视频不要传cookie，只传Referer！**

### Step 4：写状态清单
```bash
# 有新视频
echo "| 抖音 | {博主} | 视频 | ✅ | {N} | videos/{博主}_{ID}.mp4 | 待转录 |" >> /tmp/monitor_status.md
# 0新
echo "| 抖音 | {博主} | 视频 | ✅ | 0 | - | 0新 |" >> /tmp/monitor_status.md
# 失败
echo "| 抖音 | {博主} | 视频 | ❌ | ? | - | 失败-待重试 |" >> /tmp/monitor_status.md
```

## 失败处理
| 故障 | 解决 |
|:----|:-----|
| status_code=5 | 先查&aid=1128参数，再查cookie |
| Cookie过期 | 返回❌，框架调用[通用/环境准备]刷新 |
| API限流 | sleep(10s)→sleep(15s)重试，还失败→标记待重试 |
