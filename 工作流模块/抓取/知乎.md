# 知乎抓取

> **输入：** url_token（从 `thinkers/INDEX.md` 第4列读，格式 `people/xxx`）
> **输出：** 新文章/回答→保存到 `~/.hermes/work/texts/` → 写 `/tmp/monitor_status.md` 一行

## 主路：zhihu-scraper（wrapper）
```bash
python3 ~/.hermes/scripts/zhihu-scraper-wrapper.py <知乎用户名>
# 输出：发现N条新内容，保存到~/.hermes/work/zhihu_scraper_output/
```
**原理：** Wrapper自动从 `~/桌面/凭证/zhihu_com_cookie_最新.txt` 读cookie，注入scraper，自动比对断点。
**⚠️ 依赖 `/tmp/zhihu-venv`，重启就清！重启后需重装。**

## 副路：curl（知乎API）
**🚨 每个用户拆两行（文章+回答）各自独立追踪！**

```bash
COOKIE=$(cat ~/桌面/凭证/zhihu_com_cookie_最新.txt)
# 文章
curl -s -b "$COOKIE" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  -H "Referer: https://www.zhihu.com/" \
  "https://www.zhihu.com/api/v4/members/{url_token}/articles?limit=20"
# 回答
curl -s -b "$COOKIE" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  -H "Referer: https://www.zhihu.com/" \
  "https://www.zhihu.com/api/v4/members/{url_token}/answers?limit=20&order_by=created"
```
**⚠️ {url_token}就是纯用户名（如gududatong、lrc-8），不加people/前缀！**

## 获取详情并保存
```bash
# 文章详情
curl -s -b "$COOKIE" -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  "https://www.zhihu.com/api/v4/articles/{article_id}" > ~/.hermes/work/texts/{用户}_{article_id}.json
# 回答详情
curl -s -b "$COOKIE" -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  "https://www.zhihu.com/api/v4/answers/{answer_id}" > ~/.hermes/work/texts/{用户}_{answer_id}.json
```

## ID锚点比对
```
第一级：搜断点ID在列表中 → 找到→看前面条目，ID>断点且时间≥断点→算新
第二级：没找到→时间降级（created>断点时间→算新）
```

## 正文为空→降级链
```
content为空 → 是付费文？→是→标记"付费文-跳过"
              →否→调`工作流模块/浏览器/兜底.md`读网页正文→成功✅/验证码→标记"待用户处理"
```
**🚨 正文为空≠放弃！必须走浏览器兜底。**

## 🚨 知乎特有
- API间歇性返回空（尤其连续请求时）→**不是cookie过期**，第一反应是sleep→重试
- 文章vs回答必须分开追，每个用户拆两行断点
