# 知乎抓取

> **输入：** url_token, 断点ID, 断点时间
> **输出：** 新文章/回答→保存到~/.hermes/work/texts/ → 写/tmp/monitor_status.md一行

## API请求（副路：curl）
**🚨 知乎铁律：每个用户拆两行（文章+回答）各自独立追踪！只追踪一个类型会漏内容。**

> **主路待装**：`zhihu-scraper`（GitHub开源项目）因服务器连不上GitHub暂未安装。后续GitHub通了再装，装好就耦合进主路。
```bash
COOKIE=$(cat ~/桌面/凭证/zhihu_com_cookie_最新.txt)
# 文章
curl -s -b "$COOKIE" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  -H "Referer: https://www.zhihu.com/" \
  "https://www.zhihu.com/api/v4/members/{url_token}/articles?limit=20"
# 回答
curl -s -b "$COOKIE" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  -H "Referer: https://www.zhihu.com/" \
  "https://www.zhihu.com/api/v4/members/{url_token}/answers?limit=20&order_by=created"
```
url_token从`thinkers/INDEX.md`第五列读（格式`people/xxx`）。
**⚠️ {url_token}就是纯用户名（如gududatong、lrc-8），不加people/前缀！people/xxx是网页版URL格式，API不能用。**

## ID锚点比对
```
第一级：搜断点ID在列表中 → 找到→看前面条目，ID>断点且时间≥断点→算新
第二级：没找到→时间降级（created>断点时间→算新）
```

## 获取详情并保存
```bash
# 文章详情
curl -s -b "$COOKIE" -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  "https://www.zhihu.com/api/v4/articles/{article_id}" > ~/.hermes/work/texts/{用户}_{article_id}.json
# 回答详情
curl -s -b "$COOKIE" -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  "https://www.zhihu.com/api/v4/answers/{answer_id}" > ~/.hermes/work/texts/{用户}_{answer_id}.json
```

## 正文为空→降级链
```
content为空 → 是付费文？→是→标记"付费文-跳过"
              →否→[浏览器/兜底]读网页正文→成功✅/验证码→标记"待用户处理"
```
**🚨 铁律：正文为空≠放弃！必须走浏览器兜底，不准直接标记"正文折叠不可读"跳过。**

## 写状态清单
```
有新内容: "| 知乎 | {用户} | 文章/回答 | ✅ | {N} | texts/{用户}_{ID}.json | 待分析 |"
0新:      "| 知乎 | {用户} | 文章/回答 | ✅ | 0 | - | 0新 |"
失败:     "| 知乎 | {用户} | 文章/回答 | ❌ | ? | - | 失败-待重试(间歇性空) |"
```

## 失败处理
| 故障 | 解决 |
|:----|:-----|
| API返回空 | **不是cookie过期！** sleep(10s)→sleep(15s)重试 |
| 正文content为空 | [浏览器/兜底]读网页 |
| 浏览器也拿不到 | 标记"待用户处理-需手动过验证码" |
| Cookie过期 | 返回❌，框架调[环境准备]刷新 |

**🚨 知乎返回空的第一反应：sleep→重试，不是刷新cookie！**
