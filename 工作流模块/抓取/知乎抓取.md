# 知乎抓取

## 输入
| 参数 | 类型 | 说明 |
|:----|:----|:----|
| url_token | 字符串 | 用户url_token（从thinkers/INDEX.md第五列读） |
| 断点ID | 字符串 | 最后处理文章/回答ID |
| 断点时间 | 时间戳 | 最后处理时间 |

## 输出（返回值）
```
{
  "平台": "知乎",
  "用户": "xxx",
  "状态": "✅/❌",
  "新内容": [{"类型": "文章/回答", "ID": "xxx", "路径": "xxx"}],
  "失败原因": "xxx"
}
```

## 执行步骤

### Step 1：拉文章列表
```bash
COOKIE=$(cat /tmp/zhihu_cookies.txt)
curl -s -b "$COOKIE" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  -H "Referer: https://www.zhihu.com/" \
  "https://www.zhihu.com/api/v4/members/{url_token}/articles?limit=20"
```

### Step 2：拉回答列表
```bash
curl -s -b "$COOKIE" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  -H "Referer: https://www.zhihu.com/" \
  "https://www.zhihu.com/api/v4/members/{url_token}/answers?limit=20&order_by=created"
```

**⚠️ 文章+回答都要查！** 只查文章会漏大量信息。

### Step 3：ID锚点比对
```
第一级：搜断点ID在列表中的位置
  ├─ 找到 → 看它前面的条目，ID>断点ID 且 时间≥断点时间 → 算新
  └─ 没找到 → 第二级：时间降级（created > 断点时间 → 算新）
```

### Step 4：获取详情并保存
```bash
# 文章详情
curl -s -b "$COOKIE" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  "https://www.zhihu.com/api/v4/articles/{article_id}" \
  > ~/.hermes/work/texts/{用户}_{article_id}.json

# 回答详情
curl -s -b "$COOKIE" \
  -H "User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 16_0 like Mac OS X)" \
  "https://www.zhihu.com/api/v4/answers/{answer_id}" \
  > ~/.hermes/work/texts/{用户}_{answer_id}.json
```

### Step 5：正文为空→浏览器CDP兜底
```
API拿到列表 → 逐条获取详情 → content为空
  ↓
判断：是付费文？ → 是 → 标记"付费文-跳过"
  ↓ 否
浏览器CDP兜底
  ↓ 成功
✅ 正常进入分析管道
  ↓ 浏览器也拿不到（验证码拦截）
❌ 标记"待用户处理-需手动过验证码"
```

**🚨 铁律：正文为空≠放弃！必须走浏览器兜底，不准直接标记"正文折叠不可读"然后跳过。**

**浏览器CDP兜底方案：**
```bash
# 用cdp-browser-reader.py直连
python3 ~/.hermes/scripts/cdp-browser-reader.py "https://www.zhihu.com/answer/{answer_id}"
# 返回JSON: {"title": "...", "text": "...", "url": "..."}
```

**浏览器兜底核心原则：**
1. browser_navigate报404 → 用cdp-browser-reader.py直连
2. 每打开一个页面读完就关（Target.closeTarget），不堆积标签页
3. API被拦截→浏览器兜底→浏览器还被拦截→**喊用户过验证码**

### Step 6：写状态清单
```bash
echo "| 知乎 | {用户} | 文章/回答 | ✅/❌ | {N} | texts/{用户}_{ID}.json | 待分析/失败-待重试 |" >> /tmp/monitor_status.md
```

## 失败处理
| 故障 | 解决 |
|:----|:-----|
| API返回空 | **不是cookie过期！** sleep(10s)→sleep(15s)重试 |
| 正文content为空 | 浏览器CDP兜底 |
| 浏览器也拿不到 | 标记"待用户处理-需手动过验证码" |
| Cookie过期 | 返回❌，框架调用[通用/环境准备]刷新 |

**🚨 知乎返回空的第一反应：sleep→重试，不是刷新cookie！**
