# 抖音抓取

> **输入：** sec_uid, 断点ID, 断点时间
> **输出：** 新视频列表 → 保存到~/.hermes/work/videos/ → 写/tmp/monitor_status.md一行

## API请求
```bash
COOKIE=$(cat ~/桌面/凭证/douyin_com_cookie_最新.txt)
curl -s -H "Cookie: $COOKIE" \
  -H "User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/147.0.0.0 Safari/537.36" \
  -H "Referer: https://www.douyin.com/user/{sec_uid}" \
  "https://www.douyin.com/aweme/v1/web/aweme/post/?sec_user_id={sec_uid}&count=30&aid=1128"
```
**⚠️ &aid=1128 必须带！** sec_uid从`thinkers/INDEX.md`第三列读。

**🚨 抖音API铁律（违反会误判）：**
- **count必须≥30**，小count值对部分账号（但斌）返回0条
- **UA必须用完整Chrome版本号**，不能用`Mozilla/5.0`太笼统→空响应
- **Referer必须带/user/{sec_uid}**，光`https://www.douyin.com/`不行
- **API排序不是时间倒序**→必须先搜断点ID在列表中的位置，再比ID大小
- **ID不严格单调递增**→必须ID+时间双验证（ID大且时间新才算新）

## ID锚点比对
```
第一级：搜断点ID在列表中的位置 → 找到→看前面条目，ID>断点ID且时间≥断点时间→算新
第二级：没找到→时间降级（create_time>断点时间→算新）
```

## 下载新视频
```bash
curl -L -o ~/.hermes/work/videos/{博主}_{aweme_id}.mp4 \
  -H "Referer: https://www.douyin.com/" "{play_url}"
```
**⚠️ 下载只传Referer，不要传cookie！**

## 写状态清单
```
有新视频: "| 抖音 | {博主} | 视频 | ✅ | {N} | videos/{博主}_{ID}.mp4 | 待转录 |"
0新:      "| 抖音 | {博主} | 视频 | ✅ | 0 | - | 0新 |"
失败:     "| 抖音 | {博主} | 视频 | ❌ | ? | - | 失败-待重试 |"
```
追加到 `/tmp/monitor_status.md`

## 失败处理
| 故障 | 解决 |
|:----|:-----|
| status_code=5 | 先查&aid=1128参数，再查cookie |
| API空/JSON失败 | sleep(10s)→sleep(15s)重试，还失败→标记待重试 |
| Cookie过期 | 返回❌，框架调[环境准备]刷新 |
