# 抖音抓取

> **输入：** sec_uid（从 `thinkers/INDEX.md` 第2列读）
> **输出：** 新视频→保存到 `~/.hermes/work/videos/` → 写 `/tmp/monitor_status.md` 一行

## API请求（拉列表）
```bash
COOKIE=$(cat ~/桌面/凭证/douyin_com_cookie_最新.txt)
curl -s -H "Cookie: $COOKIE" \
  -H "User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/147.0.0.0 Safari/537.36" \
  -H "Referer: https://www.douyin.com/user/{sec_uid}" \
  "https://www.douyin.com/aweme/v1/web/aweme/post/?sec_user_id={sec_uid}&count=30&aid=1128"
```
**⚠️ &aid=1128 必须带！**

## 获取play_url（detail API）
```bash
curl -s -H "Cookie: $COOKIE" \
  -H "User-Agent: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/147.0.0.0 Safari/537.36" \
  -H "Referer: https://www.douyin.com/" \
  "https://www.douyin.com/aweme/v1/aweme/detail/?aweme_id={视频ID}&aid=1128"
# 从返回 JSON 的 aweme_detail.video.play_addr.url_list[0] 拿到 play_url
```

## 下载新视频
```bash
curl -L -o ~/.hermes/work/videos/{博主}_{aweme_id}.mp4 \
  -H "Referer: https://www.douyin.com/" "{play_url}"
```
**⚠️ 下载只传Referer，不要传cookie！**

## 🚨 抖音API铁律（违反会误判）
1. **count必须≥30**，小count值对部分账号（但斌）返回0条
2. **UA必须用完整Chrome版本号**，不能用太笼统的UA→空响应
3. **Referer必须带/user/{sec_uid}**，光`https://www.douyin.com/`不行
4. **API排序不是时间倒序**→必须先搜断点ID在列表中的位置，再比ID大小
5. **ID不严格单调递增**→必须ID+时间双验证（ID大且时间新才算新）

## ID锚点比对
```
第一级：搜断点ID在列表中 → 找到→看前面条目，ID>断点且时间≥断点→算新
第二级：没找到→时间降级（create_time>断点时间→算新）
```
