# 阶段四收尾

> **输入：** 读`/tmp/monitor_state.json`（全部阶段结果）
> **输出：** 写`/tmp/monitor_state.json` → 返回{断点更新, 日报路径, 邮件发送, 自检结果}

## 执行步骤

### 1. 更新断点
**🚨 断点更新="信息已完整走完管道"的确认标记。提前更新=信息永久丢失。**

```
前置条件（全部满足才能更新）：
  ✅ events/当天.md已归档  ✅ thinkers/{博主}.md已更新
  ✅ positions/{资产}.md已更新（含P值链校验）  ✅ 23-贝叶斯日志已更新
  ✅ 异动审核（如触发）已完成
```
**中间任何一步中断→断点不更新→下次重新捕获（浪费token但信息不丢）**

**⛔ 强制校验：**
```
出日报前必须：①打开断点.md ②逐个核对今天处理的博主ID是否最新 ③有没更新的→立即补 ④核对完→才能日报
```

### 2. 写 Excel 看板（一次性批量写入）
**🚨 看板是最终输出，必须一次性写完！** 从 state.json 读"看板待写入"列表，批量写入 `~/桌面/持仓贝叶斯跟踪.xlsx`。
```
读 state.json → 取"看板待写入"列表 → 一次性 append 到 Excel 总看板
```
**禁止：** 阶段三逐条写。中断恢复后重新攒数据，只在阶段四写。

### 3. 出日报
**格式：** 宏观→中观→微观自上而下。每项：事件链→P值→3选1动作（不动/加/减）。
末尾查长期主题表：≥7天的搜新进展，<7天的跳过。

### 3. 强制汇报
**🚨 任何任务完成/中断，必须主动汇报。不能无声无息结束。**
```
✅ 完成：抖音X人→Y条新视频→Z条有观点→P值调整N项
❌ 阻塞：B站（原因）、知乎（原因）
📋 待办：下次继续做什么
```
**中断也要汇报：** 最后一条消息="当前进度+下次继续做什么"

### 4. 🚨 强制自检清单（10项，缺任何一条不准发消息）
```
□ ①主页ID从INDEX.md读的  ②长期主题≥7天搜了  ③P值链可追溯
□ ④P'虚值标了  ⑤events写了  ⑥hindsight_retain存了
□ ⑦状态清单写了  ⑧失败项重试了  ⑨内容存暂存目录  ⑩三平台交替了
```

### 5. 写状态文件
```python
json.dump({
  "当前Step": "全流程完成",
  "断点已更新": true/false,
  "日报路径": "xxx",
  "邮件发送": "✅/❌",
  "自检": "10/10"
}, open("/tmp/monitor_state.json","w"), ensure_ascii=False, indent=2)
```

**返回值：** `{断点:✅/❌, 日报:路径, 邮件:✅/❌, 自检:X/10}`
