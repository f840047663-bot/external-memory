# 阶段三分析

> **输入：** 读`/tmp/monitor_state.json`（待分析列表+开关）
> **输出：** 写`/tmp/monitor_state.json` → 返回{影响持仓, 异动审核, 事件归档}

## 执行步骤

### 1. 提炼核心观点
从`~/.hermes/work/transcripts/`和`texts/`读内容。**砍废话，只留：关键数据、逻辑链、时间节点、持仓关联度。**
**🚨 不能凭标题推测，必须读完转录文字再分析。**

### 2. 贝叶斯更新（五道门，缺一不调）
```
A(事件/系统/噪声) → B(宏观折价) → C(机械去偏) → D(定性，读position文件) → E(事实/观点分离+共识衰减+查重)
```
**0pp也必须写理由。**

**各门规则：**
- **A门**：判断是事件/系统性变化/噪声。噪声→±0pp
- **B门**：利率折价。先判断利率为什么高（通胀/地缘→可能利好黄金），再看敏感度。规则见`00-宏观传导框架.md §利率`
- **C门**：逻辑簇去重+机械去偏。规则见`21-贝叶斯防偏差规则.md §C门`和`positions/贝叶斯计算标准.md`
- **D门**：读position文件定性。资产本质变了才大调
- **E门**：事实进主P，观点进P'。多源同逻辑只加1~2pp（共识衰减）

**⛔ P'强制规则：**
```
未落地事件/预测→不准进主P值，只写虚值P'
P'转实条件：①事件落地 ②价格先行确认(连续3天同向+动因校验) ③超窗口→作废
P'转实=必须推用户确认，不允许自动转实
```

**⛔ P值链强制校验：**
```
格式：文件头P = 链累加结果 = 账户确认值
例：养殖ETF 51% = 50-2-2+4+1 = 51% ✅
```

### 3. 归档
```
events/当天.md → thinkers/{博主}.md → positions/{资产}.md → 23-贝叶斯日志
```
宏观事件跟踪表也要检查要不要更新。

### 4. ⛔ 异动自动审核（if块，归档完必须判断！）
```
IF P值变动≥±3pp OR 重大事件 OR 左侧资产:
  → 问用户：「检测到异动，影响资产：{列表}。要不要跑完整审核？」
  → 用户说"跑" → 宏观前置(00-宏观传导框架.md) + 三关(巴菲特审核框架.md)
  → 审核结果写入position文件底部「自动审核记录」区
```
**🚨 ⑨不是参考章节，是流程里的if块。归档完必须判断，不能直接跳到⑩。**

### 5. 攒看板数据（不写，等阶段四统一写）
**🚨 看板是最终输出，阶段三不准写！** 中途失败/中断会导致看板数据被污染（部分数据进了但管道没走完）。
```
每条走完管道 → 把看板条目追加到 state.json 的 "看板待写入" 列表
阶段四收尾时 → 一次性批量写入 Excel 看板
```

### 6. 写状态文件
```python
json.dump({
  "当前Step": "阶段三完成",
  "影响持仓": [{"资产":"xxx","P值":"51%→54%","原因":"xxx"}],
  "异动审核": "已执行/未触发/用户跳过",
  "事件归档": "events/2026-05-31.md",
  "看板待写入": [{"时间":"xxx","博主":"xxx","标题":"xxx","结论":"xxx","影响":"xxx"}, ...]
}, open("/tmp/monitor_state.json","w"), ensure_ascii=False, indent=2)
```

**返回值：** `{影响持仓:[...], 异动审核:已执行/未触发/跳过, 事件归档:路径, 看板待写入:N条}`
