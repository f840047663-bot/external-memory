# 阶段二转录+重试

> **输入：** 读`/tmp/monitor_state.json`（待转录列表+失败项）
> **输出：** 写`/tmp/monitor_state.json` → 返回{转录结果, 重试结果, 待分析列表}

## 执行步骤

### 1. 检查VideoCaptioner容器
```bash
docker ps --filter name=videocaptioner  # 未运行→docker start videocaptioner
```

### 2. 转录视频（逐个，转录=天然限速）
```bash
# 1.送容器
sg docker -c "docker cp ~/.hermes/work/videos/{博主}_{ID}.mp4 videocaptioner:/app/temp/v.mp4"
# 2.提mp3（必须！transcribe_new.py只接受音频）
sg docker -c "docker exec videocaptioner ffmpeg -i /app/temp/v.mp4 -vn -acodec mp3 -y /app/temp/v.mp3"
# 3.转录——等完成！
sg docker -c "docker exec videocaptioner python3 /app/temp/transcribe_new.py /app/temp/v.mp3 /app/temp/result"
# 4.取回文字
sg docker -c "docker cp videocaptioner:/app/temp/result_全文.txt ~/.hermes/work/transcripts/{博主}_{ID}.txt"
# 5.清理
sg docker -c "docker exec videocaptioner rm -f /app/temp/v.mp4 /app/temp/v.mp3 /app/temp/result.srt /app/temp/result_全文.txt"
rm -f ~/.hermes/work/videos/{博主}_{ID}.mp4
# 6.写状态
echo "| 转录 | {博主} | 视频 | ✅ | - | transcripts/{博主}_{ID}.txt | 待分析 |" >> /tmp/monitor_status.md
```

**🚨 踩坑铁律：**
- `transcribe_new.py`只接受mp3/wav，**不接受mp4！** 必须先ffmpeg提音频
- ffmpeg在容器里，主机不需要装
- 卡住先读脚本源码：`docker exec videocaptioner head -30 /app/temp/transcribe_new.py`
- 抖音CDN音视频分流→只有h264无aac→需[浏览器/兜底]合并

### 3. 重试阶段一失败项
**🚨 铁律：阶段一标记"失败-待重试"的项，必须回头重试！不能放弃！**
```
读/tmp/monitor_status.md找❌项 → sleep(15s)重试→成功✅→更新状态
                                  →还失败→sleep(20s)再试→成功✅
                                                   →还失败→检查cookie→无效则刷新→有效则标记"最终失败"
```

### 4. 任务编排原则
```
能立即处理的→先处理  需要浏览器的→排"浏览器队列"最后集中处理
需要刷新cookie的→标记"待刷新"最后统一  需要用户的→标记"待用户"最后一次性汇报
```
**🚫 禁止：** 遇障碍就停等用户，搁置其他能做的任务。

### 5. 写状态文件
```python
json.dump({
  "当前Step": "阶段二完成",
  "转录结果": [{"视频ID":"xxx","状态":"✅/❌","路径":"xxx"}],
  "重试结果": [{"博主":"xxx","状态":"✅/❌","原因":"xxx"}],
  "待分析": ["~/.hermes/work/transcripts/xxx.txt","~/.hermes/work/texts/xxx.json"]
}, open("/tmp/monitor_state.json","w"), ensure_ascii=False, indent=2)
```

**返回值：** `{转录:{成功X,失败Y}, 重试:{成功M,最终失败N}, 待分析:[]}`
