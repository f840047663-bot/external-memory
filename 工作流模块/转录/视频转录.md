# 视频转录

## 输入
| 参数 | 类型 | 说明 |
|:----|:----|:----|
| 待转录列表 | 列表 | 来自抓取模块的新视频ID列表 |

## 输出（返回值）
| 字段 | 类型 | 说明 |
|:----|:----|:----|
| 视频ID | 字符串 | 原始视频ID |
| 状态 | ✅/❌ | 转录是否成功 |
| 文本路径 | 字符串 | 转录文本保存路径 |
| 失败原因 | 字符串 | 如果❌，返回原因 |

## 执行步骤

### Step 1：检查VideoCaptioner容器
```bash
docker ps --filter name=videocaptioner
```
- 如果未运行 → `docker start videocaptioner`

### Step 2：对每个视频执行转录
```
FOR 每个视频ID IN 待转录列表:
  1. 下载音频（yt-dlp）
  2. 调用VideoCaptioner ASR
  3. 保存文本到 events/{日期}_{视频ID}.md
```

### Step 3：汇总结果
- 成功列表 → 返回文本路径
- 失败列表 → 返回失败原因

## 返回值示例
```
[
  {
    "视频ID": "BV1xx4y11A23",
    "状态": "✅",
    "文本路径": "events/2026-05-31_BV1xx4y11A23.md"
  },
  {
    "视频ID": "video_456", 
    "状态": "❌",
    "失败原因": "下载失败"
  }
]
```

## 失败处理
| 故障 | 解决 |
|:----|:-----|
| 容器未运行 | 自动启动 |
| DNS错误 | sleep(5)重试1次 |
| 下载失败 | 返回❌，不影响其他视频 |
