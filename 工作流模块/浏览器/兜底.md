# 浏览器兜底

> **隔离目录！** 不稳定因素单独关起来。只有框架降级时才调用。
> **输入：** 失败项列表（来自状态文件）
> **输出：** 返回{成功项, 仍失败项}

## 标准操作流程（必须遵守）
```
1. Target.createTarget创建新标签页
2. Page.navigate导航到目标URL
3. 等页面加载（Page.loadEventFired）
4. 执行具体操作
5. Target.closeTarget关闭标签页 ← 🚨必须关！处理完一个立刻关一个！
```

**🚨 标签页铁律：**
- 有用的时候开着，没用的时候**立刻关**
- 三个平台同时处理时最多3个标签页
- 不用的标签页占用内存和CPU，必须回收
- 核心原则：用完就关，不要堆积

## 操作类型

### 知乎正文兜底
```bash
# API返回content为空→浏览器读网页
# v2自动注入~/桌面/凭证/zhihu_com_cookie_最新.txt，无需手动指定
python3 ~/.hermes/scripts/cdp-browser-reader.py "https://www.zhihu.com/answer/{answer_id}"
# 返回JSON: {"title":"...","text":"...","url":"..."}
```
**browser_navigate报404→用cdp-browser-reader.py直连，不要反复重试。**

### B站列表兜底（-799限流处理）
```
收到-799
  ↓
sleep(15秒) → 重试1次 → 成功 → ✅ 继续
  ↓ 还限流
跳过此平台，换下一个平台（三平台交替）
  ↓
阶段二回头重试（冷却时间已足够长）
```
**🚨 B站-799铁律：**
- 不同博主请求间必须sleep(5-8秒)
- 收到-799必须sleep(15秒)，不能只sleep(5秒)
- **不能连续重试同一平台！** 必须换平台
- 浏览器兜底对B站SPA页面效果差（动态加载），优先用skip→retry策略
- 如果阶段二仍然-799→标记"最终失败-待用户处理"

### 抖音视频下载兜底
```
API拿不到play_url→CDP Network拦截douyinvod URL→下载音视频流→ffmpeg合并
⚠️ 抖音CDN音视频分流：看到只有h264不要放弃！必须合并。

详细步骤：
1. CDP Network拦截所有douyinvod URL（通常2个），分别下载
2. 用ffprobe区分video/audio流：
   sg docker -c "docker cp /tmp/file.tmp videocaptioner:/app/work-dir/check.mp4"
   sg docker -c "docker exec videocaptioner ffprobe -v error -show_entries stream=codec_type -of csv=p=0 /app/work-dir/check.mp4"
   # 输出: video 或 audio
3. ffmpeg合并：
   sg docker -c "docker cp video_file videocaptioner:/app/work-dir/v.mp4"
   sg docker -c "docker cp audio_file videocaptioner:/app/work-dir/a.mp4"
   sg docker -c "docker exec videocaptioner ffmpeg -y -i /app/work-dir/v.mp4 -i /app/work-dir/a.mp4 -c copy /app/work-dir/merged.mp4"
   sg docker -c "docker cp videocaptioner:/app/work-dir/merged.mp4 /tmp/final.mp4"
```

### Cookie提取
```
Cookie过期→打开网站→等用户登录→Network.getAllCookies→保存Netscape格式
```

## cdp-browser-reader.py v2 说明
- 位置：`~/.hermes/scripts/cdp-browser-reader.py`
- **自动cookie注入**：根据URL自动匹配`~/桌面/凭证/{平台}_com_cookie_最新.txt`
- 也可手动指定：`--cookies ~/桌面/凭证/bilibili_com_cookie_最新.txt`
- SPA页面支持：轮询等待`.RichContent-inner`等选择器出现，最多等`--wait`秒
- `/tmp/chrome-debug`是独立profile，v2通过Network.setCookie注入登录态

## 降级链
```
API失败→浏览器兜底→浏览器也拿不到（验证码）→🚨喊用户手动过验证码
```
**🚨 浏览器兜底是最后一道防线，浏览器都拦不住→必须喊用户。**

## 返回值
```
{
  "成功项": [{"平台":"知乎","用户":"xxx","数据":"正文文本"}],
  "仍失败项": [{"平台":"知乎","用户":"xxx","原因":"验证码拦截"}]
}
```
