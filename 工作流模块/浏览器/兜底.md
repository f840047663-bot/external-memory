# 浏览器兜底

> **隔离目录！** 不稳定因素单独关起来。只有主路+副路都失败时才调用。

## 标准操作流程（必须遵守）
```
1. Target.createTarget创建新标签页
2. Page.navigate导航到目标URL
3. 等页面加载（Page.loadEventFired）
4. 执行具体操作
5. Target.closeTarget关闭标签页 ← 🚨必须关！处理完一个立刻关一个！
```

**🚨 标签页铁律：** 用完就关，不要堆积。三个平台同时处理时最多3个标签页。

## 抖音视频下载兜底（CDP Network拦截）
抖音CDN音视频分流，API拿不到play_url时用：
```
1. CDP Network拦截所有douyinvod URL（通常2个），分别下载
2. 用ffprobe区分video/audio流：
   sg docker -c "docker cp /tmp/file.tmp videocaptioner:/app/work-dir/check.mp4"
   sg docker -c "docker exec videocaptioner ffprobe -v error -show_entries stream=codec_type -of csv=p=0 /app/work-dir/check.mp4"
   # 输出: video 或 audio
3. ffmpeg合并：
   sg docker -c "docker cp video_file videocaptioner:/app/work-dir/v.mp4"
   sg docker -c "docker cp audio_file videocaptioner:/app/work-dir/a.mp4"
   sg docker -c "docker exec videocaptioner ffmpeg -y -i /app/work-dir/v.mp4 -i /app/work-dir/a.mp4 -c copy /app/work-dir/merged.mp4"
   sg docker -c "docker cp videocaptioner:/app/work-dir/merged.mp4 /tmp/final.mp4"
```
**⚠️ 看到只有h264不要放弃！抖音就是把音视频分开的，必须合并。**

## cdp-browser-reader.py v2 说明
- 位置：`~/.hermes/scripts/cdp-browser-reader.py`
- **自动cookie注入**：根据URL自动匹配 `~/桌面/凭证/{平台}_com_cookie_最新.txt`
- 也可手动指定：`--cookies ~/桌面/凭证/bilibili_com_cookie_最新.txt`
- SPA页面支持：轮询等待 `.RichContent-inner` 等选择器出现
- `/tmp/chrome-debug` 是独立profile，v2通过Network.setCookie注入登录态
