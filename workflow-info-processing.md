# 信息处理三步骤工作流（永久流程）

> 收到任何信息源（抖音转录/知乎文章/B站视频/新闻）后，必须按以下三步走。
> **铁律：不准跳步。不准光写events不归档。不准推半成品给小龙虾。**

---

## 第0步（新增铁律）：先定性，后定量

**每收到任何信息，必须先回答：这是什么类型的信息？**

**第一步（估值框架检查）：如果涉及对某个资产的"值不值/能不能买"的判断，必须先读 `00-估值方法论.md` 的丛林三问框架，再用下面类型分类。**

| 类型 | 定义 | 处理方式 |
|:----|:-----|:--------|
| 🔴 **基本面事件** | 影响资产本身价值 | 正常贝叶斯pp加减 |
| 🟡 **系统性因子** | 改变所有资产定价环境 | 统一折价系数，不分资产各自打分 |
| ⚪ **噪声** | 无影响 | 忽略 |

**跳步警告：如果跳过第0步直接开始算pp → 就是胡扯，不如不用贝叶斯。**

完整规则见 `21-贝叶斯防偏差规则.md` 第7章。

### 第0步B：查当前系统性折价系数

**更新任何持仓P值前，必须先读 `00-宏观传导框架.md` 的「当前系统性折价系数」字段。**

因为利率等系统性因子改变的是所有资产的定价环境，不是单个资产的事件。这个系数已在宏观框架里统一记录，持仓P值更新时自然地把它算进去。

```
当前折价系数 → 对应10年美债收益率位置
  < 4.0%  → 系数=0（正常环境）
  4.0-4.5% → 系数=-2pp（偏紧）
  4.5-5.0% → 系数=-4pp（收紧）
  > 5.0%  → 系数=-6pp（严重收紧）
```

**不查系数就更新P值 = 漏掉了系统性因子，算出来的数不对。**

### 第0步C：系统性调整必须显式写入事件链（禁止隐形）

利率、汇率、流动性等系统性因子的调整，不能静默修改P值。**每条调整必须显式记录在受影响持仓的事件链中。**

格式：
```
YYYY-MM-DD | ⚙️ 系统性调整：原因（如10Y美债~4.5%，折价系数-4pp） | ±Xpp | 新P值
```

规则：
- 用 ⚙️ 符号前缀，跟基本面事件🔴一目区分
- 写明调整原因 + 折价系数数值
- 折价系数变则增，不变不动
- **禁止不写事件链条目直接改文件头的P值**

## 第1步：判断信息属于哪层

| 层 | 判断标准 | 更新到哪个文件 |
|:--|:---------|:--------------|
| 🟤 **宏观层** | 利率/美债/地缘/全球/经济数据/货币政策/通胀 | `00-宏观传导框架.md` → 对应传导路径的「当前状态」 |
| 🟠 **中观层** | 行业/产业链/板块/供需/政策（如芯片、养殖、银行） | 更新对应板块分析，或补充到 `00-宏观传导框架.md` 的「影响持仓」表 |
| 🟢 **微观层** | 具体持仓操作/净值/买卖信号/贝叶斯更新 | `positions/{资产}.md` → 决策历史+当前P值 |

## 第2步：跨层传导

不能只写一层。每层之间必须传导：

```
宏观事件 → 查对应哪些中观板块受影响 → 查微观(具体持仓)受影响
                  ↓
           中观板块变化 → 再查微观(具体持仓)
```

**如何执行：**
- 宏观层更新后 → 在00-宏观传导框架.md的“影响持仓”表里更新每个持仓的方向
- 中观层更新后 → 更新对应positions/{资产}.md
- 每次更新必须标注：来源（哪个博主/哪个视频/哪篇文章）+ 链接

### 第2步B（强制校验）：事件链更新后必须执行五项检查

**任何一条事件链更新后，必须逐项检查以下5条，全部通过才算完成：**

```
□ ① 时间顺序正确
   新插入的事件在正确的时间位置，不是无脑追加在末尾
   检查方法：日期应该按从旧到新排列

□ ② P值链连续
   本条事件的「起始P值」= 上一条事件的「结束P值」
   检查方法：找到上一条事件（按时间排序后），看P值能否接上
   如果接不上 → 说明之前的链就有断，必须从头校准

□ ③ 无重复起点
   不同事件不能从同一个P值开始各自减pp
   检查方法：扫描整个链，所有「起始P值」应该是一对一的

□ ④ 方向合理
   利好事件 → P上调；利空事件 → P下调
   如果利好事件导致P下跌（或反之），必须解释为什么

□ ⑤ 赋值区间合理
   pp赋值参考标准：
   框架级利好/利空 → ±5~8pp（多源共识/逻辑坚实）
   事件级利好/利空 → ±2~4pp（单源/非共识）
   微调/确认 → ±1pp
   超出此范围必须有理由
```

**五项全过才算一次合格的贝叶斯更新。缺一项没查就直接推给用户 → 属于不负责任，用户原话：'跟胡扯没区别'。**

### 第2步C（定期校验）：P值与价格走势的互锁验证

**不是一个事件来就验证，而是定期回头检查已做出的判断。**

```
价格走势持续向上  +  P值也在高位（如65%+）
  → ✅ 判断与市场一致，P值被验证

价格走势持续向下  +  P值在高位（如65%+）
  → ⚠️ 可能漏了什么利空，或P值有偏
  → 去查：这段时间有没有没被记录的事件？
  → 如果找到新事件 → 补进事件链
  → 如果没找到 → P值维持不变（价格波动不参与推理）

价格走势持续向上  +  P值在低位（如40%）
  → ⚠️ 可能漏了什么利好，或P值偏保守
  → 同上，去查

价格持续震荡  +  P值在中位（45-55%）
  → ✅ 正常，方向未定
```

**关键原则：**
- 验证是对**判断逻辑**的检验，不是对**数字**的修正
- 发现不一致 → 去查事件链有没有遗漏，不是倒推P值
- 净值不参与推理（此铁律不变）

## 第3步：归档+更新+推送

每一层都更新完后，再做以下归档：

### 3.1 归档到thinker
```
thinkers/{博主}.md → 更新「最新观点」栏
```
格式：
```
### 日期 标题（平台 ※已转录）
- **来源：** 链接
- **核心观点：** 1-2句浓缩
- **对持仓的影响：** 持仓名 → 方向 → P值调整
- **🔗 完整分析：** events/{日期}-{博主}-{标题}.md
```

### 3.2 更新贝叶斯日志
```
23-贝叶斯更新日志.md → 追加一行
格式：| 资产 | P变化 | 证据 | 验证 |
```

### 3.3 推成品给小龙虾
全部处理完后，推一次成品数据：
- 已更新的P值
- 事件链摘要（宏观→中观→微观）
- 新增的事件文件路径
- **不准推半成品**（说需要验证的数据不算成品）

## 2026-05-22 重要架构变更：监控清单统一化

**以前的问题：** Thinker档案与监控清单各自独立 → 加人忘加监控 = 有档案但脚本不监控。

**新架构（已实施）：**
```
thinkers/INDEX.md（唯一主数据源）
  ├─ 抖音监控脚本 → 自动读第2列（MS4w开头=抖音博主）
  ├─ B站监控脚本  → 自动读第3列（mid:数字=B站UP主）
  └─ 知乎监控脚本 → 自动读第4列（people/xxx=知乎用户）
```
**结论：加新博主 → 只改INDEX.md一行 → 三个脚本全自动同步。**
独立txt文件保留为fallback，脚本优先读INDEX.md。

## 流程D（新增）：每日风险信号汇总 — 自动化监控

**每天10:00 cron自动执行，独立于手动流程。**

### 工作流
```
cronjob → 读positions/*（当前P值）+ events/近3天 + 00-宏观传导框架
     ↓
判断是否有≥2条不同信号源指向同一方向的风险
     ↓
有预警 → ①存完整版到 events/YYYY-MM-DD-risk-summary.md
          ②微信发摘要
          ③Hermes看完后手动判断是否补持仓事件链
无预警 → 静默
```

### 铁律
- ⛔ 不写 positions/ 任何文件
- ⛔ 不改任何 P 值
- ⛔ 自动化系统不做决策
- ✅ Hermes 看到新信号后，手动更新对应持仓的事件链

### 输出文件格式
`events/YYYY-MM-DD-risk-summary.md` 末尾必须附【影响持仓建议列表】

## 流程E（定时期）：P值与价格走势互锁验证 — 耦合进「决策回看·每两周」

**不是独立任务，是「决策回看·每两周」cronjob的第一步。**

```
cronjob 每14天(14:00)
    ↓
运行 p_price_verify.py
    → 读所有持仓的P值 + 实时净值/估算涨跌
    → 比对：P值高位但价格下跌？P值低位但价格上涨？
    → 输出验证报告到 events/YYYY-MM-DD-p-price-verify.md
    ↓
接着执行决策回看（判断过去14天的决策是否正确）
    → 验证通过的 ✅
    → 部分验证的 ⚠️
    → 验证失败的 ❌
    ↓
输出完整报告给大好人
```

**p_price_verify.py 位置:** `~/.hermes/scripts/p_price_verify.py`

**验证逻辑（脚本内）：**
| 场景 | 判断 |
|:----|:-----|
| P≥65% + 价格上涨 | ✅ 一致性验证通过 |
| P≥65% + 价格下跌>1% | ⚠️ 可能漏了利空 |
| P≤45% + 价格上涨>1% | ⚠️ 可能漏了利好 |
| 45%<P<55% | ⏸️ 中性，方向未定 |

**铁律：脚本只出报告、不改P值。背离信号由Hermes手动跟进。**

## 流程F：信源可信度评分（双向动态，耦合进3个入口）

### 入口①：日常处理博主内容时（流程A — Step 3.1之后）

**每次我处理完一个博主视频/文章，如果其中有明确预判，立即录制：**

```bash
python3 ~/.hermes/scripts/source_credibility.py --record <信源> <看多/看空> <资产代码> <P前> <P后> "说明"
```

**提前判断规则：**
- 有明确涨跌/趋势预判 → 录
- 有因果逻辑链+方向预判 → 录
- 纯解释/新闻复述 → 不录
- 框架级观点（无时间窗口） → 不录，标注"长期"即可

### 入口②：每日风险信号汇总（流程D）

**报告末尾追加信源可信度快照：**
```
📊 信源可信度概览
  ✅ 高采信（4-5★）: 宋鸿兵、岩松笔记、付鹏
  ⚠️ 方向预警（2★）: 蒋宇飞
  🚫 暂停（0★）: （无）
```

### 入口③：决策回看·每两周（流程E，第二部分）

**在P-Price验证之后，执行信源验证：**

```bash
cronjob 每14天(14:00)
    ↓
Step 1: p_price_verify.py (已有的)
    ↓
Step 2: source_credibility.py --pending
    → 列出所有待验证的预测
    → 人工/自动查实际走势
    → 逐条跑 --verify
    ↓
Step 3: source_credibility.py --report
    → 输出信源可信度报告
    → 标记需要关注的源（⚠️方向预警 / 🚫暂停）
    ↓
Step 4: 整合成综合报告推给大好人
```

### 前置校验规则补充（流程A — Step 1）

**当你处理博主视频/文章时，判断是否录制为预测：**

| 内容类型 | 是否录预测 | 示例 |
|:---------|:---------:|:-----|
| 明确涨跌/趋势预判 ✅ | 是 | "我判断黄金会涨""芯片还要跌" |
| 因果逻辑链+方向预判 ✅ | 是 | "利率上行→科技估值承压→看空芯片" |
| 纯解释/新闻复述 ❌ | 否 | "今天A股跌了因为...""这个政策内容是..." |
| 框架级观点（无时间窗口） ⚠️ | 可选，标注"长期" | "芯片产业链长期看多" |

**铁律：有方向+有验证窗口的才录预测。纯分析不录。**

### 命令速查

```bash
# 录制预测（日常处理博主时）
python3 ~/.hermes/scripts/source_credibility.py --record <信源> <看多/看空> <资产代码> <P前> <P后> "说明"

# 验证预测（回头看时）
python3 ~/.hermes/scripts/source_credibility.py --verify <信源> <索引> <涨/跌/平> [涨跌幅] [偏差说明]

# 查看报告
python3 ~/.hermes/scripts/source_credibility.py --report
python3 ~/.hermes/scripts/source_credibility.py --report <信源>   # 单人详细
python3 ~/.hermes/scripts/source_credibility.py --pending         # 待验证列表
python3 ~/.hermes/scripts/source_credibility.py --history <信源>  # 评分变化历史
```

### 数据位置
- 评分数据：`~/.hermes/output/source_credibility.json`
- 主控制表：`thinkers/INDEX.md`（星级规则+系数表）
- 单人档案：`thinkers/thinker-{name}.md`（可信度评分区）

## 流程G：用户要求审核资产时的标准流程（估值方法论驱动）

**当用户说"审核/看看/评估一下XX资产"时，按以下流程执行。** 

**核心原则：用 `00-估值方法论.md` 的估值框架审核，不是单纯报价格涨跌。**

### Step 1：加载两把尺

```
① 宏观尺 → 00-宏观传导框架.md 的「当前系统性折价系数」
② 估值尺 → 00-估值方法论.md 的丛林三问+所有者盈余
```

### Step 2：丛林三问

逐项回答，不能跳步：

**❶ 丛林里有没有鸟？（有价值/护城河吗？）**
- 这资产有什么价值？竞争优势是什么？
- 它是"有形可打折"（资产型）还是"未来利润"（成长型）？
- 如果看不懂 → 找懂行的人问（农场逻辑→对应thinker/博主）

**❷ 什么时候飞出来？（催化剂在哪？）**
- 向上催化剂是什么？给的预期时间窗口？
- 向下风险/利空是什么？有没有提前触发机制？
- 当前价格是已经price in了还是没有？

**❸ 有多少只鸟？（上行空间多大？）**
- 估算正常情况下的所有者盈余（利润+折旧-维持性资本开支）
- 跟无风险利率（当前约5.18%）比
- 给一个区间（不是单一点），取保守端做决策依据

### Step 3：跟现有贝叶斯P值对账

```
估值判断 → 方向是看多还是看空？
          ↓
跟 positions/{资产}.md 的当前P值对比：
- 方向一致 → ✅ 估值与信念相互印证
- 方向不一致 → ⚠️ 查事件链：是事件链漏了信息，还是估值判断有问题？
```

### Step 4：输出结论

格式参考：
```
【XX资产 — 估值审核】

❶ 丛林里有没有鸟？
   价值判断：有（理由：……）
   护城河：……

❷ 什么时候飞出来？
   催化剂：……（时间窗口）
   风险：……（提前触发条件）

❸ 有多少只鸟？
   正常年份盈余估算：……
   跟无风险利率比：……
   估值区间：……
   保守端取：……

现有P值：XX% | 跟估值方向：✅一致/⚠️有差距
操作建议：买入/持有/卖出/观察
```
