## 第零步（收件）：先收全，再分析

**用户被骂最狠的一条，放在最上面：**

```
收到新事件/新信息
  ↓
第①步：先收全所有待处理数据
  ├─ 所有博主今日视频都转录完了？（按INDEX.md逐个过，不跳）
  ├─ 昨天未处理的B站/知乎完成了？
  └─ 全部完成前，禁止往下走

第②步：验证"无更新"的博主 — 必须返回最新内容日期+标题
  ├─ [博主] → 最新: {标题} | {日期} → 已处理/有新内容/API坏
  ├─ 不接受的回答：❌ "无新增" ❌ "脚本返回0" ❌ "API可能坏了"
  ├─ API坏了 → 走浏览器验证
  └─ 浏览器也坏 → 说"扫不了，XX平台可能封/需重登录"

第③步：全部到手后 → 统一做三件事
  1. 串联旧框架+新信息
  2. 全资产贝叶斯更新（一次性覆盖）
  3. 出结果（推用户或推小龙虾）

第④步：hindsight_retain存进度
```

**一票否决：** 完成第①步前，禁止搜索/分析/改P值/创建文件/推给任何人。

## （新增）虚值贝叶斯 P' 规则

高置信度信源的预测观点不进主P值，走P'机制：

| 值 | 含义 | 依据 |
|:---|:-----|:-----|
| **P** | 主P值 | 仅基于已落地事实 |
| **P'** | 虚值参考 | P + 高置信度预测调整，括号标记 |

登记格式：`events/当天.md | 【虚值P']资产±Xpp→触发条件`
跟踪：events/宏观事件跟踪.md
自动转实：每天10:00 cron检查触发条件→达成后自动转→写入23-贝叶斯日志+positions事件链
全程留痕：为什么是虚值→触发条件→转实证据。

### ⚠️ P' 转实前置校验（2026-05-25 追加）

**P' 触发条件满足后，不能自动转实，必须先过动因校验：**

大宗商品 price action → 锁定价格涨跌的核心诱因，再决定是否转实：

```
条件满足（如油价3日累跌≥2%）
  ↓
第1步：问「为什么跌/涨？」
  ├─ 需求驱动（经济衰退/订单萎缩）→ 原条件成立，可转实
  ├─ 供给驱动（地缘缓和/增产/解封）→ 原条件作废，反向评估
  └─ 混合驱动 → 拆分量级，不搞一刀切
  ↓
第2步：盘面对照
  ├─ 相关板块走势与P'方向一致？→ 可转
  └─ 板块走势与P'方向相反？→ 条件误设，作废
  ↓
第3步：决定
  通过 → 转实，写证据入日志
  不通过 → 标记「条件误设」，留案底但不执行
```

**实操案例（2026-05-25）：** 环中星鉴预测「油价跌→利空芯片」，P'条件为油价3日累跌≥2%→芯片-3pp。油价实际从108跌到91.5（-15%），但跌因是美伊协议（供给解封），非需求衰退。盘面验证：科创50+5.7%，中芯+18%。作废，不转实。

## 第二步：管道处理流程

```
数据到手（API/浏览器）
  ↓
events/{日期}.md → 登记事件摘要+持仓影响
  ↓
thinkers/{博主}.md → 追加核心观点（首次出现的新信源才建）
  ↓
00-宏观传导框架.md → 更新对应章节（宏观/中观层）
  ↓
positions/{资产}.md → 贝叶斯P值更新（走5道门）
  ↓
23-贝叶斯更新日志.md → 更新总览表
  ↓
推成品（用户或小龙虾）
  ↓
hindsight_retain → 存进度到向量库
```

## 第三步：贝叶斯更新（精简）

### ⚙️ 5道门（缺一不可）

| 门 | 内容 | 检查 |
|:--|:-----|:----|
| **A** | 事件/系统/噪声分类 + 事实/观点分离 | 不是噪声？观点要分离 |
| **B** | 查宏观折价系数 | `00-宏观传导框架.md` 当前折价 |
| **C** | 查重脚本 + 逻辑簇去重 | `bash ~/桌面/脚本/07-贝叶斯查重检查.sh` |
| **D** | 定性检查（读positions文件） | 打中主要矛盾？ |
| **E** | 事实/观点分离 + 共识衰减 + 查重 | 首单±1pp/第二±0.5pp/第三0pp |

### 宏观三段式协议（新增2026-05-25）

```
Day①（事件日）：只动框架，不动P值
  → 更新events/ + 00-宏观传导框架.md
  → 登记到 events/宏观事件跟踪.md（Day计数=1）
  → 查领先指标（原油/美债/黄金等）
  
Day②~③（延迟窗口）：选中枢看持续
  → 连续2天方向一致 → 事件持久
  → 用价格反应幅度定pp赋值
  
Day⑤~⑦（一周收网）：一次性覆盖全部同链
  → 更新框架折价系数 + 全部同链持仓P值
  → 用 ⚙️ 系统性调整 标记
  → 日志标注「源自X月X日XX事件」
```

### 背离检测前置规则（新增2026-05-25）

```
发现P值vs价格方向不一致
  ↓
第(-1)步：查同链其他持仓是否也背离
  2~3+个同链同时背离 → 宏观框架漏了事件
  单个背离 → 走下一步
  
第〇步：拉近20日σ
  <1σ → 扰动，忽略
  1σ~2σ → 黄灯
  >2σ → 真背离！查原因
```

### 事件链五项强制校验

```
□ ① 时间顺序正确（日期从旧到新）
□ ② P值链连续（本条起始P = 上条结束P）
□ ③ 无重复起点
□ ④ 方向合理（利好→上调，利空→下调）
□ ⑤ 赋值区间合理（框架级±5~8pp，事件级±2~4pp）
```

## 第五步：关键坑（必读）

### 🚨 推成品给小龙虾（2026-05-22确立）
- ❌ 不准推半成品（"这个信息可能影响XX，你分析一下"）
- ✅ 只推成品（P值已更新、事件链已建好、持仓映射做完）

### 🚨 不猜价位（用户禁令）
- ❌ "我觉得能涨到3.5" — 蒙的
- ❌ 设止损/目标价
- ✅ 只问：外部逻辑还在吗？持有逻辑还在吗？

### 🚨 先验时间衰减
- 每7天无新信息 → P向50%靠3pp
- 每14天 → 累计向50%靠5pp
- 每30天 → 强制重置先验到50%

### 🚨 美元路径规则（利率传导时用）
判断利率对黄金利好还是利空：
- 美债涨+美元涨 → 流动性路径主导 → 黄金承压
- 美债涨+美元跌 → 衰退路径主导 → 黄金利好
