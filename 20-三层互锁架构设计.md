# 三层嵌套互锁架构设计

> **核心理念（大好人确立）：一个神经元不能跟更多神经元交互，就无法产生有意义的电信号，更不可能产生智能。**
> 改变贝叶斯 → 持仓文件跟着变；改变持仓文件 → 贝叶斯也变。
> 不是两个文件互相引用，是**一个实体的两个面**。

---

## 一、当前问题

### 现状

```
23-贝叶斯更新日志.md  ←交叉引用→  chip-etf-tracklog.md
```

仅交叉引用的两个独立文件，用户批评：「如果贝叶斯记忆不能直接对应到每一个持仓，就是空谈。」

### 根因

每项持仓的「叙事」（tracklog）和「概率」（贝叶斯）分居两个文件，更新时容易只改一个。即使都改了，读的时候也得跨文件关联。

---

## 二、核心设计：文件融合（File Fusion）

### 根本变化：每项持仓只有一个文件

**老方案（已废弃）：**
```
chip-etf-020628-tracklog.md    ← 叙事/事件
23-贝叶斯更新日志.md           ← 概率/更新
```

**新方案：**
```
positions/chip-etf-020628.md   ← 叙事+概率 = 一个文件
23-贝叶斯更新日志.md           ← 聚合索引（指向positions/*.md）
```

**每个 L3 持仓文件 = Tracklog + 贝叶斯链合一的单个文件。**
**23-贝叶斯更新日志 = 所有持仓文件的聚合索引视图。**

## 四、互锁机制（Interlocking Mechanism）

### 4.1 公共锚点：日期（YYYY-MM-DD）

每条记录以日期为锚点。同一个日期在 L3 文件中出现一次，同时包含 **叙事** 和 **概率**。

```
### 2026-05-18 大牛洲泡沫警告
**事件**: [叙事 — tracklog内容]
**贝叶斯更新**: 
  - 先验: P(上行)=X%
  - 推理: [逻辑链]
  - 后验: P(上行)=Y% (±Zpp)
  - 仓位影响: [动作]
  - 验证标记: ⏳/✅/❌
```

**这就是互锁**：写事件时必须写概率更新，读事件时同时读到概率变化。不可能只写事件忘了概率，也不可能只改概率丢了上下文。

### 4.2 三条互锁铁律

| # | 铁律 | 违反后果 |
|:--|:-----|:---------|
| 1 | **L3文件的每个日期条目必须同时包含「事件」和「贝叶斯更新」两个字段** | 信息孤岛，回到老问题 |
| 2 | **23-贝叶斯更新日志的每一行必须通过锚点链接指向L3文件的具体条目** | 变成另一个独立文件 |
| 3 | **L1只记录「有更新」的路牌，完整内容在L3** | 内部记忆膨胀 |

### 4.3 三层之间如何嵌套互锁（视觉化）

```
L3 (完整体系):
┌─────────────────────────────────────────────────────┐
│ positions/chip-etf-020628.md                        │
│                                                     │
│  ### 2026-05-18 ────────────── 公共锚点 ──────┐    │
│  **事件**: 大牛洲160PE警告             ←┐      │    │
│  **贝叶斯更新**:                        │      │    │
│    先验: 70% → 后验: 60%              ─┘      │    │
│  ── 叙事和概率在同一条目，无法分离 ──          │    │
│                                                     │
│  ### 2026-05-16                                     │
│  **事件**: 付鹏「存储=AI基建」                      │
│  **贝叶斯更新**: 60% → 70%                         │
└─────────────────────────────────────────────────────┘
                      ↑
                      │ anchor link (#2026-05-18)
                      ↓
L2 (聚合索引):
┌─────────────────────────────────────────────────────┐
│ 23-贝叶斯更新日志.md                                │
│                                                     │
│  | 芯片ETF | 70%→60% | 大牛洲160PE | ⏳ | [→] |    │
│  | 恒生科技 | 50%→60% | 腾讯三层蛋糕 | ⏳ | [→] |   │
│                                                     │
│  每行只有一行摘要 + 锚点链接到L3                    │
│  可被L3自动重建                                     │
└─────────────────────────────────────────────────────┘
                      ↑
                      │ "关注芯片P下行+10pp"
L1 (注意力):          │
┌──────────────────────┘
│ MEMORY.md (内部记忆 2200字内)
│  ...【互锁铁律】L3的持仓文件同时包含tracklog+贝叶斯链。
│  23-贝叶斯更新日志是聚合索引，可被L3重建。
│  今日关注: 芯片P下行+10pp ⏳待验证
└─────────────────────────────────────────────────────┘
```

### 4.4 两个方向的自锁

```
写入锁: 证据到达 → 打开L3 → 在同一条目写事件+概率 → 更新L2索引行
                                        ↕
读出锁: 读L3条目 = 同时读到"发生了什么"+"概率变化了多少+为什么"
```

## 五、更新流程

### 5.1 日常贝叶斯更新流程

```
[Step 0] 新证据到达（博主动态/财联社/数据发布）
    │
[Step 1] 三闸门过滤
    Gate1: 跟哪个资产相关？ → 定位到 positions/{asset}.md
    Gate2: 证据等级够吗？（坚实逻辑/数据 vs 纯观点）
    Gate3: 置信度变化≥10%？ → 如果否：记入待观察缓冲区，不入链
    │
[Step 2] 写入L3（核心互锁操作） ← 这是关键变化
    ┌─────────────────────────────────────────────────┐
    │ 打开 positions/{asset}.md                        │
    │ 追加一条:                                        │
    │                                                  │
    │ ### YYYY-MM-DD [标题]                           │
    │ **事件**: (叙事 — 什么发生了)                    │
    │ **贝叶斯更新**:                                  │
    │   - 先验: P(上行)=X%, 来源: (上期后验的日期)     │
    │   - 推理: (为什么这个证据改变了概率)             │
    │   - 后验: P(上行)=Y% (±Zpp)                     │
    │   - 仓位影响: (实际意义)                        │
    │   - 验证标记: ⏳待验证 / ✅验证正确              │
    └─────────────────────────────────────────────────┘
    │
[Step 3] 更新L2索引 ← 派生操作
    打开 23-贝叶斯更新日志.md
    在对应日期的表格中追加一行：
    | {资产} | {P变化} | {证据摘要} | {验证标记} | [→](positions/{asset}.md#{YYYY-MM-DD}) |
    │
[Step 4] 更新L1路牌（如变化显著）
    MEMORY.md 追加一条：「今日关注: {资产} P{方向}+{幅度}pp」
    │
[Step 5] 如到报告时间 → 报告从L3直接读取
```

### 5.2 无新信息时的处理

- **不产生空更新** — 没有 Gate3 通过的证据就不动 L3 文件
- **L2 索引不追加空行**
- **L1 路牌不更新**
- **报告中说**：「芯片ETF — 无新信息，维持 P(上行)=60%」

### 5.3 错误修正流程

```
[触发条件] 预测方向与实际走势相反
[操作]
1. 打开 positions/{asset}.md
2. 在被修正的条目旁追加注释：
   ⚠️ 修正: YYYY-MM-DD 回顾，实际走势为{X}，原判断偏差原因：{原因}
3. 不删原记录（错误本身是数据）
4. 在 L2 索引更新验证标记为 ❌
```

## 六、与报告的关联方式

### 6.1 日报中的「今日贝叶斯更新」区块

日报不复制完整推理，只从 L3 提取摘要 + 链接：

```
━━━ 今日贝叶斯更新 ━━━

🔄 有更新（2项）：
  【芯片ETF】70%→60% | 大牛洲160PE警告 | [完整链](positions/chip-etf-020628.md#2026-05-18)
  【恒生科技】50%→60% | 腾讯AI三层蛋糕 | [完整链](positions/hstech.md#2026-05-18)

➡️ 无更新：养殖/黄金/电池/银行/纳指 — 今日无相关信息
```

### 6.2 周报中的「贝叶斯周汇总」

```
━━━ 贝叶斯链周汇总 ━━━

本周有更新资产（2项）：
  【芯片ETF】P上行: 60%→70%→60% (↗↘)
    - 05-16 付鹏利好 +10pp → [详情](positions/chip-etf-020628.md#2026-05-16)
    - 05-18 大牛洲泡沫警告 -10pp → [详情](positions/chip-etf-020628.md#2026-05-18)
    - 结论: 国产替代逻辑vs泡沫风险双线拉扯，净变化0
  【恒生科技】P上行: 50%→60% (↗)
    - 05-18 大牛洲腾讯三层蛋糕 +10pp → [详情](positions/hstech.md#2026-05-18)
```

### 6.3 月报中的「置信度校准」

月报直接从 L3 读出所有月度条目，按资产汇总：

```
━━━ 月度置信度校准 ━━━

【芯片ETF】本月3次更新
  | 日期 | 更新 | 实际走势 | 判断 |
  |:----|:-----|:--------|:----|
  | 05-12 | 70% | ↑+2.1% | ✅ |
  | 05-16 | 60%→70% | ↑+1.5% | ✅ |
  | 05-18 | 70%→60% | ↔-0.3% | ⏳待验证 |

【恒生科技】本月1次更新
  | 05-18 | 50%→60% | ↔+0.8% | ⏳待验证 |
```

### 6.4 核心原则：不重复，只链接

**报告不承载完整推理。完整推理在 L3 文件里，报告只提供摘要和链接。**

优势：
- 报告保持紧凑（用户只读故事）
- 推理链条只有一份（永远一致，不会因多次拷贝而漂移）
- 任何位置都能沿着链接追溯到最原始的决策

## 八、关键设计决策

### Q1: 为什么不保留两个文件 + 更强交叉引用？

因为交叉引用是「事后补救」，不是「事前锁定」。

两个文件意味着：
- 写的时候可能只改一个 → 数据不一致
- 读的时候要跳来跳去 → 认知负担大

一个文件意味着：
- 写的时候必须同时写叙事和概率 → 天然一致
- 读的时候同时读到上下文和变化 → 理解完整

### Q2: 23-贝叶斯更新日志变成索引后，内容少了怎么办？

23 文件从「完整推理」变成「聚合索引」，完整推理在 positions/*.md 中。

好处：
1. 23 文件变小，L2 读取成本降低
2. 推理只在 L3 存一份 → 不会出现两份推理不一致
3. 23 文件变得可派生（可由脚本从 L3 自动生成）

### Q3: 一个信息源影响多个资产，怎么写？

每个资产独立条目，共享证据来源但不共享推理（因为推理是针对特定资产的）。

```
Example: 大牛洲视频同时影响芯片+恒生科技

操作：
1. 打开 positions/chip-etf-020628.md
   ### 2026-05-18 大牛洲: 科创50 160PE泡沫
   **事件**: 大牛洲视频指出科创50 PE 160倍...
   **贝叶斯更新**: [芯片专属推理]

2. 打开 positions/hstech.md
   ### 2026-05-18 大牛洲: 腾讯AI三层蛋糕
   **事件**: 大牛洲视频认为腾讯16倍PE低估...
   **贝叶斯更新**: [恒生科技专属推理]

3. 打开 23-贝叶斯更新日志.md
   在 2026-05-18 表加两行
```

### Q4: 已归档的旧条目怎么办？

```
归档条件：
- 条目距今 > 180天
- 或用户已清仓该资产 > 30天

归档操作：
1. 从 positions/{asset}.md 移到 positions/archive/{asset}-{year}Q{q}.md
2. 在 ARCHIVE_INDEX.md 追加一条

不影响互锁结构 — 归档后的文件仍然是「叙事+概率」合一的。
```
