# 完整机制流程说明书（2026-06-14 最终版）

> **一句话：** cron负责抓数据，我负责分析出日报，你只看微信+看板就够了。

## 一、整体架构：四层流水线

```
┌─────────────────────────────────────────────────────┐
│ 第一层：数据采集层（cron自动）                       │
│   监控抓取+转录 → 输出到 ~/.hermes/work/transcripts/ │
│   每2天16:00跑一次（0 16 */2 * *）                   │
│   只抓不分析，只存不归档                              │
├─────────────────────────────────────────────────────┤
│ 第二层：分析归档层（cron自动 20:00）            │
│   一天只出一份。检测全天新转录≥5条 → 读转录 → 提取信号 → 写events/      │
│   写看板（Sheet2+总看板）→ 出日报 → 发微信+邮件        │
│   <5条 → 安静，不发日报                               │
├─────────────────────────────────────────────────────┤
│ 第三层：你的手动信息（随时发给我）                     │
│   你发视频/观点/图片 → 我立刻处理 → 三处联动归档        │
│   → events/ + 看板 + narratives/                      │
│   → 如果有新P值影响 → 追加看板 → 下次日报合并输出       │
├─────────────────────────────────────────────────────┤
│ 第四层：交付层（微信+邮件）                            │
│   日报：完整标准格式（4叙事+8持仓+跟踪标的+黄金+一句总结）│
│   你问的资产：实时回答，不走日报排队                    │
└─────────────────────────────────────────────────────┘
```

## 二、各环节详细流程

### 2.1 Cron抓取+转录（数据采集）
```
定时: 每2天一次 16:00（0 16 */2 * *）
名称: 监控抓取+转录 (job_id: 10cb8ee72d02）

执行内容:
  ① 读断点（各博主独立last_ts，精确到秒）
  ② 三平台交替抓取（抖音→知乎→B站）
  ③ 下载新视频 → VideoCaptioner Docker转录
  ④ 转录原文保存到 ~/.hermes/work/transcripts/
  ⑤ 更新断点的"抓取终点"时间戳
  ⑥ 绝对不分析、不写events、不写看板

输出:
  - 转录文件: ~/.hermes/work/transcripts/{博主}_{视频ID}.txt
  - 断点更新: monitoring_checkpoint.json 的"抓取终点"字段
  - 结果摘要: 今天cron产出X个新转录
```

### 2.2 自动检测→出日报（核心新增）
```
定时: 每天13:00 + 20:00（分早/晚两批）
名称: 自动检测新转录→出日报 (job_id: 1c65f254924b）

执行内容:
  ① 读 checkpoint 的"看板归档截止时间戳"
  ② 检测 transcripts/ 目录，mtime > 截止时间戳的新文件
  ③ 判定"有效投资信号"（硬条件，不模糊判断）:
     - 提及持仓的基本面/价格变化 → 算
     - 提及跟踪标的的具体变化 → 算
     - 宏观大事件+明确方向判断 → 算
     - 纯情绪/个人生活/历史回顾/抽奖 → 不算
  ④ 有效信号≥5条 → 走完整管道:
     读转录→提取信号→写events/（一事件一文件）→
     写看板Sheet2（insert_rows + 逐列写）→
     写总看板Row5（4列事件）→
     出标准格式日报（4叙事+8持仓+跟踪标的+黄金+一句总结）→
     发all（微信+邮件）
  ⑤ 有效信号<5条 → 发一句"今天有效信号仅X条，不够出日报"
  ⑥ 有效信号0条 → 安静（不发任何消息）

输出:
  - events/{日期}-{事件名}.md
  - narratives/{日期}.md
  - pvalue_changelog.md
  - 看板更新（Sheet2 + 总看板）
  - 完整联合日报（标准格式）
  - 断点更新为精确时间戳
```

### 2.3 手动信息处理流程（你随时发给我）
```
触发条件: 你给我发视频/链接/文字/图片
执行内容:
  ① 读内容 → 确定是几个独立事件
  ② 每个事件独立归档:
     - events/YYYY-MM-DD-事件名称.md（含来源+观点+影响+P值建议+同步链）
  ③ 三处联动（缺一不可）:
     - 看板Sheet2对应持仓行（追加或新行）
     - 看板总看板Row5对应列
     - narratives/当天.md（多空格局更新）
  ④ 如果有P值调整:
     - 写pvalue_changelog.md
     - 实时回复你结论（不等日报）
  ⑤ 如果没有新P值: 
     - 回复你"方向一致/逻辑不变/无新信息"

铁律: 不准只归档不汇报。归档完必须告诉你结论。
```

```

## 🌍 四条叙事主线
① 主线名 🔴/🟢/🟡（带来源）

## 📊 全部持仓逐个过（8个）

用比喻不用术语
```

## 三、三处联动归档规则（事件→看板→narratives→看板，双向互指）

```
新信息进来
  └──→ events/YYYY-MM-DD-事件名称.md
           ├──→ narratives/YYYY-MM-DD.md（跟日报走，不出日报不写）
           │        每条判断旁标← events/哪个事件.md
           ├──→ 看板 Sheet2（对应持仓列Row5）
           │        行尾标← events/事件名称.md
           └──→ 看板 总看板Row5（对应列）
                    行尾标← events/事件名称.md
```

**铁律：events/文件末尾必须写看板位置，看板单元格末尾必须写←events/文件名。缺一个都不算完成。**

## 四、断点规则（精确时间戳，不是纯日期）

```
"看板归档截止时间戳": "2026-06-14 10:45:00"
"各博主下次抓取起点": {
    "付鹏": "2026-06-14 01:33:00",
    "蒋宇飞": "2026-06-14 01:35:00",
    ...
}

铁律: 
- 每个博主独立last_ts（精确到秒），脚本用ts>=last_ts比较
- 看板归档截止时间戳≠抓取终点。归档截止=分析写完的时间，抓取终点=cron跑完的时间
- 下次抓取从各博主各自的last_ts开始查，不会漏不会重
```

## 六、常见问题

### Q: 为什么早上的内容没自动出日报？
A: 之前没有"自动检测→出日报"这个cron。现在已经加上了（13:00+20:00各跑一次）。

### Q: 晚上宋鸿兵/付鹏发的视频会不会漏？
A: 20:00的cron会检测到晚间发布的新转录。如果≥5条有效信号，自动出日报发给你。

### Q: 同一天会不会出多份日报？
A: 不会。13:00的cron分析上午内容，20:00的cron分析晚间内容。但只有信号够（≥5条有效信号）才出日报。如果上午不够5条→不出，等20:00合一起出。如果两次都不够→全天不出，告诉你信号不够。

### Q: 你给我手动发的信息会不会被cron覆盖？
A: 不会。手动信息实时处理→实时归档→实时更新看板。cron只处理 transcripts/ 目录里的新转录文件，不会动你已经处理过的内容。

### Q: 如果我突然问"XX怎么样了"？
A: 实时回答，不走日报排队。不等到20:00才回你。
