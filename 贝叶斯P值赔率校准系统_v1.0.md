# 贝叶斯P值+赔率 动态校准系统 v1.0

## 核心原则

用户只看两个指标：
1. **贝叶斯P值** = 上涨概率（胜率）
2. **赔率** = 上行空间/下行风险（盈亏比）

**正期望操作条件：P值 × 赔率 > 1**

---

## 一、P值计算

```
P值 = 基础率(50%) + Σ(事件PP_i × 信源权重w_i)
```

### 1.1 事件PP分配

每个事件发生，根据强度给±Npp：
- 单一博主观点：±1~3pp
- 产业事实（财报/数据）：±2~5pp
- 多博主交叉验证：±3~8pp
- 重大宏观事件：±5~10pp

**虚P（预期事件）也参与P值计算** — 预期推动现实（索罗斯反身性）。但如果虚P长期不被价格验证，该信源的权重会自动衰减。

### 1.2 信源权重w_i（动态校准）

**方法：Spearman秩相关**

对每个信源S → 持仓H：

```
第1步：收集信源S影响持仓H的所有事件
  事件j: (日期t_j, PP_j)

第2步：找传导验证日的价格
  传导窗口L = 该信源对该持仓的最优传导时间
  验证价格 = 持仓H在t_j + L天后的涨跌幅%

第3步：算Spearman ρ
  X = [PP_1, PP_2, ..., PP_n]
  Y = [price_change_after_L_days_1, ..., price_change_after_L_days_n]
  ρ = spearmanr(X, Y)

第4步：w_i = ρ
```

**权重含义：**
- ρ = +1：该信源方向100%吻合价格 → 权重1.0
- ρ = 0：该信源与价格无关 → 权重0（忽略）
- ρ = -1：该信源方向总是反的 → 权重-1（反着来）

**样本不足处理：**
- <2事件：w=1.0（不衰减，无法判断）
- 2-4事件：用Spearman算，标注"低置信度"
- ≥5事件：权重正式生效

**传导窗口L估计：**
- 对每个信源S→持仓H，遍历lag k∈[0,30]天
- 找使事件PP与lag天后价格变化相关性最大的k
- L = argmax_k(相关性)
- 样本<3时默认L=5个交易日

**信源×持仓二维矩阵：**
- 付鹏对芯片的权重 ≠ 付鹏对养殖的权重
- 一个信源可能对A持仓很准，对B持仓很差

## 二、赔率计算

```
赔率 = 上行空间 / 下行风险
```

### 2.1 下行风险

```
下行风险 = (1 - P值) × 历史年化波动率 / √252 × √持有天数
```

- P值越低，下行概率越大
- 波动率越大，下行幅度越大
- 持有天数越长，不确定性越大

### 2.2 上行空间

```
上行空间 = P值 × 确定性系数 × 溢价系数
```

- **确定性**：独立信源指向同一方向的数量（三源互锁=高确定性）
- **溢价系数**：该持仓的历史弹性
  - 芯片ETF波动46% → 高溢价
  - 银行ETF波动13% → 低溢价

### 2.3 赔率解读

| 赔率 | 含义 | 操作建议 |
|:----|:----|:-----|
| > 3:1 | 极高赔率 | 即使P值51%也值得做 |
| 2-3:1 | 高赔率 | P值>55%即可操作 |
| 1.5-2:1 | 中等赔率 | P值>60%才操作 |
| 1-1.5:1 | 低赔率 | 需要P值>70% |
| < 1:1 | 负期望 | 不做 |

---

## 三、P值与赔率的互相约束

```
正期望 = P值 × 赔率 > 1
```

| 场景 | P值 | 赔率 | 正期望 | 操作 |
|:----|:----|:----|:------|:-----|
| 高胜率+高赔率 | 65% | 2.0:1 | 1.3 ✓ | 持有/加仓 |
| 低胜率+高赔率 | 51% | 3.0:1 | 1.53 ✓ | 可操作 |
| 高胜率+低赔率 | 70% | 0.4:1 | 0.28 ✗ | 方向对但赚不到钱 |
| 低胜率+低赔率 | 30% | 0.5:1 | 0.15 ✗ | 不做 |

**约束规则：**
1. P值<40% → 无论赔率多高，下行太大，不操作
2. P值>60% 但赔率<1:1 → 方向对但期望负，不追
3. P值50-60% 且赔率>2:1 → 正期望，可操作
4. P值>60% 且赔率>1.5:1 → 最佳，两条腿都稳

## 五、执行流程

```
每次抓取新事件后：
1. 事件分类（实P/虚P/宏观/产业/情绪）
2. 分配PP强度
3. 查信源权重矩阵（Spearman ρ）
4. P值 = 50% + Σ(pp × w)
5. 算赔率 = 上行/下行
6. 正期望判断
7. 写Excel看板
8. 积累数据，更新信源权重（每有新价格数据就重算Spearman）
```
