## 二、机制3v3：现金锁定系数（CLC）

### 公式
```
CLC = 信号概率 × 互锁强度 × 时间衰减²
```

### 变量定义
| 变量 | 定义 | 当前值 | 更新规则 |
|:----|:----|:------|:--------|
| 信号概率 | 达利欧预测AI泡沫概率 | 70% | 每月按信号计分板更新，底线55%上限85% |
| 互锁强度 | 三源互锁确认程度 | 1.0（三源强互锁） | 达利欧6月修正>2次→降级0.85 |
| 时间衰减 | 距触发月数/12的平方 | (5/12)²=0.174 | 每月自动重算 |

### 为什么用平方衰减
- **v2问题**：线性衰减导致越近触发日锁得越少（10月只锁5.8%），但10-11月正好是泡沫最脆弱的时候
- **v3改进**：平方函数→远离时多锁（6月锁12.2%），临近时少锁（10月锁0.7%）
- **逻辑**：离触发日远时，等待价值高→多锁；临近时，要么触发要么不触发→少锁准备行动

### 当前计算
```
6月：0.70 × 1.0 × (5/12)² = 0.122 → 锁定12.2%现金，可用87.8%
7月：0.70 × 1.0 × (4/12)² = 0.078 → 锁定7.8%现金
8月：0.70 × 1.0 × (3/12)² = 0.044 → 锁定4.4%现金
9月：0.70 × 1.0 × (2/12)² = 0.021 → 锁定2.1%现金
10月：0.70 × 1.0 × (1/12)² = 0.005 → 锁定0.5%现金
11月：0.70 × 1.0 × (0/12)² = 0.000 → 现金全解锁
```

## 三、机制4v3：折后仓位计算（4级β分级）

### 公式
```
折后仓位% = 原目标仓位% × (1 - CLC × β系数)
```

### 4级分级
| 级别 | β系数 | 含义 | 示例 | 逻辑 |
|:----|:----:|:----|:----|:----|
| L1高度暴露 | 0.8 | AI是主营收入主力 | 通富微电（AMD封测独供） | 如果AI泡沫破灭→AMD订单暴降→通富营收腰斩 |
| L2中度暴露 | 0.5 | AI是重要但非唯一 | 长电科技（客户多元化） | 如果AI泡沫破灭→英伟达订单↓但汽车/工业/消费电子还在 |
| L3弱相关 | 0.1 | AI对业绩影响可忽略 | 养殖ETF | AI泡沫与猪周期几乎无关 |
| L4反向受益 | -0.3 | AI崩盘时资金涌入避险 | 黄金ETF | AI泡沫破灭→避险资金流入→黄金涨 |

### 当前计算
```
通富微电：原33% × (1-0.122×0.8) = 29.8%
长电科技：原37% × (1-0.122×0.5) = 34.8%
养殖：原15% × (1-0.122×0.1) = 14.8%
电池：原14% × (1-0.122×0.1) = 13.8%
黄金：原8% × (1-0.122×-0.3) = 8.3%
```

### 分级逻辑
- **不是连续β**，是按"切断条件"二值判断：扛得住vs扛不住
- 通富：单一客户单一赛道→扛不住→L1
- 长电：客户多元化→扛得住→L2
- 两级就够了，区分L1和L1.5是虚假精度

## 四、机制5v3：虚P→实P转换（确认窗口+因果链）

### 转换条件
```
视觉信号（必要）：
  - 纳指七姐妹>15%回调（从52周高）
  - 或破200日均线

确认窗口（必要）：
  - 跌穿15%后维持5个交易日不反弹回15%以上
  - 防单日噪音和假触发

因果链（必要，至少2条）：
  ① 英伟达/AMD营收指引下调
  ② AI相关ETF（BOTZ/AIH）资金大幅净流出
  ③ 主流投行下调AI板块评级
  ④ Mag7裁员/资本开支削减
```

### 为什么加因果链
- **问题**：15%下跌可能是地缘冲击不是AI泡沫，两者操作完全不同
- **解决**：必须区分"为什么跌"，不能只看跌幅

### 转换后操作
- 虚P转实P→计入持仓P值
- CLC→0（现金全解锁）
- 折后仓位→恢复原目标仓位

## 六、执行流程

### 每日
1. 更新信号计分板（逐股计分，不是指数级）
2. 累计净计分

### 每月
1. 信号概率 = 70% + 上月净计分×5pp
2. CLC = 信号概率 × 互锁强度 × 时间衰减²
3. 折后仓位 = 原仓位 × (1 - CLC × β)
4. 检查虚P→实P转换条件

### 信号落地时
1. 视觉信号触发
2. 5个交易日确认窗口
3. 因果链≥2条
4. → 虚P转实P → CLC=0 → 现金解锁
