# 贝叶斯闭环验证方法论 — 信源可信度 ↔ P值 ↔ 价格走势 三方互锁

## 核心原理：控制论闭环

```
新闻/博主观点 → 更新贝叶斯P值 → 预测价格方向
                                        ↓
                               等待N天，观察实际价格
                                        ↓
                          ┌────────────────────────┐
                          ↓                        ↓
                     价格验证通过              价格验证失败
                      ✅ 拟合                   ⚠️ 背离
                          ↓                        ↓
                 信源可信度+1                 信源可信度-1
                 P值维持                     检查：漏了信息还是逻辑错了？
                                               ↓
                                         修正P值或修正信源权重
```

## 第一步：背离检查（每周执行）

### 怎么检查

```
对于每项持仓：
  取 positions/*.md 中的 P(上行) 值
  取天天基金API的实时净值
  对比：
    P≥55% 看多 + 价格涨 → ✅ 拟合
    P≥55% 看多 + 价格跌 → ⚠️ 背离（看多逻辑有问题）
    P≤45% 看空 + 价格跌 → ✅ 拟合
    P≤45% 看空 + 价格涨 → ⚠️ 背离（可能漏了好消息）
    P=46%-54% 中性区 → 不报警
```

### 背离后怎么处理

| 背离类型 | 可能原因 | 处理方式 |
|---------|---------|---------|
| P看多但价格跌 | ①漏了利空新闻 ②市场情绪/流动性主导 ③核心逻辑变了 | ①补信息→重算P值 ②标记为"流动性扰动" ③重设先验 |
| P看空但价格涨 | ①漏了利好新闻 ②技术性反弹 | ①补信息→重算P值 ②等趋势确认 |

**连续2周背离 → 自动向50%回退3pp**（时间衰减规则）

### 实际操作

每周日10:00自动执行脚本：
```bash
python3 ~/.hermes/scripts/bayesian_divergence_check.py
```

输出报告推给大好人微信。

## 第二步：信源可信度评分（每两周更新）

### 评分规则

每条预测记录：{信源, 方向, 时间, P变化, 实际结果, 准确?}

准确率计算：
```
准确率 = 准确次数 / (准确次数 + 不准次数) × 100%
```

权重调整：

| 准确率 | 权重等级 | 贝叶斯更新时影响 |
|:-----:|:-------:|:---------------:|
| ≥80% | 高(信任) | +3pp/条，上限+10pp |
| 60-79% | 中(标准) | +2pp/条，上限+8pp |
| 40-59% | 低(谨慎) | +1pp/条，上限-8pp（单博主非共识最高-8pp规则） |
| <40% | 暂停 | 该信源暂时不从其更新P值 |

### 实际操作

每两周推送报告：
```bash
python3 ~/.hermes/scripts/source_credibility.py --report
```

输出推给大好人微信。

## 第三步：P值更新规范（每次更新时执行）

### 更新流程

```
收到新信息（新闻/博主视频）
  ↓
① 判断信息质量（有无数据支撑？）
  ↓
② 判断信息来源可信度（查thinker档案中的权重）
  ↓
③ 判断是否与已有逻辑簇重复（同一逻辑不重复+pp）
  ↓
④ 贝叶斯计算后验
  ↓
⑤ 更新 positions/*.md + 23-贝叶斯更新日志.md
  ↓
⑥ 记录预测到信源可信度系统（source_credibility.json）
    等待未来价格验证
```

### 防偏差铁律（21-贝叶斯防偏差规则.md）

1. **逻辑簇去重** — 同一逻辑不看来源/角度/次数，只加一次pp
2. **反向预设** — 每项资产预设利空触发规则，不等到亏了再找原因
3. **先验时间衰减** — 每7天无新信息P向50%靠3pp
4. **价格大波动时旧对立观点先作废再叠新信息**
5. **单博主非共识最高-8pp**
6. **卖出三层** — P>50%持有，P<50%期望值负考虑卖，P≤20%无脑清仓

## 第四步：仓位建议规范

### 基本原则
1. **右侧交易** — 等反转信号，不猜底不左侧扛
2. **逻辑没松动 ≠ 加仓** — 逻辑没松是"不卖"，加仓要等额外触发条件
3. **机会成本优先** — 钱拿出来放哪更划算？不只看亏了多少
4. **P值+仓位结构双看** — 重仓(万元级)重点分析，蚊子仓(10元级)忽略

### 什么情况不能加仓
| 情况 | 原因 | 怎么做 |
|:----|:----|:------|
| P<55% | 上行概率不够高 | 等P值回升再考虑 |
| 价格在下跌通道中 | 右侧交易纪律 | 等企稳信号 |
| 宏观压制因素未解除 | 系统性风险 | 等宏观转好 |
| 逻辑链中有未验证环节 | 不确定性高 | 等验证结果 |

### 什么情况可以加仓
| 情况 | 原因 | 怎么做 |
|:----|:----|:------|
| P≥60%且价格企稳 | 上行概率明确+趋势配合 | 分步建仓 |
| 新催化剂出现 | 新逻辑簇打破原有平衡 | 评估后更新P值，达标就进 |
| 安全边际足够（大幅回调） | 价格低于合理估值 | 按策略入场（但非左侧抄底） |
