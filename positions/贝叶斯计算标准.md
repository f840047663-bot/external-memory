## 六、贝叶斯计算标准（2026-05-18 定版）

### 6.1 核心公式

```
先验赔率 = P(上行) / P(下行)
似然比 (LR) = P(证据 | 假设真) / P(证据 | 假设假)
后验赔率 = 先验赔率 × LR
后验P(上行) = 后验赔率 / (后验赔率 + 1)

LR > 1 → 证据支持假设 → P上行↑
LR < 1 → 证据削弱假设 → P下行↑
LR = 1 → 证据无信息 → P不变
```

### 6.2 信息来源与置信度

| 信息类型 | 典型可信度范围 | 举例 |
|:---------|:-------------|:-----|
| 官方新闻/政策/数据 | P(E|...)设在60%-90% | 美联储声明、猪价数据、产能调控方案 |
| 博主观点+坚实逻辑链 | P(E|...)设在50%-80% | 付鹏芯片分析、大牛洲三层蛋糕、伊娃猪周期 |
| 博主纯观点/喊话 | P(E|...)设在40%-60% | "会涨""会跌"没原因 |
| 净值/价格 | 不参与推理 | 只做月校准核验 |

### 6.3 每条更新的标准格式

```
贝叶斯更新:
  先验: P(上行)=X%, P(下行)=Y%（来源：之前哪次更新）
  新证据: [具体信息]
  P(E|H)=Z% （推理：为什么假设真时这个证据会出现）
  P(E|¬H)=W% （推理：为什么假设假时这个证据会出现）
  似然比=Z/W
  后验赔率=先验赔率×(Z/W)
  后验: P(上行)=A%, P(下行)=B%
  仓位影响: ...
  验证标记: ⏳待验证 | YYYY-MM检查
```

### 6.4 净值在系统中的正确角色

**净值≠推理原料。** 净值是事实层面的结果，只在以下场合使用：
1. **月校准**：对比月初P(上行)预测 vs 月末净值方向
2. **触发检查**：银行001595净值是否跌破1.60
3. **仓位计算**：当前盈利/亏损状态
4. **异常报警**：单日净值波动>±4%触发归因

净值永远不参与贝叶斯的P(E|H)/P(E|¬H)计算。

### 6.5 pp赋值规则表（2026-05-22 补充 — 防主观拍脑袋）

| 档次 | 条件 | 例子 |
|:----|:-----|:-----|
| ±5pp | 多源同向共识 / 同一事件链上多事件叠加 | 碳酸锂跌价+巴克莱看多+特斯拉+5 |
| ±3~4pp | 单源坚实逻辑+关键数据 / 政策已落地 | 付鹏风险极值-3、8部门整治-4 |
| ±2pp | 单源逻辑但无关键数据支撑 | 宋鸿兵利率风暴→避险+2 |
| ±1pp | 确认性/重复/微调信息 | 机构研报再次确认+1 |
| ±0pp | 逻辑簇重复 / 信源权重不足 | 闻号说经济⭐⭐仅标记不调 |

> ⚠️ **上下限：** 单次更新不超过±10pp。同一标的单日更新不超过±15pp。

### 6.6 时间衰减规则

每7天无新信息 → P向50%靠拢3pp。

示例：05-19黄金+8pp→P=83%。如果到05-26无新增利好，自动衰减-3pp→P=80%。如果到06-02仍无新增，再-3pp→P=77%。

### 6.7 逻辑簇去重规则

**同一逻辑簇，同一方向，只加一次pp。** 后续同逻辑簇新信息视为"确认/递进"，最多+1pp（确认性增量），不得叠加。

| 场景 | 处理 |
|:----|:-----|
| 第一个信源说"A→利好" | 正常+pp |
| 第二个信源也说"A→利好" | ±0pp（逻辑簇已存在） |
| 第三个信源提供A的递进数据 | +1pp（递进验证，非重复） |
| 4个以上信源共识A | +1pp总计（不叠加） |

### 6.8 多源同逻辑处理

**同一个逻辑点，10个人说也只比正常多加1~2pp，不叠加。**
- 单人首提：正常pp（±3~5）
- 第2~3人确认：+1~2pp（多源共识溢价）
- 第4+人再说：±0pp（饱和）
