## 六、贝叶斯计算标准（2026-05-18 定版）

### 6.1 核心公式

```
先验赔率 = P(上行) / P(下行)
似然比 (LR) = P(证据 | 假设真) / P(证据 | 假设假)
后验赔率 = 先验赔率 × LR
后验P(上行) = 后验赔率 / (后验赔率 + 1)

LR > 1 → 证据支持假设 → P上行↑
LR < 1 → 证据削弱假设 → P下行↑
LR = 1 → 证据无信息 → P不变
```

### 6.2 信息来源与置信度

| 信息类型 | 典型可信度范围 | 举例 |
|:---------|:-------------|:-----|
| 官方新闻/政策/数据 | P(E|...)设在60%-90% | 美联储声明、猪价数据、产能调控方案 |
| 博主观点+坚实逻辑链 | P(E|...)设在50%-80% | 付鹏芯片分析、大牛洲三层蛋糕、伊娃猪周期 |
| 博主纯观点/喊话 | P(E|...)设在40%-60% | "会涨""会跌"没原因 |
| 净值/价格 | 不参与推理 | 只做月校准核验 |

### 6.3 每条更新的标准格式

```
贝叶斯更新:
  先验: P(上行)=X%, P(下行)=Y%（来源：之前哪次更新）
  新证据: [具体信息]
  P(E|H)=Z% （推理：为什么假设真时这个证据会出现）
  P(E|¬H)=W% （推理：为什么假设假时这个证据会出现）
  似然比=Z/W
  后验赔率=先验赔率×(Z/W)
  后验: P(上行)=A%, P(下行)=B%
  仓位影响: ...
  验证标记: ⏳待验证 | YYYY-MM检查
```

### 6.4 净值在系统中的正确角色

**净值≠推理原料。** 净值是事实层面的结果，只在以下场合使用：
1. **月校准**：对比月初P(上行)预测 vs 月末净值方向
2. **触发检查**：银行001595净值是否跌破1.60
3. **仓位计算**：当前盈利/亏损状态
4. **异常报警**：单日净值波动>±4%触发归因

净值永远不参与贝叶斯的P(E|H)/P(E|¬H)计算。
