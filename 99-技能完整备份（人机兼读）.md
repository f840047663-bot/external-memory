## 技能1：cookie-extractor（Chrome < v147 专用）

**用途：** 从 Chrome 提取登录 cookie，用于 curl/API 访问需要登录态的平台。

**⚠️ 限制：** Chrome v147+ 的 v11 加密不可破解，yt-dlp 只能提取 44/229 个非关键 cookie（z_c0 拿不到）。不要试任何 Python 解密方案。

### 操作步骤

**Step 1：提取 cookie**
```bash
yt-dlp --cookies-from-browser chrome --cookies /tmp/cookies.txt "https://www.example.com" 2>/dev/null
```
先决条件：Chrome 正在运行且已登录目标平台。**绝对不要杀用户 Chrome。**

**Step 2：验证登录态**
```bash
curl -s --cookie /tmp/cookies.txt 'https://目标平台.com/api/v4/me'
```

**Step 3：用 cookie 干活**
```bash
curl -s --cookie /tmp/cookies.txt -H 'Accept: application/json' 'https://目标平台.com/api/v2/some_endpoint'
```

### 已知坑
- 需要 Chrome 正在运行
- cookie 会过期，失效了重跑 Step 1
- 输出 Netscape 格式，curl 直接认
- URL 必须用单引号包裹（防 `[*]` glob 展开）

### 输出文件
`/tmp/cookies.txt`

## 技能2：local-chrome-cdp-bridge（Chrome v147+ 唯一方案）

**用途：** 通过 Chrome DevTools Protocol (CDP) 连接本地 Chrome 调试端口，用 WebSocket 提取 cookie。Chrome v147+ 时是**唯一**能拿到 z_c0 的方法。

### 全自动流程（AI 执行，用户不用管）

**Step 1：先试现有 cookie**
```bash
curl -s --connect-timeout 5 \
  -H "Cookie: $(cat ~/桌面/zhihu_com_cookie_最新.txt 2>/dev/null)" \
  "https://www.zhihu.com/api/v4/me" | grep -c '"name"'
# 返回 1 → cookie 有效，直接干活
# 返回 0 → 执行下面
```

**Step 2：重启 Chrome（调试模式）**
```bash
# 清理旧 profile + 杀旧 Chrome
rm -rf /tmp/chrome-debug
mkdir -p /tmp/chrome-debug
ln -sf ~/.config/google-chrome/Default /tmp/chrome-debug/Default
kill -9 $(ps aux | grep "[c]hrome" | awk '{print $2}') 2>/dev/null

# 用 crontab 启动（Wayland 下唯一可靠方式）
crontab -l | grep -v "remote-debugging\\|chrome-debug\\|One-shot" > /tmp/cron_new
echo "# Chrome CDP" >> /tmp/cron_new
echo "$(date -d '+1 minute' '+%M %H %d %m *') WAYLAND_DISPLAY=wayland-0 XDG_RUNTIME_DIR=/run/user/1000 DBUS_SESSION_BUS_ADDRESS=unix:path=/run/user/1000/bus google-chrome --remote-debugging-port=9222 --remote-allow-origins=* --user-data-dir=/tmp/chrome-debug --no-first-run --ozone-platform=wayland > /tmp/chrome_cdp.log 2>&1" >> /tmp/cron_new
crontab /tmp/cron_new
rm /tmp/cron_new
```

**Step 3：等 9222 端口就绪**
```bash
for i in $(seq 1 30); do
  curl -s --connect-timeout 2 http://127.0.0.1:9222/json/version && echo "OK" && break
  sleep 4
done
```

**Step 4：提取 cookie**
```bash
python3 ~/.hermes/scripts/cdp-get-cookies.py --domain zhihu.com --test --output ~/桌面/
```

**Step 5：用 cookie 干活**
```bash
curl -H "Cookie: $(cat ~/桌面/zhihu_com_cookie_最新.txt)" \
  "https://www.zhihu.com/api/v4/me"
```

### 核心脚本
`~/.hermes/scripts/cdp-get-cookies.py` — 通用脚本，`--domain` 支持任何网站，`--test` 自动验证

### 铁律
1. `--remote-allow-origins=*` **必须加**，否则 WebSocket 连不上
2. `--user-data-dir` 必须用非默认目录（`/tmp/chrome-debug`）
3. **不要杀用户正在用的 Chrome** — 先报备
4. Chrome 重启会闪一下，标签页自动恢复
5. cookie 约 7-30 天过期，过期重跑即可
6. **桌面 cookie 文件路径：** `~/桌面/{domain}_cookie_最新.txt`

### 验证 cookie 是否有效
```bash
# 知乎
curl -s -H "Cookie: $(cat ~/桌面/zhihu_com_cookie_最新.txt)" \
  "https://www.zhihu.com/api/v4/me" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['name'], d['headline'])"
```

## 技能3：waf-bypass

**用途：** 当目标网站 WAF（如知乎 40362）封禁服务器 IP 时，用手机 UA 绕过。

### 核心原则
不要用 headless Chrome（指纹已被标记）。正确方法：curl + 手机 UA。

### 手机 UA 绕过（已验证有效）
```bash
curl -s --cookie /tmp/cookies.txt \
  -H 'User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.0 Mobile/15E148 Safari/604.1' \
  -H 'Accept: application/json' \
  -H 'x-requested-with: XMLHttpRequest' \
  'https://目标平台.com/api/v4/some_endpoint'
```

### WAF 拦截判断

| 症状 | 判断 |
|------|------|
| curl 返回 40362 | WAF 拦截，用手机 UA |
| JSON 解码报错 `Expecting value` | WAF 静默拦截，先看原始输出，切手机 UA |
| browser_navigate 只出"美观输出"checkbox | headless Chrome 被识别 |
| 返回 `"code":4041` | 非 WAF，内容不存在 |

## 技能4：zhihu-reader

**用途：** 读取知乎帖子/回答的完整内容。依赖技能1/2（cookie）+ 技能3（绕WAF）。

### 依赖执行顺序
```
cookie（技能1或2） → waf-bypass（可选）→ zhihu-reader
Chrome v147+ → 技能2 (CDP)
Chrome <147  → 技能1 (yt-dlp)
```

### 从 URL 提取 ID
```
https://www.zhihu.com/question/2036008867021186957
                               └─ question_id ─┘

https://www.zhihu.com/question/2016832855842522430/answer/2039402143422083320
                               └─ question_id ─┘         └─ answer_id ─┘
```

### 知乎 API 端点

| 用途 | 端点 |
|------|------|
| 读问题下的回答 | `/api/v4/questions/{question_id}/answers` |
| 读单个回答 | `/api/v4/answers/{answer_id}` |
| 读专栏文章 | `zhuanlan.zhihu.com/p/{article_id}` |
| 取当前用户 | `/api/v4/me` |
| 搜索用户/内容 | `/api/v4/search_v3?q={关键词}&type=people` |

### 方式 A：读问题下的回答（推荐）
```bash
curl -s --cookie /tmp/cookies.txt \
  -H 'User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.0 Mobile/15E148 Safari/604.1' \
  -H 'Accept: application/json' \
  -H 'x-requested-with: XMLHttpRequest' \
  'https://www.zhihu.com/api/v4/questions/{QUESTION_ID}/answers?include=data[*].content,author.name,voteup_count,comment_count,created_time&limit=5&offset=0&order=default'
```

### 方式 B：读单个回答
```bash
curl -s --cookie /tmp/cookies.txt \
  -H 'User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.0 Mobile/15E148 Safari/604.1' \
  -H 'Accept: application/json' \
  'https://www.zhihu.com/api/v4/answers/{ANSWER_ID}?include=content'
```

### 方式 C：搜索用户
```bash
curl -s --cookie /tmp/cookies.txt \
  -H 'User-Agent: Mozilla/5.0 (iPhone; CPU iPhone OS 17_0 like Mac OS X) AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.0 Mobile/15E148 Safari/604.1' \
  -H 'Accept: application/json' \
  "https://www.zhihu.com/api/v4/search_v3?q={关键词}&limit=10&type=people"
```

### 内容提取（Python）
```python
import re, json
content = re.sub(r'<[^>]+>', '', raw_content)
text = content.replace('\n\n\n', '\n\n').strip()
```

### 排除清单

| 症状 | 原因 | 解法 |
|------|------|------|
| 4041 "荒原" | 帖子不存在/已删除 | 让用户确认链接 |
| 40362 WAF | IP被封 | 用手机 UA |
| 空响应 | WAF 静默拦截 | 加 Accept/x-requested-with header |
| 返回无 content 字段 | 缺 `?include=content` | 加 include 参数 |
| curl 报 "bad range" | `[*]` 被 shell 展开 | URL 用单引号 |

## 技能5：topic-monitor（编排器）

**用途：** 组合 cookie-extractor/CDP + waf-bypass + zhihu-reader，从搜索 → 读帖 → 分析 → 长期跟踪全流程。

### Phase 1：搜索话题
```bash
curl -s --cookie /tmp/cookies.txt \
  -H 'User-Agent: Mozilla/5.0 (iPhone ... ) Safari/604.1' \
  'https://www.zhihu.com/api/v4/search_v3?q={关键词}&limit=10'
```
筛选标准：点赞>5或回答数>3，1年内，标题命中核心关键词。

### Phase 2：读帖
按技能4 (zhihu-reader) 流程读取。

### Phase 3：分析（三维度框架）

| 维度 | 问什么 |
|------|--------|
| 叙事一致性 | 回答之间观点是否打架？主流叙事是什么？ |
| 逻辑链检验 | 每个观点的论据硬不硬？（数据/案例/推理） |
| 反常检测 | 有没有跟主流叙事相反的异见？新角度？ |

输出格式：
```
话题: XXX
主流观点: ...
异见观点: ...
跟持仓关系: ...
置信度: ...
关注点: ...
```

### Phase 4：归档到 L3
分析结果写入外部记忆存档，格式：
```
## 追踪: {话题名}
### 2026-05-19 首次分析
来源: 链接
核心发现: ...
主流观点: ...
我的判断: ...
```
