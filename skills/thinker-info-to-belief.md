# 博主信息处理流程

## 触发条件

任何博主（蒋宇飞/付鹏/宋鸿兵/古都闲云/伊娃等）的新视频/文章内容首次处理时。
也用于：前瞻扫描（每周日10:00自动跑BTC流动性/黄金占比/板块估值百分位/宏观指标）、决策回看（每两周14:00自动：对比决策vs结果→给信息源打分）。

### 🚨 用户问持仓时 = 自上而下整链回答（2026-05-22 用户亲自确立）

**核心规则：** 用户问任何持仓问题时，必须从 🌍宏观框架 → 🏭中观传导 → 📦微观持仓，自上而下串成完整链条回答。**不能跳过前两层直接报P值。**

```text
用户问：「黄金现在怎么样？」
  ↓
✅ 正确：
  ① 先看00-宏观传导框架.md: 当前美债收益率压制（§一）+ 美国财政撑不住加息（环中星鉴）+ 地缘紧张未缓（§二）
  ② 再看中观：资金从成长股→硬资产的结构性转移
  ③ 最后微观：黄金P=83%维持，短期被压但长期逻辑更扎实
  → 一条从上往下流的信息链

❌ 错误：
  「黄金P=83%，持有」
  → 断层，用户看不出判断依据
```

**问持仓前必读文件顺序：**
1. `00-宏观传导框架.md` → 知道宏观/中观在发生什么
2. `positions/{资产}.md` → 看具体P值和事件链
3. `thinkers/INDEX.md` → 看最近哪个博主说过相关观点

❌ 不读框架直接跳位置文件 = 建了框架但没用 = 白建。用户原话：「你假如不看的话，那就完全就写这个东西就没啥意义了」

### ⚠️ 新Thinker首次处理：文章+回答都要读

建立新thinker档案时，**不能只看文章**。知乎用户的回答往往更有实时性和针对性：

```bash
# 文章列表（看他的专栏内容方向）
GET /api/v4/members/{user_token}/articles?limit=20
# 回答列表（看他近期在讨论什么热点，跟持仓相关度更高）
GET /api/v4/members/{user_token}/answers?limit=20
```

**阅读范围规则（2026-05-22用户纠正后确立）：**
- ✅ 只读最近10-20天的内容
- ✅ 只读跟用户持仓相关的（看标题判断相关性，无关跳过）
- ✅ 要读文章也要读回答，两者都查
- ✅ 不要默认文章是付费的——先试API，失败再用浏览器兜底
- ❌ 不要翻到几个月前的旧文章（会被用户训斥）
- ❌ 不要只看文章不查回答（环中星鉴就主要靠回答输出）

### ⚠️ 内容处理顺序：先定宏观/中观/微观层级

读完内容后，先判断它在三层中的位置：

| 层级 | 判断标准 | 写到哪里 |
|:----|:---------|:--------|
| 🌍 **宏观层** | 利率/美联储/地缘/关税/经济数据/全球流动性/居民杠杆 | **只写进00-宏观传导框架.md** |
| 🏭 **中观层** | 行业板块叙事/资金流向/板块轮动/产业链逻辑 | **写进00-宏观传导框架.md**的中观传导区块 |
| 📦 **微观层** | 具体基金/公司/ETF的新闻、净值、公告 | **写进positions/{资产}.md** |

**重要：** 宏观内容提到微观影响 → 宏观文件写主要分析，微观文件只带一嘴。不要重复写到多个持仓文件里。

这个三层自上而下的顺序不能跳。用户问任何持仓时，必须先读00-宏观传导框架.md看宏观/中观状态，再读positions/具体持仓。

## 核心原则

> **信息本身不等于更新。必须说清楚：以前P(上行)=X%（因为A理由）→ 现在P(上行)=Y%（因为B理由）。**

> **新闻收集是主要的，贝叶斯是自然而然的结果。** 先有信息链，再有概率更新。不要反过来。

> **不要瞎编。** 不知道某资产受什么宏观事件影响→查/搜/问，而不是编一个连接。

> **时序决定一切。** 新闻必须严格按时间排序，时序乱了贝叶斯就是乱算的。过去的后验=未来的先验。

> **用价格校准。** 每个事件发生后看价格怎么走（当日/次日净值变化），用来验证这个事件的实际影响力。不是过拟合，是看大趋势。

> ⚠️ **不过拟合价格。** 用户原话：\"不需要拟合的太好，太紧了会过拟合。这个就是参考，我顶多看大的趋势。\" 只看大方向的确认/背离，不算精确的相关性系数。|

## 第〇步-乙：信息/事件分类层级 + 持仓映射

> 收到一条新的信息/事件（博主视频、新闻、数据等），标准处理流程：

### ① 判断层级（宏观/中观/微观）

| 层级 | 判断标准 | 应该写到哪里 |
|:----|:---------|:------------|
| 🌍 **宏观层** | 利率/美联储/地缘/关税/经济数据/全球流动性/居民杠杆 | **只写进 `00-宏观传导框架.md`** |
| 🏭 **中观层** | 行业板块叙事/资金流向/板块轮动/产业链逻辑 | **只写进 `00-宏观传导框架.md`** 的中观传导区块 |
| 📦 **微观层** | 具体基金/公司/ETF的新闻、净值、公告 | **写进 `positions/{资产}.md`** |

### ② 逐项持仓映射

分类后，**逐一检查每个持仓**，按这个标准流程走：

```
问题1：这条信息是否影响该资产？           → 是/否（无关就跳过）
问题2：如果影响，方向是什么？             → 🟢利好 / 🔴利空 / 🟡中性
问题3：逻辑传导链是什么？                → 2-3句话说明事件如何传导到该资产
问题4：当前P值是多少？这条信息改不改P？   → ①改P，②不改P（已体现），③巩固逻辑（不调P但加固叙事）
```

**实操示例（2026-05-22 无知半人「前4月住户贷款狂减4902亿」）：**

```
分类：🌍 宏观层 → 经济周期传导路径（00-宏观传导框架.md §三）

逐项映射（从23-贝叶斯更新日志.md 拉出所有持仓的当前P，逐个过）：
银行001595     (57%) → 🔴 住户贷款是银行零售核心，4902亿减少＝息差受损 → 巩固已有下调趋势
养殖ETF        (52%) → 🔴 住户不借钱→不敢消费→猪肉消费端继续承压
恒生科技       (58%) → 🔴 居民消费萎缩→内需型科技股业绩承压
有色金属008826 (45%) → 🔴 经济走弱→工业金属需求预期下降
黄金(实物)     (83%) → 🟢 住户去杠杆＝经济衰退信号→避险逻辑再夯实
债券012858     (52%) → 🟢 经济差→宽松预期利好债市，但有付鹏滞胀对冲
自由现金流ETF  (55%) → 🟢 防守型配置，经济下行周期吸引力上升
电池ETF 018926 (75%) → 🟡 间接影响（渗透率60%+出口对冲内需）
芯片ETF 020628 (61%) → 🟡 几乎无关（芯片偏政策驱动+独立AI基建逻辑）
```

### ③ 互锁标注 + 跨持仓扫描

- 宏观事件 → 标注 `→ 中观: 科技板块🔴 避险板块🟢`；中观 → 标注 `→ 微观: 芯片ETF🔴`
- 持仓文件只引用宏观/中观结论，**不重复**宏观事件原文

**跨持仓扫描（重要）：** 如果在映射中发现 **2条以上不同来源的信号指向同一方向** → 主动预警用户。

> ⚠️ **踩坑教训：** 付鹏预警+宋鸿兵利率风暴+古都闲云贷款负值+大牛洲泡沫警告——4个不同来源全指向「美债利率上升→A股回调」。因为没做跨持仓扫描，漏了主动预警。详见 cron `每日风险信号汇总`。

### ❌ 错误做法 vs ✅ 正确做法

```
❌ 付鹏说美债5.18% → 直接写进 chip-etf-020628.md 和 gold.md → 同一事件在3个持仓文件里重复 → 冗余且不一致

✅ 付鹏说美债5.18% → 只写进 00-宏观传导框架.md §一
   → 标注「→ 中观: 科技🔴 避险🟢」
   → 持仓文件只写「受宏观美债传导影响(见00-宏观传导框架§一)」
```

## 第一步：先定宏观框架，再处理信息

> **当博主论述被数据验证但纯ETF不存在的处理方式（2026-05-20 实战）：**
> 蒋宇飞MLCC/堆叠逻辑硬、市场数据吻合，但不存在纯MLCC ETF，用户只能买个股（风华高科/三环集团）或已有的芯片ETF（芯+存+叠已覆盖）。
> **方案排位：** ①如果用户已有能覆盖该主题的ETF → 优先补仓已有仓位，不开新仓 ②如果用户想买个股但从未操作过 → 明确告知风险，让用户自己决定 ③如果两者都不行 → 直接说「不能直接买」，不要绕弯。

> **在事件表里标注中观层传导。** 不单独开中观文件，但在events/的事件表里用`【中观】`标记标明传导路径。

> **每条信息进来，先问：这个信息在宏观→中观→微观的哪个位置？**

```
🌍 宏观层：美伊地缘/中美关税/美债利率/美联储/中国政策
   ↓ 影响哪些中观板块
🏭 中观层：成长科技/新能源/避险/周期/消费
   ↓ 影响哪些具体持仓
📦 微观层：芯片020628/电池018926/海外科技007476/黄金/etc
```

**例：** 美债30Y 5.18%\n- 宏观层：全球利率上行\n- 中观层：成长科技🔴 | 避险🟢 | 债券🔴\n- 微观层：芯片🟡 | 海外科技🔴🔴 | 电池🟡 | 黄金🟢 | 债券🔴\n\n### ⭐ 框架层原则：两种收益率上升的区别（2026-05-22 豆包分析确认）\n\n**核心区分：** 美债收益率上升本身不是信号，它的**驱动原因**决定了市场情绪。\n\n| 类型 | 驱动因素 | 市场情绪 | 典型案例 |\n|:----|:---------|:--------|:--------|\n| **复苏式** | 经济过热 + 企业盈利高速增长 | 乐观，股市照涨 | 2021年：10年从0.5%涨到1.7%，美股大涨 |\n| **滞胀式** | 通胀粘性 + 财政赤字 + 增长乏力 | 悲观，资金避险出走 | **2026年5月现在**：30年5.18%，全球股市下跌 |\n\n**现在的市场是滞胀式收益率上升：**\n- 市场悲观不是因为利率高，而是因为**利率高+增长弱** → 风险溢价被压缩到极致\n- 资金流向美债不是因为美债收益高吸引人，而是**市场对其他资产未来没信心**\n- 高收益率本身是结果，不是原因——它是市场用真金白银投票出来的悲观预期\n\n**对持仓分析的指导意义：**\n- 2026年5月的回调是系统性（滞胀式收益率上升），不是个别持仓逻辑坏了\n- 防御资产（黄金/债券）起保护作用\n- 在宏观信号转变（收益率回落）之前，不要期待V型反弹\n- 把黄金的P值和收益率上升的程度挂钩：收益率越高→黄金保护力越强

## 第一步：按时间线收集信息

**时序铁律：** 新闻必须按从旧到新的顺序排列。每一个后继事件的前一个后验=其先验。

### 🚨 事件链=完整发展过程，不是最新几天的新闻（2026-05-20 用户严厉纠正）

**用户原话：** "什么叫事件链，就是它的发展过程" + "你整一堆今天的新闻，你过去的信息你都不知道"

**正确做法：** 事件链必须覆盖从起点（价格开始变动/逻辑开始形成）到今天为止的**全过程**，不能只看最近1-2天的新闻。

| 阶段 | 做法 | 工具 |
|:----|:-----|:-----|
| 最近1周 | 财联社电报搜索（cls.cn/searchPage?type=telegram&keyword=XXX），精确到分钟 | 浏览器 |
| 更早（1-4周） | 财联社翻页，搜狗/百度补充，关键词覆盖三层层级（资产名+宏观事件+产业链） | 搜狗curl + 财联社 |
| 起点确认 | 找到价格走势变化的起始点，对应最早的催化事件 | 净值API + 新闻交叉 |

**搜索关键词必须覆盖三层（2026-05-20 纠正确立）：**
```
第1层：基金/资产名称（"电池ETF" "锂电池"）
第2层：宏观政治事件（"储能政策" "CIBF展会" "四部门座谈会"）
第3层：产业链上下游（"碳酸锂价格" "固态电池" "恩捷" "赣锋锂业"）
```

**财联社翻页限制（2026-05-20 实战踩坑）：** 财联社的页码参数（?page=2/3/4）可能不翻到真正更早的内容——超过~10天的内容可能不在此搜索结果中。此时用搜狗/百度补充搜索历史新闻。

**检查清单：** 事件链建好后自问——"我能看到从起点到现在每一天的催化事件吗？中间有没有断层？"

```
价格数据是贝叶斯的校准工具。方法：
  ① 收集历史新闻（带精确日期）
  ② 拉对应资产的净值/价格曲线
  ③ 每个新闻事件后看价格反应（涨/跌/幅度）
  ④ 用这个来参考这个类型的事件有多大影响
  ⑤ 注意：不过拟合，只看大趋势
```

**新闻来源优先级（中国网络环境）：**
1. 财联社电报搜索 `cls.cn/searchPage?type=telegram&keyword=XXX`（浏览器，内容最全）
2. 东方财富网 `eastmoney.com`（浏览器，新闻+研报）
3. 已配置的RSS聚合器（本地，稳定但有限）
4. 已知博主观点（thinker档案，最高权重）
5. 仅在以上都失败时，承认"未找到相关信息"，不编造

## ⚠️ 知乎信息源处理铁律（2026-05-22 用户纠正）

**核心教训：** 知乎博主的信息处理，**文章和回答都要看**，而且**只看最近10-20天**。用户原话：\"你听听听听这最基本的东西\"。只读文章不看回答 = 漏掉大量高质量内容。

### 正确流程

```python
# ① 拉最近20条文章
curl -s "https://www.zhihu.com/api/v4/members/{user_id}/articles?limit=20&offset=0" \
  -H "Cookie: $(cat ~/桌面/zhihu_com_cookie_最新.txt)"

# ② 拉最近20条回答
curl -s "https://www.zhihu.com/api/v4/members/{user_id}/answers?limit=20&offset=0" \
  -H "Cookie: $(cat ~/桌面/zhihu_com_cookie_最新.txt)"
```

### 时间约束

- **只读最近10-20天**的内容。被用户严厉纠正：\"你看一下现在几月啦？你妈的还在读3月的\"
- 从 `created` 时间戳判断：`datetime.fromtimestamp(created)` 在最近20天内才读全文
- 超过20天但标题与持仓高度相关 → 看摘要即可，不读全文

### 读内容的方式

```bash
# 文章内容
curl -s "https://www.zhihu.com/api/v4/articles/{article_id}?include=content" \
  -H "Cookie: $(cat ~/桌面/zhihu_com_cookie_最新.txt)"

# 回答内容
curl -s "https://www.zhihu.com/api/v4/answers/{answer_id}?include=content,question.title" \
  -H "Cookie: $(cat ~/桌面/zhihu_com_cookie_最新.txt)"
```

> ⚠️ **不要用CDP浏览器读知乎**。CDP启动的Chrome没有登录态。**Always use 存好的cookie + curl API。**

### 归档方式

| 类型 | 归档位置 | 说明 |
|:----|:--------|:-----|
| 文章 | events/{日期}.md + thinkers/{博主}.md | 完整内容写events，观点摘要入thinker |
| 回答 | thinkers/{博主}.md 的「最近回答」表格 | 多条同方向可合并，不每条单独写events |

### 最近20天无内容
告知用户即可，不从更早内容里读。

## 第二步：信息分层

每条新信息归类到三个层级之一：

| 层级 | 定义 | 处理方式 |
|:----|:-----|:---------|
| **L1 事实/数据** | 具体数字、公告、可验证事件 | 写进events/{日期}.md + thinkers/{博主}.md |
| **L2 洞察/逻辑** | 推理链、因果关系、时间窗口 | 写进thinkers/{博主}.md + 贝叶斯推理依据 |
| **L3 框架/原则** | 可复用投资原则 | 嵌入00-宏观传导框架.md + ⭐标记 |

## 第二步：写入events/

每个视频/文章写一条独立事件。框架级加`⭐【框架层原则】`标记。


### 多因子弱净结果模式

当更新包含 **4+个因子**，且利好利空互相抵消导致**净结果很小（±1~2pp）** 时：

**正确做法：** 仍然列出全部因子，不省略。
- 净结果小≠没有信息。它意味着多空力量均衡，持仓处于**观望磨底**状态

**错误做法：** "变化很小→跳过更新" 或 "只写结论不写因子拆分"

## ⚠️ 防偏差硬规则（2026-05-20 实施）

### ① 同类去重

同一方向、同一逻辑链的多个证据，**只取最新+最有力的一条，不累加**。

**错误做法（本会话踩坑）：** 芯片ETF蒋宇飞5条利好全部累加，但全是AI硬件同方向→重复加权。

**正确做法：**
- 蒋宇飞5条利好→分析哪条最有力度/最新、哪条最具代表性→只算一条综合
- 如果5条是不同逻辑（芯+光+存+叠是4条不同逻辑），则可以分别加权

### ② 行情异动优先清反向

价格出现大幅波动时，**旧对立观点先作废，再叠加新信息**。

**原理：** 价格变动本身是第一优先级贝叶斯新证据。如果股价说明方向错了，先清掉理由相反的旧事件，再叠加新信息。

**操作流程：**
```
行情剧烈波动（单日±3%+）→
  ① 检查positions文件里现有事件链→标记所有与当前方向相反的旧观点为「已作废」
  ② 重新评估剩余有效证据→更新P
  ③ 再叠加新信息
```

**例：** 有色金属如果突然暴涨3%，但文件里现在全是岩松笔-8pp、资金流出-8pp等利空→先清掉这些利空的理论基础（价格在反驳它们），再叠加「为什么涨了」的新信息。

## 🚨 估值温度计原则（2026-05-21 用户亲自确立）

### 不同资产用不同指标，不是统一公式

| 资产类型 | 温度计指标 | 说明 |
|:--------|:---------|:-----|
| 银行/金融 | **PB历史百分位** | PB<1=折价，银行业全球通用 |
| 科技/芯片/海外 | **PEG历史百分位** | PE+增长率，⚠️要检查泡沫风险 |
| 成熟制造业 | **PE历史百分位** | 业绩稳定的公司有效 |
| 周期股（有色/养殖） | **PB（底部时）** | 亏损时PE为负无意义 |
| 恒生科技 | 指数PE百分位 + PEG | 成分复杂，两者结合 |
| 黄金 | **不适用** | 看全球资产占比+实际利率 |

### 核心原则：逻辑是根本，估值只是参考

```
逻辑没病 + 百分位低位 = ✅ 低估有道理，可以买
逻辑没病 + 百分位高位 = 🟡 好东西贵了，等回调
逻辑变了 + 百分位低位 = ⚠️ 价值陷阱！绝对不能碰
逻辑变了 + 任何位置 = ❌ 卖掉
```

百分位只看冷热，不决定买不买。决定买不买的是**逻辑是否还在**。

## 补充规则（2026-05-21 用户确认 三条新规则）

### 规则A：逻辑簇去重（防同逻辑重复加pp）

**问题：** 同一个底层原因，被不同博主从不同角度重复提及，容易累加pp导致过度乐观。

**核心公式（2026-05-21 用户确认）：**

逻辑簇总权重 = min(单个信源最大权重 × (1 + 0.2 × (独立信源数量 - 1)), 该簇预设上限)

**说明：**
- 多个独立信源验证同一个逻辑 → 比单一信源可信，但不是线性叠加
- +0.2 是边际增益系数：每多一个独立信源只加20%增量
- 预设上限：该逻辑能影响的pp上限（比如一个宏观事件最多影响±10pp）

**判定方法：** 每来一条新信息，问自己「这条信息的底层逻辑链条是什么？跟近期哪条信息是同一个因果链条？」

| 示例 | 事件序列 | 底层逻辑 | 旧做法 | 新做法（公式计算） |
|:----|:--------|:---------|:------|:----------------|
| 宋利率风暴+2pp + 付鹏风险极值+3pp + 付鹏滞胀+2pp | 全指向「不确定性到避险」 | +7pp夸大 | 单个最大权重3pp×(1+0.2×(3-1))=3×1.4=4.2pp，上限10pp→取4pp |
| 长鑫580亿+3pp + HBM 4-5%→30%+2pp + 存储>光模块+1pp | 全指向「国产存储崛起」 | +6pp夸大 | 单个最大权重3pp×(1+0.2×(3-1))=3×1.4=4.2pp，上限8pp→取4pp |

**实操：** 写贝叶斯更新前，检查positions文件里最近3-5条更新的逻辑簇归类。如果新事件和已有的同簇，就用上面的公式算合并权重，不单独加pp。

### 规则A2：动态信源权重（2026-05-21 用户确认）

**问题：** 不同的信息源（博主、新闻渠道）可信度不同，但默认给相同的初始权重。需要用事后价格验证来动态调整。

**核心公式（贝叶斯平均）：**

```
信源当前权重 = (冷启动基准权重 × 基准次数 + 兑现次数 × 兑现准确率) / (基准次数 + 兑现次数)
```

**冷启动：**
- 用户指定的高置信度博主（如蒋宇飞）：基准权重=0.7，基准次数=4
- 普通博主：基准权重=0.5，基准次数=2
- 未验证的信源：基准权重=0.3，基准次数=1

**动态调整（每次价格验证后）：**
- 信息方向与价格走势一致（方向对了）→ 兑现次数+1，兑现准确率更新
- 信息方向与价格走势相反（方向错了）→ 兑现次数+1，兑现准确率降低
- 兑现准确率 = 对的方向次数 / 总兑现次数

**操作流程：**
```bash
① 每条信息进来 → 按信源当前权重折抵pp
② 价格验证后 → 更新该信源的兑现次数+准确率
③ 下次该信源再说话 → 用更新后的权重
```

**价格验证的时间窗口：**
- 短期事件（财报、政策公告）：3-5天内验证
- 中期事件（行业趋势、宏观判断）：2-4周内验证
- 长期事件（产业变革、地缘格局）：月度回顾

### 规则B：反向利空预设（补信息源天然偏多的缺陷）

**问题：** 你监控的博主全是对持仓有利的（宋鸿兵死多黄金、蒋宇飞看多芯片、付鹏看多黄金），信息输入本身就是单向过滤器。

**落地：** 每个持仓文件底部已有 ## 利空触发规则 表，预设5-6条利空事件及对应pp下调。更新贝叶斯时必须扫一眼。但注意：已体现在事件链中的利空不重复算。

### 规则C：先验时间衰减（防P值卡在历史高位不动）

**问题：** 一个资产两周没新消息了，但P还停在75%。实际上没消息不意味着原来的消息仍然有效。

**落地规则：**
- 每7天无新信息 → P向50%靠3pp
- 每14天无新信息 → 累计向50%靠5pp
- 每30天无新信息 → 强制重置先验到50%

写更新时自动计算距上次更新的天数。超过7天就先衰减再叠新信息。

## ⚠️ 关键坑：博主自己的贝叶斯

用户原话：**"一个人说话的权重应该从他过去说的有没有被验证来定，不是看到一条视频就拍脑袋设的。这个本身也是一个贝叶斯。"**

## 背离检查：P值与价格走势的定期校准（2026-05-20 用户确立）

> **核心原理：** P值=我的判断，价格=市场的判断。两者应长期拟合。持续背离说明我的逻辑有问题。

### 检查频率：每周一次（配合周日周报）

### 检查逻辑

| 我的判断（P值） | 市场价格走势 | 结论 |
|:---------------|:------------|:-----|
| P≥55%（看涨） | 涨了 | ✅ 拟合 — 系统有效 |
| P≥55%（看涨） | 跌了 | ⚠️ 背离 — 逻辑有漏 |
| P≤45%（看跌） | 跌了 | ✅ 拟合 — 系统有效 |
| P≤45%（看跌） | 涨了 | ⚠️ 背离 — 逻辑有漏 |
| P在45-55%之间 | 任何方向 | 🟡 中性区间，不判断 |

```

② 检查该资产本周/最近价格走势
③ 逐个标记：拟合✅ / 中性🟡 / 背离⚠️

⑤ 出现背离 → 说明该资产有我没捕捉到的信息
```

## 卖出阈值与盈亏比计算（2026-05-20 确立）

### 卖出三层规则（2026-05-21 用户修正）

| P(上行) | 操作 | 逻辑 | 例外 |
|:-------|:----|:-----|:-----|
| **> 50%** | ✅ 持有 | 盈亏比为正值，正期望值 | 无 |
| **< 50%** | ⚠️ 考虑卖出 | 盈亏比转负，负期望值 | **战略配置**（纳指、黄金HALO等有长期信念支撑的）可以维持 |
| **≤ 20%** | 🔴 无脑清仓 | 证据优势比≥4:1指向下行 | 无例外 |

**关键认知：**
- P<50%是第一道警戒线，不是P≤20%才想卖
- 但P<50%不强制卖——只有没有长期信念的仓位才需要出
- 战略配置（纳指）和HALO对冲（黄金）即使P<50%也可以持有，因为买入逻辑不是短期概率，而是长期配置价值
- 这个框架替代了旧版"两步走+双逻辑状态"的判断方式

**实操（2026-05-21 案例）：**
- 华宝海外科技 P=36%（<50%），不是战略配置（没有长期信念）
- → 盈亏比转负 → 应该卖出
- +22%利润落袋 ✅ 完全正确
- 当时不需要等P≤20%才出

### 盈亏比计算公式

```
盈亏比 = (⬆分 × ⬆%) ÷ (⬇分 × ⬇%)
```

- **> 1** = 划算，上行期望值大于下行
- **= 1** = 刚好打平
- **< 1** = 不划算，下行期望值大于上行
- 各资产的⬆分/⬇分是当前价格到目标位/支撑位的**估算分值**，需定期校准

## 🏦 现金P和黄金P的特殊逻辑（2026-05-20 用户亲自阐述）

**现金的P值不是从新闻链推导的，而是由其他资产的确定性反向决定的：**
- 资产确定性越低 → 现金的确定性（P值）越高
- 当没有确定性的去处时，现金就是高确定性的选择
- 现金P = f(其他资产的不确定性)

**黄金的P值逻辑：**
- 经济越差/宏观不确定性越高 → 黄金的确定性越高
- 黄金=经济不确定性时的确定性资产
- 黄金P = f(宏观不确定性)

**应用场景：** 当所有持仓的P值都在下降（逻辑松动/证伪），现金和黄金的P值反而在上升。这就是为什么框架说「现金也是仓位」。

## 🚨 贝叶斯控制论闭环（2026-05-22 完整实现）

> **用户反复纠正的核心问题：** *"P值贝叶斯的上行概率是吧？他他这个新闻消息面不可能就是说递增的呀。"* + *"你如果不跟着现实的走势去印证那些人的说法可信度或者还有p值的话，那这个只会越来越差。"*
>
> **贝叶斯的本质是闭环：** 新闻→更新P值→对比实际价格→背离→调整信源权重→重新更新P值

### 闭环架构

```
① 输入层：新闻/博主观点 → events/ → P值更新 → positions/*.md
         ↓
② 验证层：每周日10:00 背离检查（自动）
         ├─ P看多+价格涨 → ✅ 拟合 → 信源可信度+1
         ├─ P看多+价格跌 → ⚠️ 背离 → 信源可信度-1，查漏更新
         ├─ P看空+价格跌 → ✅ 拟合
         └─ P看空+价格涨 → ⚠️ 背离 → 同上
         ↓
③ 反馈层：信源可信度动态调整
         ├─ 高(≥80%) → P值更新时权重+3pp
         ├─ 中(60-79%) → +2pp  
         ├─ 低(40-59%) → +1pp，上限-8pp
         └─ <40% → 暂停使用
         ↓
④ 输出层：推给用户判断
         ├─ ⚠️ 背离 → 用户判断"是我漏了还是市场错了"
         └─ ✅ 拟合 → 系统正常
```

| 脚本 | 路径 | 功能 |
|:----|:----|:-----|
| 背离检查 | `scripts/bayesian_divergence_check.py` | 每周查P值 vs 净值，报背离 |
| 信源可信度 | `scripts/source_credibility.py` | 跟踪每个博主的预测准确率 |

| 审核类型 | 审什么 | 典型问题 |
|:--------|:------|:---------|
| **工作流审核** ✅ 优先 | 管道是否稳定、有没有静默失效、闭环验证够不够 | 管道断了我知不知道？cookie过期了谁提醒我？监控脚本跑没跑通？ |
| **结果审核** ❌ 非默认 | P值对不对、事件链全不全 | 黄金P=83%合理吗？电池事件链有没有漏？ |

### 正确的执行顺序

```
① 先审工作流稳定性（默认）：
   - 监控管道通不通（cron直跑状态）
   - 有没有自我健康检查（cookie过期/报告超时/脚本失败）
   - 单点故障在哪里
   
② 用户明确要求 -> 再审结果质量：
   - P值逻辑一致性
   - 事件链完整性
   - 宏中微三层互锁
```

### 踩坑教训

**错误做法（本会话）：** 用户说"用Pro模型审核"，我直接送了P值和事件链给Pro审结果。用户纠正："我让你审工作流，不是审P值。我要的是你整个工作机制的稳定性。"

**正确做法：** 先送管道架构（cron状态、脚本直跑、cookie检查、自检机制）给Pro审稳定性问题。审完再问用户要不要审结果。
