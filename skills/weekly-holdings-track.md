# 持仓周跟踪 — 四层框架 + P&L计算

> **新增周检查项（2026-05-20 用户确立）：**
> 1. **背离检查** — 对比每个持仓的P值与本周价格走势，标记拟合✅/中性🟡/背离⚠️
> 2. **双逻辑状态检查** — 每个持仓的外部逻辑+持有逻辑状态是否还完好
>
> 这两个检查是每一期周报的固定环节，不是可选项。详见 `thinker-info-to-belief` 技能的对应章节。
>
> **核心理念：** 每周日晚上的跟踪不是为了"对比预测准不准"，而是为了搞清事情是怎么发展的——叙事变了没有？因果链走通了没有？有没有框架解释不了的事？下周怎么看？
>
> ⚠️ 大好人明确要求：报告必须含三要素——①精确P&L（亏多少赚多少）②事件归因（哪个事件导致回撤/上涨）③周叙事串联。小龙虾只做短任务（验证数据点），我（Hermes）主刀出完整报告。
>
> 🔥 **叙事来源铁律（2026-05-15 用户纠正）：报告只用自家抖音监控博主（伊娃/宋鸿兵/付鹏/拿幸等）的最新动态做依据，不用外部搜索的过时新闻。** 用户原话："这明显就是旧新闻" → "以后周报我只用咱监控的博主内容做依据，不去外面搜过时新闻"。叙事要把持仓涨跌跟博主说的内容串成一条线让用户看懂"正在发生什么"。
>
> 灵感来源：小龙虾（OpenClaw）设计四层架构 + Hermes-3-405B 提供因果链视角。

## 🔥 两大核心原则（2026-05-16 用户确立 — 写在最前面）

### 原则一：每个资产的新闻链必须可追溯

用户原话："我要知道每一个资产它的新闻链是怎样的，它的涨跌幅度跟它背后的消息链是怎样一个走向。"

**周报中每个关键资产必须包含：**

```
【资产名】本周涨跌：X%
新闻链（日期 | 净值变化 | 触发事件/新闻 | 叙事链）：
  5/12 +2.3% | 台积电营收超预期15% | AI需求→芯片需求→涨
  5/14 -1.7% | 美国访华谈判消息 | 中美缓和→国产替代预期降温→跌
  5/16 -0.8% | 习即将飞美确认 | 国事级信号→继续降温
```

**输出要求：** 用户要看的是"外部敏感度"——知道哪天发生了什么、为什么涨/跌、叙事怎么演变的。不是看净值数字，是看**消息链**。

### 原则二：机会成本是一切（2026-05-16 用户纠正：不是"偏多头"，是"现金 vs 持仓谁更亏"）

用户原话："机会成本就是一切。你不知道机会成本的话，你怎么给哪个仓位更多的钱、哪个仓位更少的钱？"

**核心逻辑（2026-05-16 用户纠正我的理解）：**
- 不是偏多头想打仓位。不是找理由买入。
- 而是始终在对比：**现金通胀蒸发（机会成本）vs 持仓被套1-2年不能动（机会成本）**，选那个亏得少的。
- 安全 = **本金可回收 + 回收时间可控**
- 三重要素对决：①现金通胀蒸发 ②踏空错失收益 ③被套1-2年≈永久损失

**强制规则：**
1. 每次出周报/日报，必须先做**跨机会排名**（哪个最优先/最划算）
2. 排名依据：置信度 × 盈亏比 × 时间窗口紧急性
3. 排名结果直接决定**仓位倾斜方向**
4. 如果框架设计不确定 → **找更强的模型多轮商量**，不要自己硬想
5. **每项资产必须问：** 现金放着的通胀蒸发 vs 买入该资产可能被套的损失，哪个更大？

**每周六追问（框架修正后的正确工作流）：**
1. 这周发生了什么 → 改变了哪些资产的**上行/下行/回收期**？
2. 有资产的**机会成本排名**变了吗？（现金 vs 每个持仓 vs 空仓候选）
3. 我的仓位现在是在**"机会成本更小"的一边**吗？

### P&L 计算步骤

```
每日估算P&L = 持仓份额 × (今日估算净值 - 昨日净值)

累计P&L = ∑每日P&L（或用买入成本算累计）
```

## 四层框架

```
第一层：叙事一致性        ← 这周的故事还是上周那个吗？
第二层：逻辑链检验        ← 上周说的因果关系走通了吗？
第三层：反常检测          ← 有什么解释不了的事？
第四层：前瞻更新(含置信度) ← 下周怎么办？
```

---

## 第一层：叙事一致性

**做什么：** 把本周所有关键叙事提炼成3-5句话，跟上周的叙事对比。

**判断三档：**
| 状态 | 含义 | 例子 |
|:---|:-----|:-----|
| ✅ **延续** | 同样逻辑链，新数据支持 | 上周"AI资本涌入"→本周"DeepSeek融资500亿" |
| ⚠️ **调整** | 核心逻辑没变，侧重点偏移 | "猪周期反转"→"能繁目标下调但亏损加剧" |
| ❌ **转向** | 核心假设被推翻 | "停战→能源通"→"霍尔木兹继续封锁" |

**输入来源：** 小龙虾的 daily log + 日报PDF。

---

## 第二层：逻辑链检验

**做什么：** 把上周每个判断拆成"因为A→所以B"的链条，逐环检查。

**示例拆解：**
```
上周：DeepSeek融资500亿 → 国产AI获资本认可 → 利好芯片/AI
本周检查：
  链环1：DeepSeek融资500亿 ✅ 确认
  链环2：资本认可→有后续吗？✅ AI行业基本面加强
  链环3：芯片/AI涨了？✅ 科创芯片A +29%，AI联接C站稳上方
结论：全链走通 ✅
```

**判断标准：**
- ✅ **全链走通** — 每环都兑现 → 可信度高
- ⚠️ **中间断链** — A发生了但没导致B → 逻辑可能有漏洞
- ❌ **A根本没发生** — 前提不成立 → 立即重新评估

---

## 第三层：反常检测

**做什么：** 在①和②做完后，专门问：有什么当前框架解释不了的数据点？

**排查方向：**
- "本应如此但实际没如此"的事
- "在大家的叙事范围外"的大新闻
- 持仓走势跟预期严重偏离的项

**判断标准：**
- ✅ 无反常 → 安心
- ⚠️ 有反常但可解释 → 更新框架边界条件
- ❌ 有反常且不可解释 → **重大警报，立即分析**

---

## 第四层：前瞻更新（含置信度）

**做什么：** 基于前三层，更新下周判断。**不仅说结论，还要说信心变化。** 同时输出**跨机会排名**——所有可能的机会放一起比。

**现有机会窗口（每周重新评估排序）：**

```
排序标准：置信度 × 盈亏比 × 时间窗口紧急性

1️⃣ 机会A（当前最优先）
   逻辑链：______
   本周进展：______
   置信度：↑/→/↓
   时间窗口：______
   机会成本：______

2️⃣ 机会B
   逻辑链：______
   本周进展：______
   置信度：↑/→/↓
   时间窗口：______
   机会成本：______

3️⃣ 机会C
   ...
```

**排序规则：**
- 置信度高 + 时间窗口近 → 排前面（如芯片回调可能在几周内）
- 置信度高 + 时间窗口远 → 排前面但策略是"等"（如恒生科技等右侧）
- 置信度低 + 任何时间窗口 → 排后面或不入排名

## 🔵🟢 左侧/右侧分类框架（2026-05-19 用户确立 — 核心分类逻辑，非公式）

### 核心原则：分类不是公式算出来的

**用户原话纠正了我的错误做法：** 我之前试图用"回撤>5%=左侧"这种公式来分类，被用户明确否定了。左侧/右侧不是数据指标算出来的，是对**资产终局确定性的判断**。

### 左侧（🔵 越跌越买，安全垫保护）

**条件（全部满足）：**
1. **赌它一定会回来** — 有确定的终局逻辑（如"人肯定吃猪肉"→养殖）
2. **足够便宜** — 历史百分位够低，不会高位接盘
3. **接受时间不确定** — 你知道它会回来，但不确定什么时候

**操作逻辑：** 左侧建安全垫，等它回来。如果逻辑不变，越跌越买。

**示例：**
- 养殖ETF：人肯定吃猪肉 ✅ + 历史低位 ✅ → 🔵 左侧
- 银行：历史低位 ✅ 但"一定会回来"不确定 ❌ → 不是左侧，观望

### 右侧（🟢 等趋势确认再入场）

**条件（全部满足）：**
1. **逻辑够硬** — 这个资产为什么能涨
2. **新闻面支持** — 当前事实/政策/数据都在指向这个方向
3. **价格已确认** — 已经在涨，或正在震荡等突破关键位

**操作逻辑：**
- 趋势已经在涨 → 右侧趋势跟踪，等回调上车
- 还在震荡但逻辑硬 → 等突破关键位再进

**示例：**
- 芯片ETF：逻辑硬 ✅ + 今天+4% ✅ + 但还没突破关键位 → 🟢 右侧待确认
- 债券：经济弱→宽松→涨 ✅ → 🟢 右侧已确认

### 时间窗口由新闻定义，不是凭空定的

**这是用户最核心的纠正：** 不要硬给一个资产编时间窗口。

```
养殖的时间窗口 = 猪周期自身的节奏（不看单个新闻）
  因为"人肯定吃猪肉"这个逻辑不依赖任何新闻事件

芯片没有时间窗口 = 等下一个重大新闻
  → 习近平访美结果/美国芯片禁令/国产替代政策加码
  → 新闻来了→定义了时间窗口→才知道现在是左还是右

债券的时间窗口 = 宽松周期持续中
  → 逻辑链在持续兑现中，不靠单个新闻激活
```

**events/ 系统刚好承载这个逻辑：**
- events/ 里没芯片的重大事件 → 芯片在"等"，不判断左/右
- events/ 出现"美国新芯片禁令" → 定义新时间窗口 → 重新判断

## 🚨 事件链枢纽 + 贝叶斯紧耦合（2026-05-19 用户确立 — 最高优先级架构更新）

### 核心问题

一个宏观事件影响多个持仓。写在任何一个持仓文件里都会漏其他持仓。旧方案（thinker文件→持仓文件 双向互锁）本质还是两个文件之间留链接，没有真正耦合。

**用户原话：** "一个事件对应好几个东西，如果你直接在持仓那写，可能就会漏一些东西。两种都搞比较好。"

### 解决方案：events/ 作为事件链枢纽

```
events/YYYY-MM-DD.md（事件只写一次）
│
├── 古都闲云：4月贷款负值
│   ├── ▶ 银行🔴(贷款需求极弱) | 债券🟢(宽松预期) | 芯片🟡(A股承压)
│   └── 🎲 贝叶斯建议: 银行-5pp | 债券+3pp | 芯片-2pp
│
├── 古都闲云：韩国闪崩
│   ├── ▶ 芯片🟡(情绪传导) | 海外科技🟡(估值联动)
│   └── 🎲 贝叶斯建议: 芯片-5pp→修正为-2pp(独立性因子)
│
└── ...

            ↕  互锁（双向链接）

positions/chip-etf-020628.md
  2026-05-19 【外部事件→events/2026-05-19.md#韩国闪崩】
  古都闲云：韩国闪崩
  📖 思考：情绪可能传导，但中国芯片有国产替代独立逻辑
  贝叶斯更新: P(上行) 60%→53%→修正为56%（独立性因子校正）
  🔗 events/2026-05-19.md#韩国闪崩
```

### 三个目录的关系

| 目录 | 用途 | 互锁方式 |
|:----|:-----|:---------|
| `events/` | 每个事件只写一次，列出所有受影响持仓 + 贝叶斯建议幅度 | 🔗 互锁→ positions/*.md（所有受影响） |
| `positions/` | 每个持仓的事件+概率历史 | 🔗 互锁→ events/YYYY-MM-DD.md(事件名) |
| `thinkers/` | 博主世界观浓缩+关键预判 | 🔗 互锁→ events/ & positions/ |

### 日报贝叶斯直驱流程

```
events/ 记录新事件（影响面+贝叶斯建议幅度）
  ↓ 日报时读取
positions/*.md 应用贝叶斯更新（写入事件+概率变化+推理过程）
  ↓ 输出
日报（宏观传导→仓位分析→贝叶斯更新）
```

**关键：** 一个文件读完所有受影响持仓。不是读持仓→跳转thinker→再跳回来。

### P(上行)↔买入触发联动规则（2026-05-19 用户确立）

**核心逻辑：** 买入触发价随贝叶斯概率联动，不是固定死的。

```
触发价 = 当前净值 × (1 - 安全垫)
安全垫 = 基准安全垫 × (1 - P(上行))

P(上行)=80% → 安全垫极小 → 触发价接近现价（敢买）
P(上行)=50% → 安全垫=基准 → 触发价较低（需打折）
P(上行)=30% → 安全垫放大 → 触发价更低（需大幅打折）
```

**为什么必须联动：** 如果 P(上行) 越来越高但触发价越定越低，永远买不到。用户原话："你反过来跟他背道而驰，照这样说我就永远没机会能买它了。"

**硬条件独立于概率：** 某些条件（如"连续3周不创新低+放量30%"）是独立门槛，不随P变化。P再高也得满足硬条件才动手。

### 概率标注规则

```

✅ P(下行) 30%→35%（+5pp，宽松预期减弱）
❌ 65%→57%
❌ P 65%→57%
```

## 📊 月度贝叶斯校准（每月末执行）

> 贝叶斯系统必须有反馈环。月度校准对比月初预测 vs 月末实际走势。

### 校准内容

| 维度 | 方法 | 输出 |
|:----|:-----|:-----|
| **方向准确率** | 对比月初P(上行)判断方向 vs 月末实际涨跌 | "芯片方向判断正确率 80%" |
| **置信度偏差** | 是否过度自信/过于保守 | "本月偏向保守，低估了养殖ETF" |
| **似然比偏差(δ)** | 计算P(E|H)/P(E|¬H)的系统性偏差因子 | "看多偏差δ=1.15，需校准" |
| **信源质量评分** | 每条触发更新的博主→统计准确率 | "大牛洲3条触发→2条方向正确" |

### EC误差计算（红色警戒指标）

EC = 平均|预测概率 - 实际频率|，每项资产月初的P(上行) vs 月末实际走势分类（上涨=1，下跌=0）：
- EC < 0.08 → ✅ 绿色（正常）
- EC 0.08-0.15 → ⚠️ 黄色（警惕，自动校准）
- EC > 0.15 → 🔴 红色（暂停更新，人工介入）

### 偏差检测

检查一个月内向上修正次数 vs 向下修正次数的比例：
- 连续4周向上修正比例>60% → 警惕看多偏差
- 连续4周向下修正比例>60% → 警惕看空偏差
- 查看单品资产偏差系数δ是否>1.20——指向需要人工审查的信源或推理逻辑

### 更新验证标记

对 23-贝叶斯更新日志.md 中所有 ⏳ 记录：
- 预测方向与实际一致 → ✅，写入修正更新
- 预测方向与实际相反 → ❌，追加修正分析偏差原因
- 不确定 → 保持 ⏳ 下月复查

### 信源权重调整

| 准确率 | 操作 |
|:------|:-----|
| ≥70% | 维持信任权重 |
| 50-70% | 降权重（需2条来源确认才触发） |
| <50% 且 ≥6条 | 降权/淘汰建议 |

### 执行时机

每月最后一个周日，在周报流程中顺带完成。
