# 资产快速调阅流程

## 🔴 对用户提问「跌了多少」的应答铁律（2026-05-22 用户严厉纠正后确立）

**当用户问「跌了多少/为什么跌」时，回答结构必须按以下顺序：**

### 第零步：先问小龙虾要最新持仓（不用本地旧数据）

**错误做法（2026-05-22 被骂）：** 直接用一周前的本地文件做分析，一直盯着芯片ETF（¥1,502 蚊子仓）说，完全忽略了用户真正的重仓——电池ETF（¥7,797）、稀有金属（¥5,811）、恒生科技（¥4,500）等。

**正确做法：**
```bash
openclaw agent --session-id main --message "大好人的实时持仓明细是什么？全部持仓（含其他平台的），按市值从大到小列出来。要最新的。" --timeout 200
```

### 第一步：按仓位大小排序分析

**铁律：仓位金额决定分析权重，不按字母/随机/或习惯顺序。**

| 仓位 | 分析优先级 | 说明 |
|:----|:---------:|:-----|
| **¥5,000+** | 🔴 **最高** | 核心仓位，每一次分析必须先覆盖 |
| ¥1,000-5,000 | 🟡 中等 | 有影响但非决定性 |
| **< ¥1,000** | ⚪ **最低（除非用户点名）** | 蚊子仓/观察仓，不主动分析，用户问了才提 |

**实战教训（2026-05-22）：** 用户持仓：电池¥7,797 + 稀有金属¥5,811 + 恒生¥4,500 + 永赢¥4,254 + 有色¥5,047 + 黄金¥5,178 + 金银珠宝 ≈ ¥32,000+
而我一直在说芯片ETF ¥1,502。用户原话："你怎么老是看我芯片啊，芯片连我仓位的一个零头都没有，是一个观察仓"

### 第二步：从宏观往下推，不是逐项念

当解释全市场回调时，先给宏观框架再逐个说持仓：

```
🌍 宏观层（驱动一切的因素）→ 一句话说清根因
   ↓
🏭 中观层（受影响板块）→ 受影响最大的板块/方向
   ↓
📦 微观层（你的每个持仓）→ 按仓位大小逐个说
```

**错误做法（2026-05-22 踩坑）：** 先逐个念每个持仓的涨幅，用户嫌啰嗦。正确是先给根因再说持仓。

### 第三步：查每个持仓是否有独立新闻

用 openclaw/终端/feed API 查该板块有没有独立催化/利空，区分「跟大盘跌」和「自己出事了」。

判断标准：
- **跟大盘跌** = 同板块都在跌，没有该持仓的独立新闻 → 逻辑没坏，不慌
- **自己出事了** = 该持仓有独立的利空消息（政策打击、业绩预警等）→ 需要重新评估逻辑

### 第四步：定位文件

```bash
# 从资产名找文件名
grep -l "{资产名}" ~/桌面/Hermes外部记忆/positions/*.md
# 或从23-贝叶斯更新日志的资产列表找
```

### 第一步：查仓位现状 + 双逻辑状态

```bash
# 从12-支付宝持仓记录.md 或 23-贝叶斯更新日志.md 拉出基本信息
read_file ~/桌面/Hermes外部记忆/12-支付宝持仓记录.md
# 快速定位到对应持仓行
```

拉出：代码、名称、当前市值、成本、盈亏、收益率

### 第二步：查事件链+贝叶斯概率变化

```python
from hermes_tools import read_file
# 直接读 positions/{资产}.md
result = read_file('/home/fw/桌面/Hermes外部记忆/positions/{asset}.md')
# 关注：完整时间线区域 + 贝叶斯更新汇总表 + 当前P值 + 卖出线状态
```

同时查23-贝叶斯更新日志的"资产汇总"区域获取每条更新的时间线。

### 第三步：拉实时净值/价格

```bash
# 天天基金API
curl -s "https://fundgz.1234567.com.cn/js/{代码}.js"
```

拉出：昨日净值、今日估算净值、今日涨跌幅%

### 第四步：输出格式

按这个顺序一次性呈现：

```

📊 仓位现状

持有逻辑（你为什么买它）：完好/松动/失效 — 依据简要说明

📋 事件链 + 贝叶斯概率变化
日期 | 事件 | P变化 | 累计P
...（完整链，从初始先验到最新）

📌 当前状态
上涨概率=XX% | 盈亏比X.XX | 卖出线:概率低于20%就卖 → ✅安全/⚠️关注/🔴危险

## 报告诚实性铁律（2026-05-22 用户纠正后确立）

**当向用户呈现分析时：**

❌ **不要说：** "全部在本轮缓存中"、"零额外读文件"、"不用读了都在缓存里"
✅ **要说：** "读了 gold.md（114行）、events/2026-05-22.md（65行）、00-宏观传导框架.md（第50-130行）"

**为什么重要：** 用户会检查。如果说没读但实际读了 → 用户觉得你在撒谎或炫耀。透明报读了多少文件、看了哪些是关键事实行的表现——不是炫耀。

**规则：** 每次汇报前，在心里默数读了几个文件、共多少行。如果>0，直接说「读了X个文件共Y行」。这不是啰嗦，这是诚实的工程汇报。

## 事件链输出模板（2026-05-22 会话验证有效的格式）

当用户要看单个资产的事件链时，使用三层格式：

```

ASCII图展示宏观传导路径
  关键数据点（数字+来源）

### 🟠 中观层
博主原话摘录
  产业链/行业影响

### 🟢 微观层（持仓）
日期 | 事件 | P变化 | 累计P

**关键原则：** 宏观→中观→微观，自上而下推演，不跳层。每个资产的事件链结束时附利空触发规则检查表。

## 重要约束

- ✅ **全中文输出**：不要用⬆⬇%/P≤等符号。写"上涨概率"、"下行空间"、"卖出线"，写完整的汉字描述。用户原话："一堆符号看着懵逼"。
- ✅ **不猜价位**：不要说"我觉得能涨到X"或"支撑位在Y"。双逻辑状态检查可以回答"该不该卖"，不需要价格目标位。
- ✅ **双逻辑状态是第一条信息**：用户最先想看的是外部逻辑和持有逻辑还在不在，不是价格走势。
