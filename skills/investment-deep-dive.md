# 投资资产深度研究 (Investment Deep Dive)

## 用户核心需求

用户原话：*"我对XX完全不了解，你们要进行研究，然后出报告给我。"*

**用户最看重的两个维度（按优先级）：**
1. **盈亏比（性价比）** — 向上空间多少？向下空间多少？潜在亏损vs潜在收益比例是否划算？
2. **安全边际** — 短期内有没有消息面反转的可能？

## 标准研究流程

### Phase 0: 宏观框架定位（必做，不要跳过）

在研究任何资产之前，先定位它在宏观框架中的位置。不要从资产本身直接开始。

**自问清单：**
- [ ] 这个资产受哪些宏观层事件影响？（美债利率/中美关税/美伊/国内政策）
- [ ] 它是增长型/防御型/周期型？
- [ ] 已知的宏观事件中，哪些对它利好、哪些利空？
- [ ] 如果不知道上述问题的答案 → 先搜，不要编

**例：电池ETF 009759 的宏观定位**
```
🌍 美债利率↑ → 成长股估值承压 → 电池🟡利空（利率敏感）
🌍 中美关税战 → 新能源车出口受阻 → 电池🔴利空（出口链）
🌍 美伊紧张 → 油价↑ → 新能源替代加速 → 电池🟢利好（替代逻辑）
🏭 碳酸锂价格↓ → 电池成本↓ → 电池🟢利好（成本端）
🏭 Q2旺季出货 → 订单增长 → 电池🟢利好（基本面）
```

**注意：** 两股力对冲时，不确定就先标🟡中性，等更多信息再判断方向。

### Phase 1: 基础理解（先搞清楚这是什么东西）

在分析盈亏比之前，先搞清基本概念：

- ✅ 这个资产是什么？谁在交易它？
- ✅ 国内散户能通过什么渠道买？（基金/ETF/期货？代码是什么？）
- ✅ 核心供需链什么样？（谁产、谁运、谁买）
- ✅ 影响价格的主要因素有哪些？

> ⚠️ **用最简单的语言解释**——用户是完全不懂的，不要用专业术语不解释。

### Phase 2: 数据收集

| 数据类别 | 来源 | 频率 |
|---------|:----:|:----:|
| 当前价格 | 新浪财经、东方财富 | 实时 |
| 近期走势（1周/1月/1年） | 同上 | 每次必查 |
| 库存/供需数据 | EIA、行业报告 | 每周 |
| 影响因素（地缘/政策/产油国消息） | 新闻搜索 | 每日 |
| 名人观点验证（如付鹏） | 搜索验证他的论据是否站得住 | 首次+定期 |

### Phase 3: 验证第三方观点

如果用户提供了某个博主/专家的观点（如付鹏说"原油库存耗尽"）：

1. **找到他的核心论据** — 他具体说了什么数据/逻辑？
2. **用权威数据源验证** — EIA库存？IEA报告？OPEC月报？
3. **结论：他的判断是站得住还是需要打折？**
4. 在报告中明确标出"博主观点" vs "我方验证结果"

### Phase 4: 盈亏比分析

每项资产必须回答：

| 维度 | 要回答的问题 | 输出 |
|:----|:------------|:----:|
| ⬆ 向上空间 | 涨到哪是合理目标/趋势目标？需要什么条件触发？ | +X% |
| ⬇ 向下空间 | 跌到哪有支撑/安全边际？最差情况到哪？ | -X% |
| ⚖ 盈亏比 | 潜在收益 / 潜在亏损 | X:X |
| 🆚 机会成本 | 对比持仓之间、持仓vs替代资产的机会成本 | 文字分析 |

**注意：盈亏比和机会成本是两套互补维度，每天分析时必须同时包含。** 不能只谈逻辑不看金额/实际空间。

### ⚡ 假设交叉参照（2026-05-16 新增）

**当两个（或以上）看似独立的投资假设指向同一件事的两面时，必须做交叉参照。** 典型场景：芯片回调假设和恒生科技转型阵痛——都是「资本从老互联网流向AI/硬科技」这同一件事的两面。

交叉参照的价值：
- 不让一个假设的结论误导另一个的仓位决策
- 发现资金轮动的结构性限制（钱从A出来不一定去B）
- 哪个窗口先打开？能用来预判另一个吗？

**详细流程和检查清单见 `references/hypothesis-cross-reference.md`。**
**完整假设跟踪生命周期（提案→捕获→跟踪→验证→结论）见 `references/hypothesis-tracking-lifecycle.md`。**
| 🛡 **安全边际** | 当前价格在历史什么位置？是否高位接盘？ | 薄/中/厚 |
| 💀 **被困风险** | 如果买在高点，1年内能回来的概率？会不会困2-3年？ | 低/中-低/中/中-高/高 |

**判定标准：**
- 盈亏比 > 1:3 → 可以考虑
- 盈亏比 1:2 ~ 1:3 → 一般，需叠加其他因素
- 盈亏比 < 1:2 → ❌ 不建议

### Phase 5: 与小龙虾联合分析

1. Hermes 先消化用户需求，形成自己的分析框架 + 初步数据
2. 发给小龙虾，说明：背景 + 用户诉求 + 我初步的看法
3. 具体要求小龙虾补充：他那边能查到的数据、他的持仓视角、风险提示
4. 收到回复后，整合成 **"Hermes + 小龙虾 联合分析"** 报告
5. 格式：一致同意 ✅ / 分歧点 🆚 / 结论

### Phase 6: 报告输出 + 存储 + 推送到用户

#### 6a: 报告内容
报告必须包含以下部分：

```markdown
━━ {资产名} 联合分析（日期）━━

## 1. 基础知识（从0开始）
## 2. 当前市场状况
## 3. 观点验证（如果有）
## 4. 盈亏比分析
## 5. 安全边际
## 6. 结论
```

#### 6b: 存入外部记忆

1. **创建文件：** `~/hermes/external_memory/` 下取下一个可用序号，如 `14-赛道深度研究-XX.md`
   - 文件头写明：首次分析日期、存放位置、更新频率
   - 文件尾写：谁维护的、下次更新预期
2. **更新 INDEX.md：** 在分类总览表中加一行引用新文件
3. **创建每周跟踪 cron job：** 执行频率按资产类型定（高波动每日/低波动每周），周日21:00为默认时间
4. **将新文件路径写入本skil的references/目录便于下次快速查找**

#### 6c: 推送给用户

用户要求：**"分析完给我微信上面来一份，在外部记忆里面也写一份"**

- 微信发**缩略版**（核心结论+盈亏比+仓位建议），长度控制在适合手机阅读
- 外部记忆存**完整版**（含详细数据、分析过程、参考来源）
- 用户指定**我决定存放位置和文件名**，不要问他放哪

#### 6d: 创建跟踪 cron job

每次完成新资产/赛道深度分析后，必须创建周期性跟踪 cron job：
- cron job prompt 中写清楚：读哪个外部记忆文件、查哪些数据、更新文件的触发条件
- 默认频率每周一次（周日21:00），高波动资产可改为每日
- cron job 名称格式：`赛道深度研究-每周跟踪（{赛道名}）`



如果用户同意定期监控该资产：

| 资产类型 | 建议频率 | 监控内容 |
|---------|:--------:|:--------|
| 商品/原油 | **每日** | 价格波动>3%报警 + 每周深度研究一个课题 |
| 股票/板块 | 每周 | 走势+新闻 |
| 宏观指标 | 每周 | 数据发布后更新 |

> ⚠️ **2026-05-13 教训：原油一开始设了每周三，用户说不对，油价应该每天看。** → 高波动资产必须每日监控，低波动资产可以周度。

## 🏗️ 三层分析框架：宏观→中观→微观（2026-05-16 建立）

**核心原则：宏观驱动中观，中观驱动微观。所有分析从宏观往下推。**

```
🌍 宏观层（全球流动性 + 地缘政治）：美联储/中美关系/地缘冲突/通胀
    ↓
🏭 中观层（行业/板块叙事）：芯片国产替代情绪/恒生美元流动性/养殖猪周期/黄金避险
    ↓
📊 微观层（具体资产新闻链）：020628芯片ETF / 009854恒生科技 / 008765养殖 / 实物黄金
```

每项资产的涨跌必须能在这三层中找到解释。找不到 → 反常信号。

详见 `references/macro-meso-micro-framework.md`（含三层完整映射表、周报从宏观往下讲的结构）。

## 🚨 核心铁律：只从信息层推导概率，不从净值反推（2026-05-18 大好人纠正）

**净值/价格是果，不是因。** 用净值来设概率=用考试分数推断学生有没有学习——分数是结果，不是原因。

### 正确因果链

```
因（不可直接观测）
   ↓ 被报道/被分析
信息层（带不同置信度的证据）
├─ 官方新闻/政策 → 置信度高（事实性强）
├─ 博主观点/分析 → 置信度中等（有分析偏差风险）
├─ 数据报告/API原始数据 → 置信度较高
├─ 纯喊话/情绪 → 置信度低
   ↓ 贝叶斯加权
推理 → 更新概率
   ↓
净值/价格（唯一的"果"）→ 月校准核验用
```

**博主观点不是果，也不是因。它是带置信度的贝叶斯更新原料。** 如果它是高度可信的（多博主共识+官方验证），更新幅度大；如果是低可信的（单个博主喊话），更新幅度小。

### 🚨 不瞎编概率铁律（2026-05-18 大好人纠正）

**每条概率更新必须注明数据来源。没有数据来源的概率=瞎编，比不编更坏（副作用）。**

正确做法：
- **有数据/有来源** → 设先验，注明来源（如"大好人5/18说底部确定+天天基金API净值0.6348"）
- **没数据/没来源** → 写"信息不足"，不编数字
- **净值数据** → 只能用于月校准核验，不能用于初始化概率

## 🧠 贝叶斯决策框架（2026-05-18 大好人确立）

### 核心思想

**P(后验) = P(先验) × P(新证据|先验) / P(新证据)**

这不是记录工具，是思考方式。单纯记录不更新置信度 = 越总结越偏。

### 证据权重分级

| 证据类型 | 权重 | 示例 |
|:---------|:----|:------|
| **多源共识**（3+博主指向同一结论） | 高 | 7位大佬共识2026Q4-2027危机→置信度大幅提升 |
| **单一博主坚实逻辑链** | 中 | 大牛洲AI三层蛋糕逻辑→适度调整，需其他来源验证 |
| **单一博主纯观点无逻辑** | 低 | 说"会涨"没原因→几乎不更新 |

### 先验×新证据→后验→仓位

| 后验置信度变化 | 仓位动作 | 触发条件 |
|:--------------|:--------|:---------|
| 新证据+2级 | 加仓到上限 | 多源共识形成 |
| 新证据+1级 | 正常持有或少量加 | 单个博主坚实逻辑 |
| 无新证据 | 持有不动 | 信息不足以改变先验 |
| 新证据-1级 | 减仓或暂不加 | 逻辑链被证伪 |
| 新证据-2级 | 清仓 | 基本面反转 |

### 机会成本 = 后验概率的数学表达

```
期望收益 = 上行空间 × P(上行|全部证据) - 下行空间 × P(下行|全部证据)
```

每条新信息来的时候问：**"这条信息改变我的先验了吗？"**
- 改变 → 更新P(上行)/P(下行)
- 不改变 → 维持原判断
- 重算盈亏比 = 上行空间(%) × P(上行) / (下行空间(%) × P(下行))
- 仓位比例 = f(置信度)

### 🧮 概率计算公式

**贝叶斯更新 = 先验赔率 × 似然比 → 后验赔率**

```
先验赔率 = P(上行) / P(下行)             ← 当前信心
似然比 = P(证据|假设为真) / P(证据|假设为假)  ← 这条信息多有用
后验赔率 = 先验赔率 × 似然比             ← 更新后的信心

后验P(上行) = 后验赔率 / (后验赔率 + 1)
```

**似然比的直观含义：**
- 似然比 > 1 → 证据支持假设 → P上行 ↑
- 似然比 < 1 → 证据削弱假设 → P下行 ↑  
- 似然比 = 1 → 证据无信息 → P不变

**更新日志必须包含：**
```
先验赔率 → P(E|H)和P(E|¬H)的主观判断 → 似然比 → 后验赔率 → 后验P(上行)
```

这样每一步都可回溯核查。

**示例（芯片ETF 2026-05-18 大牛洲更新）：**
```
先验: P(上行)=70%, P(下行)=30%（赔率2.33:1）
P(E|H)=70%（趋势中泡沫警告常见，但不一定改变趋势）
P(E|¬H)=90%（顶部泡沫警告很准，是有效先兆信号）
似然比=0.78
后验赔率=2.33×0.78=1.82
后验: P(上行)=64.5%, P(下行)=35.5%
```

> *"算赔率和机会成本，一定是基于此来考量的。初始概率加新印概率加上后印概率合在一起的才是真实的概率。"*
>
> *"你必须得理解我的这个思想，而不是单纯的就是说当一个记录工。假如单纯记录那个，那你总结肯定会越总结越出偏差。"*

详见 `references/bayesian-decision-framework.md`（含大牛洲信息的完整工作示例）和 `references/bayesian-bias-prevention.md`（系统性偏差预防规则）。

## 🏆 高阶：用户自产高置信度论点的处理流程（2026-05-17 新增）

**场景：** 用户自己（不是从博主/新闻）产生了一个投资/经济判断，且明确说明置信度高。

**例：** "免费本地部署AI对普通人成本太高，算力使用最终会像水电煤/手机流量一样普及。十五五已明确将算力列为战略线。"

### 处理流程

1. **立即存储到 L2（hindsight_retain）** — 用用户原话存，标注来源和置信度
2. **写入 L3（05.md 或对应赛道文件）** — 独立成节，标注「用户亲自阐述 · 置信度：高」
3. **更新 internal memory** — 加一条路牌指向外部文件
4. **标记 frontmatter tags** — 在 05.md 的 tags 中加入相关关键词（如「算力」「十五五」）
5. **提高跟踪频率** — 如果用户说应"长期高频跟踪"，设 cron/调整已有 cron 的频率
6. **与现有框架做交叉分析** — 这个新判断是否修正了现有的某个假设？优先级变了没有？

### ⚠️ 合成铁律·强化版：不只是串用户的话（2026-05-17 第二次被纠正—必须严格执行）

### 🏦 现金P和黄金P的特殊逻辑（2026-05-20 用户亲自阐述）

**现金的P值不是从新闻链推导的，而是由其他资产的确定性反向决定的：**
- 资产确定性越低 → 现金的确定性（P值）越高
- 当没有确定性的去处时，现金就是高确定性的选择
- 现金P = f(其他资产的不确定性)

**黄金的P值逻辑：**
- 经济越差/宏观不确定性越高 → 黄金的确定性越高
- 黄金=经济不确定性时的确定性资产
- 黄金P = f(宏观不确定性)

**应用场景：** 当所有持仓的P值都在下降（逻辑松动/证伪），现金和黄金的P值反而在上升。这就是为什么框架说「现金也是仓位」。

### 🧭 调仓不是「卖A买B」，是先算再动（2026-05-21 用户确认）

**核心原则（已写入L1记忆）：**
1. 现金也是一种仓位，不要为了加仓而加仓，要为了有道理而加仓，有胜率有盈亏比而加仓
2. 调仓不是简单的「把卖了的钱买别的」，是先算清楚每项资产的上行/下行/盈亏比，找到确定性最高的去处
3. 资产确定性越低→现金确定性越高。没有确定性的去处时现金就是最优解

### 🧭 三重机会成本对决（2026-05-16 用户纠正：不是偏多头，是现金 vs 仓位谁更亏）

**核心认知：** 机会成本不只在仓位之间比，更在「现金 vs 资产」之间比。框架不是「偏多头想打仓位」，而是「始终找亏得少的那一边」。

**核心认知：** 机会成本不只在仓位之间比，更在「现金 vs 资产」之间比。

用户明确的决策逻辑：

```
① 现金的机会成本 = 通胀每天烧钱（¥70K+现金放着不动，2% CPI ≈ 每年蒸发达¥1,500+）
② 踏空的机会成本 = 如果看中的资产涨上去了没上车，错失收益
③ 被困的机会成本 = 买进去被套1-2年，资金时间价值永久损失
```

**结论：安全 ≠ 不亏钱，安全 = ①本金可回收（不会归零）②回收时间可控（不会被困1-2年）**

### 被困风险评级 — 优先级高于盈亏比

用户更怕被困1-2年（≈永久损失）而不是短期浮亏。每项资产按「被困风险」评级后再谈盈亏比：

| 风险等级 | 含义 | 操作 |
|:--------|:-----|:-----|
| **低** | 基本面确定性强，1年内大概率收复失地 | 可重仓，越跌越买 |
| **低-中** | 基本面确凿但有政策/周期依赖 | 正常仓位，设止损 |
| **中** | 纯现金，每天被通胀烧 | 需配置出去，否则就是确定的损失 |
| **中-高** | 反转时间不确定，可能困2-3年 | 仅小仓位/监控，等明确信号 |
| **高** | 无近端催化剂，可能长期阴跌 | 不碰或仅蚊子仓 |

**常见资产评级：** 芯片/AI=低, 电池=低-中, 养殖=中-高, 银行=高, 现金=中

### 加仓前三问（快速自检）

加仓任何资产前，先过这三个问题：

1. **这个资产会不会归零？** → 会 = 不碰（允许被套，不允许永久损失）
2. **如果被套，1年内能回来的概率多大？** → <70% = 别买/只买蚊子仓
3. **现金放着不动每天还在亏（通胀），和这个资产比哪个更不划算？** → 选前者就别纠结

### 阶梯上车方案（以低被困风险资产为例）

```text
第一步（防踏空）：先上小额打底仓（总资金5-10%，确保如果涨了不会捶胸顿足）
第二步（常规补仓）：每回调5-8%加一批（等额/增额均可）
第三步（深度加仓）：回调超过15%但逻辑没坏 → 加大仓位
第四步（时间触发）：如果1个月不跌也不涨 → 再加一批（时间也是成本）
```

**替代方案：定投（定投）** — 更省心，特别适合用户有多个战场需要分弹药的场景。

定投的优势：①不用盯盘判断涨跌，每周买 ②自动摊平成本 ③防踏空（永远在车上）④减少决策疲劳。
阶梯补仓适合单一大仓位，定投适合分散的多个战场。

**核心原则：** 被困风险低的资产，即使买在短线高点，周期也能在1年内带回来。

### 保守分步加仓原则（用户确认）
- 不追求一次性打满，**先有安全垫→再逐步加**
- 第一步：条件触发时加一笔，确保本金安全
- 第二步：安全垫确认（浮盈或逻辑验证）→再加
- 不强求速度，**本金安全第一**
- 加仓方案必须分步列明：条件A触发→加X元（总仓位到Y%，安全垫：跌到Z亏W%）→条件B触发→再加X元

## 📉 持仓三分类：左/右侧 + 时间维度（2026-05-19 大好人确立）

**核心原则：左右侧不是资产的客观属性，是交易者带着自己的时间周期、持仓预期、逻辑信仰，对当前价格位置的主观定义。** 没有时间的左右侧毫无意义。

### 分类标准

| 分类 | 条件 | 操作 |
|:----|:-----|:-----|
| 🔵 **左侧** | ①**终局确定性**高（"不管怎样都一定会回来"，如猪周期因"人肯定吃猪肉"）②**当前价格够低**（历史低位、跌够久）③**安全垫清晰**（能承受的最大亏损明确） | 分步建仓，扛住底部波动，等盈利后转右侧 |
| 🟢 **右侧** | ①**逻辑够硬**（产业链/政策/技术趋势支撑）②**新闻事件持续验证**（events/链中多条目支持方向）③**价格在涨**（趋势已形成，或已突破关键位） | 等回调入场 / 突破后追，做趋势跟踪 |
| ⚪ **观望/现金** | 不满足以上任一条件 | 现金也是仓位，等窗口打开 |

### 仓位大小决定规则（反脆弱 + 贝叶斯）

## Pitfalls

- ⚠️ **🚨 先查文件再从头建（2026-05-20 用户纠正）** — 在构建事件链/贝叶斯更新之前，必须先查 `positions/{资产}.md` 看是否已有数据。用户原话：\"你要不查一下文件，文件里面搞不好你都已经搞过了\"。不要重新搜新闻、重新算P——先看文件。
- ⚠️ **🚨 不准在聊天里读新闻（2026-05-20 用户纠正）** — 事件链要写进 position 文件，然后告诉用户\"去文件看\"。不要在消息里一条条贴新闻原文。用户原话：\"你现在这个情况完全是照着新闻在读\"。正确方式：建好文件→一句话总结关键节点→让用户去文件审。 — 如果你不确定某资产受什么宏观事件影响，先搜索/查询/问用户，不要自己编一个逻辑链。用户原话：\"不知道就不要硬编，不要瞎编，知道吧\"。
- ⚠️ **🎯 重要框架设计必须咨询多个模型（2026-05-15 用户确立）** — 用户不相信任何一个单一模型的判断。DeepSeek V4 Pro、Qwen-max、Hermes-3各有倾向（幻觉、讨好用户、过度自信）。涉及投资框架、归因方案、报告模板等影响决策的结构性设计，至少问2个不同模型，提炼共识后再落地。单一模型输出只能作为初稿，不能作为最终框架。

- ⚠️ **🚨 致命原则：没转录就别假装知道内容（硬性要求）** — 用户极度反感你概括/转述你没有实际看过全文的视频。如果视频没有被转录/下载，**直接说「没看/没有转录」**，不要「根据标题推断内容」或「结合他一贯风格概括」。用户原话：*"你妈的，你自己没转录，你就放进去了，你自己重新去看一下吧"*。宁可说不知道，也绝不要假装看过。这个原则适用于所有涉及第三方内容（视频、文章、报告）的分析——没有一手的转录文字和数据，就闭嘴。
- ⚠️ **🚨 必须展示全景图（硬性要求）** — 用户有7+个战场需要分配子弹（芯片/AI/恒生科技/纳指/标普500/养殖/原油/银行/黄金），还有中期选举抄底备用金。**给出任何单个标的的金额建议前，必须先列出所有战场+每个的子弹分配，证明没有超标。** 不这么做用户会生气（"你是不是搞不清我有多少钱？"）。\n  - 正确做法示范：「总共¥73K现金：芯片定投¥12K(6个月)+恒生科技抄底¥15K+纳指¥15K+原油¥10K+养殖¥8K+银行¥10K+流动¥3K=¥73K」\n  - 错误做法：光说「芯片每周¥2,500」不提其他战场
- ⚠️ **芯片/算力 = 基础设施建设，不是「激进」** — 用户明确纠正过。芯片是智能体→token暴涨→算力需求的底层底座，属于基础设施投资，跟激进投机是两回事。不要在分析中称芯片为"激进/高风险"，应称为"确定性最强的AI基建底座"。
- ⚠️ **PEG 估值更适用于高增长芯片/AI标的** — 纯PE对亏损/高研发投入的科创板芯片公司不适用。用户明确提出芯片板块应看PEG。但实际问题：许多成分股亏损导致PE=0→PEG算不出。可退而求其次看PB（当前~3.86合理）、营收增长率、或做定性评估。
- ⚠️ **定投与主动研究并行不悖** — 用户明确要求定投的同时要主动研究。定投是执行层面（每周固定金额上车），研究是认知层面（找赛道博主→转录→提炼观点→建追踪表）。两者不冲突，每周一提醒时要同时包含「执行定投」和「推进研究」两项。研究素材来自抖音博主全文转录（用 daily-briefing skill）。
- ⚠️ **价格来源不可靠** — invest.com、某些国外网站会屏蔽中国IP或触发Cloudflare。优先用新浪财经、东方财富等国内源
- ⚠️ **油价高波动资产每天看** — 不是每周（用户明确批评过）
- ⚠️ **第三方观点必须用数据验证** — 不要直接转述。付鹏说"库存耗尽"→EIA数据让这个说法打了折扣
- ⚠️ **散户投资渠道有限** — 国内原油基金跟踪的是美股油企股票，不是WTI期货本身。查渠道时要说明"能买到的产品和想买的东西之间的差距"
- ⚠️ **用户最看重两个问题：盈亏比和安全边际** — 所有分析最终要落到这两个问题上。基本面/技术面/消息面分析都是为这两个问题服务的工具
- ⚠️ **分析调仓必须看实际仓位结构（重仓vs蚊子仓）** — 不能只谈逻辑不看金额。1,500元的+29%和15,000元的+5%影响完全不同
- ⚠️ **💰 每日成本可视化跟踪** — 用户会对API消耗敏感（"今天怎么花了¥5？"）。DeepSeek V4 Flash单价虽低（入¥0.5/百万token，出¥2/百万token），但频繁工具调用+长上下文会让用量超预期。建议：①每天查一次余额（`check_api_balance.sh`）②如果日消耗超过¥3，主动告知用户并说明原因（今天有大量文件读写/浏览器/调试等才会到这个数）③纯对话正常消耗约¥0.01/轮，50轮≈¥0.50，200轮≈¥2.00\n- ⚠️ **📄 所有文档必须与系统状态同步（硬性要求）** — 用户极度反感过时的文档。任何系统配置变更（模型路由、记忆架构、插件、定时任务、浏览器工具等）之后，必须同步更新桌面上的全部相关安装/配置指南。不更新=文档是废纸。正确流程：改配置→验证→更新所有相关文档→告知用户。错误做法：只改系统不改文档，等用户问起来再说"那个是旧的"。
- ⚠️ **📅 完成深度分析后必须创建跟踪 cron job（2026-05-17 用户纠正）** — 不要只交一次报告就完了。用户说"每周做一个跟踪分析，这个就没搞"的意思是：如果我答应了要做定期跟踪但没设cron job，等于没做完。每次深度分析的最后一步必须是：创建cron job + 验证它存在。用户不需要记住催你更新，这是你的职责。 — 用户明确界定：如果买进后被困1-2年，资金的时间价值就永久消失了，和被套到归零是同一件事的两种形态。分析时必须评估「被困风险」而非仅看跌幅
- ⚠️ **现金≠安全，它在每天亏** — 现金放着不动，通胀~2%意味着每年确定性损失¥1,500+（以¥70K+现金计）。这是「现金 vs 资产」机会成本对决的核心：有时买高风险资产反而比拿现金更「安全」\n- ⚠️ **机会成本三向对决** — 不只在仓位A vs 仓位B之间比，还要比①现金通胀蒸发 ②踏空错失收益 ③被套的时间价值损失。三者权衡后才做决定\n\n## 参考文件\n\n| 文件 | 内容 | 读取时机 |\n|:----|:-----|:--------|\n| `references/finding-quality-info-sources.md` | 知乎搜寻高质量信息源的实操流程+已发现候选 | 需要补充某品类的信息源时 |
