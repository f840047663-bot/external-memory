# 新浪财经-代码查新闻工具

## 架构原则：先解耦，再耦合（2026-05-21 大好人确立）

**本工具是「解耦/耦合」设计范式的示例：**

### 解耦（独立工具）
任何人都可以独立调用，不依赖系统上下文：
- 🔵 Hermes：拿到URL → browser_navigate 读新闻
- 🟠 小龙虾：拿到URL → web_fetch 抓内容  
- 🟢 大好人：点开链接自己看，或让我查

### 耦合（集成回工作流）
解耦后的工具再嵌入自动化流程：
- 每天13:00日报 Step 0：对所有持仓跑 sina-news.sh → 拉新浪新闻
- 每两周决策回看：用新浪新闻验证之前的判断
- 每周日前瞻扫描：查各板块行业新闻

**核心原则：必须先把功能做成谁都能用的独立工具，再集成到工作流中。** 反过来就是耦合过度——工具只能在特定上下文中用。

任何人输入一个**股票/ETF代码**，就能获得该标的最新新闻。

### 工具文件
脚本：`~/.hermes/scripts/sina-news.sh`

```bash
bash ~/.hermes/scripts/sina-news.sh 512450 畜牧养殖ETF
# 输出：https://search.sina.com.cn/search?q=512450+畜牧养殖ETF&tp=news&range=all&time=day
```

## 三种使用方式

### 🔵 Hermes（我）用
拿到URL后，用 `browser_navigate` 打开，snapshot里直接读新闻标题+摘要。要读全文就点开链接。

### 🟠 小龙虾（OpenClaw）用
把URL发给他，他 `web_fetch` 抓取：
```
openclaw agent --session-id main --message "帮我web_fetch这个链接看新闻：{URL}"
```

### 🟢 大好人自己用
自己点开链接，或者让我生成链接发给你。

---

## 耦合回工作流

### 在日报中使用（每天13:00自动）
```
Step 0: 对每个持仓调用 sina-news.sh → 拿URL → 浏览器打开 → 提取新闻
Step 1: 抖音监控
Step 2: 下载转文字
Step 3: 新浪新闻 + 博主观点 → 综合分析
Step 4: 写入 events/ → 贝叶斯更新
```

### 在决策回看中使用（每两周）
查看decision对应的资产是否有新的外部新闻验证或推翻当时的判断。

### 在前瞻扫描中使用（每周日）
用代码查各板块的最新行业新闻，判断趋势变化。

## ⚠️ Pitfalls

- **已清仓的标的不再每日查新闻。** 海外科技007476已清仓后需要从日报Step 0的列表中移除。改为每两周决策回看时顺带检查。用户原话：卖了之后每隔十几次监测一次，不是每天都监测。
- **代码映射注意：** 场外基金代码（如018926）和场内ETF代码（如159147）不同。查新闻优先用场内ETF代码，因为ETF新闻更活跃。
- **输出是URL，不是新闻内容。** 拿到URL后需要进一步处理（Hermes用浏览器打开，小龙虾用web_fetch）。这不是一个直接出答案的脚本，是生成信息来源的工具。

## 附加工具一：实时行情查询

**适用场景：** 需要快速查某只ETF/股票的实时价格、今日涨跌、成交量，不需要开浏览器。

**API接口（新浪公开行情，免费）：**
```
curl -s "https://hq.sinajs.cn/list=sz159147,sh512450" -H "Referer: https://finance.sina.com.cn"
```

**代码格式：** sz=深圳, sh=上海

**返回格式解析：**
```
var hq_str_sz159147="电池50,昨收,今开,现价,今日最高,今日最低,买一,卖一,成交量,成交额,..."
```

**常用ETF代码：**
| ETF名称 | 代码 | API查询用 |
|:-------|:----|:---------|
| 电池ETF南方 | 159147 | sz159147 |
| 畜牧养殖ETF | 512450 | sh512450 |
| 中证银行ETF | 512820 | sh512820 |
| 芯片ETF | 159995 | sz159995 |
| 恒生科技ETF | 513180 | sh513180 |
| 黄金ETF | 518880 | sh518880 |
| 纳指ETF | 513300 | sh513300 |
| 有色金属ETF | 512400 | sh512400 |

**注意事项：**
- 必须带 Referer 头，否则返回 Forbidden
- 返回内容默认GBK编码
- 可以一次查多个代码，用逗号分隔

## 附加工具二：股票基本面数据查（PB/PE/每股净资产/股息率）

**适用场景：** 需要查某只股票的PB（市净率）、PE（市盈率）、BVPS（每股净资产）、股息率等估值指标，用于判断是否低估/高估。

**注意：** Sina页面使用动态JS（占位符@pb@/@pe@），无法直接提取。以下两个可靠数据源：

### 方法A：腾讯Tencent API（推荐，最简洁）

单个查询：
```bash
curl -s "https://qt.gtimg.cn/q=sh601318" | iconv -f GBK -t UTF-8 2>/dev/null
```

**字段索引（pipe分隔，0-based）：**
| 索引 | 含义 | 示例(平安) |
|:----|:-----|:----------|
| [3] | 当前价 | 54.01 |
| [4] | 昨收 | 53.76 |
| [5] | 今开 | 54.11 |
| [32] | 涨跌额 | 0.27 |
| [33] | 涨跌幅% | 0.50 |
| [34] | 52周最高 | 54.20 |
| [35] | 52周最低 | 53.47 |
| [45] | 流通市值(亿) | 5755.37 |
| [46] | 总市值(亿) | 9776.32 |
| [47] | **PB(市净率)** | **0.96** |
| [53] | **PE-TTM(动态市盈率)** | **9.77** |
| [54] | PE(静态市盈率) | 7.26 |

**代码格式：** sh=上海, sz=深圳。不支持一次查多个，需逐个查。

### 方法B：东方财富East Money API（更全面，含BVPS）

实时行情+基本面：
```bash
curl -s "https://push2.eastmoney.com/api/qt/stock/get?secid=1.601318&fields=f43,f55,f57,f58,f84,f85,f92,f116,f117,f162,f167,f168"
```

**关键字段：**
| 字段 | 含义 | 缩放 | 示例(平安) |
|:----|:-----|:----|:----------|
| f43 | 当前价 | ÷100 | 5399→53.99 |
| f55 | 涨跌幅% | 原值 | 1.38 |
| f84 | 总股本 | 原值 | 18,107,641,995 |
| f85 | 流通股本 | 原值 | 10,660,065,083 |
| f92 | **BVPS(每股净资产)** | **原值** | **56.235** |
| f116 | 总市值 | 原值 | 977,631,591,310 |
| f162 | PE(TTM) | ÷100 | 977→9.77 |
| f167 | **PB** | **÷100** | **96→0.96** |
| f168 | 未知(非股息率) | — | 19 |

**secid格式：** 1=上海, 0=深圳 → `1.601318` / `0.000001`

**⚠️ f168不是股息率。** 实际测试验证：f168值在不同股票间不一致（平安=19, 工行=3, 建行=43），且与已知股息率对比不匹配。股息率需从年度分红数据自行计算。

### 方法C：历史K线数据（近5年月/周数据）

```bash
# 周K线(klt=102)，近260周≈5年
curl -s "http://push2his.eastmoney.com/api/qt/stock/kline/get?secid=1.601318&fields1=f1,f2,f3,f4,f5,f6&fields2=f51,f52,f53,f54,f55,f56&klt=102&fqt=1&end=20260522&lmt=260" \
  -H "User-Agent: Mozilla/5.0" \
  -H "Referer: http://quote.eastmoney.com/"
# 返回格式: date,open,close,high,low,volume
```

**注意：** 使用HTTP而非HTTPS（HTTPS有时SSL握手失败）。Python urllib在子进程中可能DNS解析失败，建议直接用curl。

### 股息率估算方法

由于无直接API返回当前股息率，采用以下方式估算：

```python
# 用2024年全年每股分红 / 当前股价
dividends_known = {
    "中国平安": 2.43,   # 0.93(中期) + 1.50(末期)
    "招商银行": 2.00,   # 约数
    "工商银行": 0.303,  # 约数
    "建设银行": 0.400,  # 约数
}
div_yield = dividends_known[stock_name] / current_price * 100
```

### PB历史百分位估算方法

```python
# 1. 获取近5年周K线
# 2. 获取当前BVPS（东方财富f92）
# 3. 推定各周PB = close_price / BVPS(对应时段)
#    更精确：假设BVPS年化增长r%，反推各周BVPS
#    bvps_then = current_bvps / (1+r)^years_ago
# 4. 计算当前PB在历史序列中的百分位
# 5. 对于银行和平安，r≈3-8%/年
```

**⚠️ Pitfall: DNS/网络不稳定。** 本机偶尔DNS解析失败，导致Python urllib/subprocess超时。Workaround：用shell `curl` 代替Python的 `urllib.request`。如果curl直接调用可行但subprocess不行，用 `shell=True` 参数。

**⚠️ Pitfall: 从price百分位不能直接推断PB百分位。** BVPS逐年增长，PB百分位通常低于price百分位。如果5年内BVPS增长30%，PB百分位可能比price百分位低10-20个百分点。
