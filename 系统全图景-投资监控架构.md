## 二、模块详细清单

### 📥 输入层（5个模块）

| # | 模块 | Skill/脚本 | 触发方式 | 输出 |
|---|------|-----------|---------|------|
| I1 | 日常监控抓取 | `monitoring-pipeline` + `scrape_all_platforms.py` | cron/手动 | 断点状态 → pipeline_states/ |
| I2 | 全量视频下载 | `douyin_batch_downloader.py` | 手动 `--all --months N` | mp4 → transcripts/ |
| I3 | 转录 | VideoCaptioner Docker | 自动（下载后） | transcripts/*.txt |
| I4 | PE/PB估值数据 | `fund-valuation-cdp-extractor` | 手动/需要时 | 蛋卷基金数据 |
| I5 | 基金净值 | fundgz API | `portfolio-pnl-linkage` 调用 | holdings_live.json |

**数据流向：** I1→I3→分析层 | I2→I3→分析层 | I4→估值层 | I5→盈亏层

---

### 🧠 分析层（7个模块）

| # | 模块 | Skill | 输入 | 输出 | 核心逻辑 |
|---|------|-------|------|------|---------|
| A1 | **事件→P值** | `thinker-info-to-belief` | 转录原文 | 事件链 → P值 | 贝叶斯后验，基准50%，事件±pp |
| A2 | **P值纪律** | `monitoring-flow-discipline` | 事件链 | P值更新规则 | pp标尺、信源门控、价格反馈、R值公式 |
| A3 | **拟合校验** | `p-value-fit-analysis` | P值序列+价格 | Spearman ρ | 六维度评分，验证温度计准不准 |
| A4 | **盈亏联动** | `portfolio-pnl-linkage` | 净值+P值+价格 | 三维联动表 | 拉净值→算盈亏→Spearman→跨品种 |
| A5 | **深度研究** | `investment-deep-dive` | 用户请求 | 研究报告 | Phase 0加载估值法→三层分析→三关审核 |
| A6 | **机会成本** | `opportunity-cost-comparison` | P值+R值 | 排序表 | P×R排序，决定现金分配 |
| A7 | **持仓原则** | `position-operation-principles` | P值信号 | 操作建议 | 左侧/右侧/待重仓/_HALO_框架 |

**核心数据流：**
```
转录原文 → A1(事件→P值) → A2(P值纪律) → Excel看板
                                    ↓
                              A3(拟合校验) → 反馈修正A1
                                    ↓
                              A4(盈亏联动) → 三维表
                                    ↓
                              A6(机会成本) → 排序表
```

---

### 📊 输出层（4个模块）

| # | 模块 | 触发 | 内容 |
|---|------|------|------|
| O1 | **Excel看板** | A1/A2/A6更新后 | Sheet2事件链 + 机会成本排序 + 拟合分析 + 盈亏联动 |
| O2 | **日报cron** | 每日15:30 | P值总览 + 净值 + 事件 → 邮件 |
| O3 | **周报** | 周日 | 叙事串联 + 梯队复盘 → 邮件 |
| O4 | **原油cron** | 工作日16/17点 | 原油事件+价格双驱动 |

### 📐 估值方法论层（3个文件）

| # | 文件 | 用途 | 路由规则 |
|---|------|------|---------|
| V1 | `valuation-methodology-buffett.md` | 巴菲特四框架+**估值路由表** | Phase 0自动加载 |
| V2 | `02-改良PEG-PB估值法.md` | 亚马逊式高投入企业估值 | 路由→PEG+PB |
| V3 | `01-盈亏比计算方法论.md` | R值计算（已简化为R=P/(1-P)） | P值系统内嵌 |

**估值路由表（最高优先级）：**
```
高投入高增长/重资产科技 → 改良PEG+PB（亚马逊、AWS）
AI/半导体/高成长       → PEG（AI产业链、芯片设计）
传统行业/稳定盈利      → PE（银行、消费、周期）
重资产/金融/地产       → PB（朱起、银行股、保险）
```

## 四、当前问题与优化建议

### 🔴 问题1：转录原文丢失
- **现状：** 以前转录完提炼就删了原文
- **已修：** transcripts/ 目录创建，全量下载器强制保存
- **待做：** 历史转录无法恢复，只能从视频重新转录

### 🔴 问题2：估值数据获取困难
- **现状：** 东方财富被墙，akshare只有20行，蛋卷CDP只能手动
- **影响：** R值只能用P值简化版 R=P/(1-P)，无法加PE折扣
- **建议：** 用户从支付宝截图 → OCR提取 → 手动录入Excel

### 🟡 问题3：Skill间耦合不清晰
- **现状：** `thinker-info-to-belief` 和 `monitoring-flow-discipline` 有重叠
  - 两者都定义pp标尺
  - 两者都提信源门控
  - 两者都涉及事件→P值流程
- **建议解耦：**
  - `thinker-info-to-belief` = 纯事件处理（转录→事件→原始pp）
  - `monitoring-flow-discipline` = 纯P值纪律（信源门控→价格反馈→R值）
  - 中间用标准接口：`event → {source, raw_pp, direction} → 门控过滤 → final_pp → P值`

### 🟡 问题4：机会成本排序表重复
- **现状：** Sheet2有排序表，机会成本排序sheet也有排序表，内容不一致
- **建议：** 只保留"机会成本排序"sheet作为唯一输出，Sheet2只做事件链记录

### 🟢 优化5：cron任务可合并
- **现状：** 日报cron + 原油cron + 周报 三个独立
- **建议：** 合并为一个"持仓跟踪cron"，按时间自动选择输出内容
  - 工作日15:30 → 日报
  - 工作日16/17 → 原油追加
  - 周日 → 周报

### 🟢 优化6：拟合分析应自动触发
- **现状：** Spearman ρ 需要手动算
- **建议：** portfolio-pnl-linkage 跑完后自动更新拟合分析sheet
- **频率：** 半月一次（数据够5个点以上）
