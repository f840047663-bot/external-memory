## 权重规则 — 双向动态评分系统

**评分由 `~/.hermes/scripts/source_credibility.py` 自动管理。**

核心原则：方向正确=硬性通关，精度偏差=技术性问题。**双向升降，扣分快加分慢。**

### 星级→采信系数→P值联动

| 星级 | 系数 | P值影响（事件级±2~4pp） | 含义 |
|:---:|:----:|:-------------------|:----|
| ⭐⭐⭐⭐⭐ | 1.0 | 取上限 +4pp/条 | 高采信 |
| ⭐⭐⭐⭐ | 0.8 | 取上~中位 +3pp/条 | 较高采信 |
| ⭐⭐⭐ | 0.6 | 取中位 +2pp/条 | 中等采信 |
| ⭐⭐ | 0.4 | 取下限 +1pp/条 | 低采信 |
| ⭐ | 0.2 | 微量 +0~1pp/条 | 微量参考 |
| 🚫暂停 | 0 | 不采信 | 连续方向错误暂停 |

### 升降规则

| 触发条件 | 变化 | 类型 |
|:---------|:---:|:----|
| 单次方向错误 | 首次仅预警，不动★ | 方向 |
| 连续2次方向错误 | 🚫暂停 | 方向 |
| 方向正确但精度偏差累计5次 | -0.5★（-0.1系数） | 精度 |
| 连续3次方向正确+精度无偏差 | +0.5★（+0.1系数） | 方向 |
| 连续5次方向正确+精度偏差≤1次 | +1★（+0.2系数） | 方向 |
| 🚫暂停中连续3次方向正确 | 恢复1★观察期 | 恢复 |

### 贝叶斯哲学
- 先验可以被新证据更新：**旧错新对也能涨回去**
- 只看最近20条预测（时间窗口），历史错误不影响当前评分
- 加分慢（需要积累）、扣分快（一次方向错就-1★）→ 不对称，保持警惕性

## 🛠️ 排错档案

> 踩过坑的记录。​出问题先查这里，再查 `00-网站内容抓取总纲.md` 第七节（详细版）。

### 抖音API（2026-05-27）
1. **count不能小于30** → 某些账号（但斌）用count=5返回0条，误判"私密"
2. **User-Agent必须完整** → `Mozilla/5.0`太笼统，换个完整Chrome UA就出数据
3. **Referer必须带user路径** → `/user/{sec_uid}`，光`douyin.com/`不行
4. **API排序不是时间倒序** → 不能拿第1条vs断点，先搜断点位置再比ID
5. **ID不严格递增** → 锚点比对用ID+时间双验证
6. **JSON解析失败=子进程超时** → timeout设120秒，不是API挂了
7. **单账号API空=限流等几分钟** → 连续调同一账号过多（比如今天长坡厚雪）→ API返回空（不是cookie过期）。等5-10分钟重试即恢复，不用怀疑cookie/账号问题

### 知乎（2026-05-27）
7. **文章vs回答分开追** → 每个用户拆两行断点，互不干扰

### 全平台
