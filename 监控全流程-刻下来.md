### 第1步：API返回空/JSON解析失败

**第一反应：不是Cookie过期，是间歇性问题！先sleep重试，不要急着刷新Cookie。**

```text
第1次请求 → 返回空/JSON解析失败
  ↓
sleep(10秒)
  ↓
第2次请求 → 成功了 → ✅ 正常处理，继续下一个博主
  ↓ 如果还失败
sleep(15秒)
  ↓
第3次请求 → 成功了 → ✅ 正常处理，继续下一个博主
  ↓ 如果还失败
检查Cookie是否有效（wc -c /tmp/xxx_cookies.txt）
  ├─ Cookie无效（<500字节）→ 按"第0步"流程刷新Cookie → 再试
  └─ Cookie有效 → 标记为"失败-待重试"，写入/tmp/monitor_status.md
     → 继续下一个博主，不要在这个博主身上浪费时间
     → 阶段二回头统一重试所有失败项
```

**知乎特别注意（00-网站内容抓取总纲.md 第116行）：**
> 知乎API间歇性返回空（尤其连续请求时），**不是cookie过期**。sleep(5-8秒)后重试一次即可恢复。

**所以知乎返回空的处理顺序：**
1. sleep(10秒) → 重试
2. 还空 → sleep(15秒) → 再重试
3. 还空 → 标记"失败-待重试"，继续下一个
4. 阶段二回头重试（这时候通常已经过了足够长的冷却时间，大概率成功）

### 第3步：transcribe_new.py报"Unsupported sound format: mp4"

**原因：直接传了mp4视频文件，脚本只接受音频文件（mp3/wav）。**

```bash
# 正确做法：先提音频，再转录
# 第1步：送视频到容器

# 第2步：用ffmpeg提mp3（必须在容器内执行）
sg docker -c "docker exec videocaptioner ffmpeg -i /app/temp/v.mp4 -vn -acodec mp3 -y /app/temp/v.mp3"
# 第3步：传mp3给转录脚本
sg docker -c "docker exec videocaptioner python3 /app/temp/transcribe_new.py /app/temp/v.mp3 /app/temp/result"
```

### 第3.5步：抖音视频下载只有h264无aac音频（2026-05-29 踩坑记录）

**问题：** 用CDP Network拦截抖音视频URL下载后，ffprobe只显示h264视频流，没有aac音频流，导致无法转录。

**根因：** 抖音CDN把视频和音频**分成两个独立的流**，都标记为`video/mp4`。用CDP拦截时，`responseReceived`的`mimeType`都是`video/mp4`，无法从mimeType区分。

**正确做法：**
1. 用CDP Network拦截所有`douyinvod` URL（通常2个）
2. 每个URL下载到临时文件
3. 用ffprobe检查每个文件的codec_type：
   ```bash
   sg docker -c "docker cp /tmp/file.tmp videocaptioner:/app/work-dir/check.mp4"
   sg docker -c "docker exec videocaptioner ffprobe -v error -show_entries stream=codec_type -of csv=p=0 /app/work-dir/check.mp4"
   # 输出: video 或 audio
   ```
4. 找到video流和audio流后，用ffmpeg合并：
   ```bash
   sg docker -c "docker cp video_file videocaptioner:/app/work-dir/v.mp4"
   sg docker -c "docker cp audio_file videocaptioner:/app/work-dir/a.mp4"
   sg docker -c "docker exec videocaptioner ffmpeg -y -i /app/work-dir/v.mp4 -i /app/work-dir/a.mp4 -c copy /app/work-dir/merged.mp4"
   sg docker -c "docker cp videocaptioner:/app/work-dir/merged.mp4 /tmp/final.mp4"
   ```
5. 合并后的文件才有完整音视频，可以转录

**⚠️ 重要：** 不要看到只有h264就放弃！抖音就是把音视频分开的，必须合并。

```text
  ├─ 成功了 → 保存到暂存目录，更新状态清单为 ✅
  └─ 还失败 → 再sleep(20秒)重试
      ├─ 成功了 → ✅
      └─ 还失败 → 检查cookie是否有效
          ├─ cookie无效 → 用 cdp-get-cookies.py 刷新 → 再试
          └─ cookie有效还失败 → 标记为"最终失败"，汇报给用户
```

**重试完成后，输出最终状态：**
```
✅ 最终成功：X条
❌ 最终失败：Y条（原因：...）→ 需要用户回来处理
```
