## ③ ID锚点比对（2026-05-27 用户修复：两级比对+排错耦合）

**规则：先搜断点ID在列表中的位置，再比ID大小+时间。每一步都必须输出证据。**

⚠️ **踩坑警告：** 抖音API不是按时间倒序返回的。付鹏的API列表[1]是1月旧视频，最新在[3]。不能拿"第1条vs断点"直接判新。

```text
第一级：搜断点ID
  ├─ 找到断点ID → 看列表中它前面的所有视频
  │    ├─ 前面有条目ID > 断点ID 且 时间 ≥ 断点时间 → 算新
  │    └─ 前面所有条目ID均 < 断点ID → ✅ 0新，断点就是最新的
  └─ 找不到断点ID → 进入第二级（时间降级）

第二级：时间降级
  原因：视频被删/隐藏了
  边界：断点表里该博主的"最后处理时间"
  规则：create_time > 断点时间 → 算新
  └─ create_time ≤ 断点时间 → 已处理，跳过
  输出：⚠️ 断点ID被删，用时间降级 → 边界{时间}
```

## ⑥ 下载+转录
API拿play_url → curl下载 → docker cp送容器 → VideoCaptioner Docker转文字（BIJIAN ASR）→ docker cp取回
**⚠️ 核心规则（违反必卡住）：**
1. **ffmpeg在容器里，主机不需要装！** 直接把视频送容器，脚本自己提音频
2. **`transcribe_new.py` 用位置参数**，不是 `--input/--output`：`python3 /app/temp/transcribe_new.py /app/temp/video.mp4 /app/temp/result`
   - 它在容器内自动提音频→转文字→生成 `result_全文.txt`
3. **卡住了先读脚本源码**，不要自己猜参数！`docker exec videocaptioner head -30 /app/temp/transcribe_new.py`
4. **下载视频不要传cookie**，只传 `-H "Referer: https://www.douyin.com/"`
```bash
# 典型流程（完整可执行版）
# 1. 找play_url（从API返回json的 aweme_list[].video.play_addr.url_list[0]）
# 2. 下载视频
curl -L -o /tmp/v.mp4 -H "Referer: https://www.douyin.com/" "{play_url}"
# 3. 送容器
sg docker -c "docker cp /tmp/v.mp4 videocaptioner:/app/temp/v.mp4"
# 4. 容器内转录（自动提音频+转文字）
sg docker -c "docker exec videocaptioner python3 /app/temp/transcribe_new.py /app/temp/v.mp4 /app/temp/result"
# 5. 取回转录文本
sg docker -c "docker cp videocaptioner:/app/temp/result_全文.txt /tmp/transcript.txt"
# 6. 清理容器内外临时文件
sg docker -c "docker exec videocaptioner rm -f /app/temp/v.mp4 /app/temp/result.srt /app/temp/result_全文.txt"
rm -f /tmp/v.mp4
# 7. 读转录文本
cat /tmp/transcript.txt
```
大视频（10分钟+）约2分钟完成。
