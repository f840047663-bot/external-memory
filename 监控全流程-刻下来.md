## ① 读断点
打开 `/home/fw/监控处理断点.md`（桌面有快捷方式），拿每个博主最后视频ID+时间。

## ② 拉视频列表
API调平台接口，拿最新N条。API跪了→浏览器拼URL直接进主页。
抖音：`/aweme/v1/web/aweme/post/?sec_user_id={id}&count=30` + cookie（cookie来源：桌面`~/桌面/凭证/douyin_com_cookie_最新.txt`）
B站：`space.bilibili.com/{mid}/video` → 浏览器
知乎：`/api/v4/members/{token}/articles?limit=20` + cookie

## ③ ID锚点比对（2026-05-26 用户亲自修复的逻辑）
列表从最新→最旧遍历：
```
找到断点ID → 它就是边界。它之前（更新的）算新，它本身不算，之后（更旧的）跳过。
找不到 → 降级到时间比对。今天零点之前的全算旧，今天的算新。
置顶视频（is_top）单独跳过。
```

## ④ 跨平台去重
同一博主在多个平台发相同内容→标题比对，跳过重复的，只处理一次。

## ⑤ 标题筛选
每条新视频看标题：
- 持仓沾边（芯片/养殖/黄金/恒科/永赢/平安等）→ 进管道
- 宏观/中观沾边（利率/地缘/猪周期/AI等）→ 进管道
- 不沾边（鸡汤/社会/个人生活/纯卖课）→ 跳过，更新断点

## ⑥ 下载+转录
API拿play_url → curl下载（不传cookie，只传Referer）→ VideoCaptioner Docker转文字（BIJIAN ASR）
```bash
# 典型流程
# 找play_url: curl API → json解析 → play_addr.url_list[0]
# 下载: curl -L -o /tmp/v.mp4 -H "Referer: https://www.douyin.com/" URL
# 送容器: sg docker -c "docker cp ..."
# ffmpeg提音频
# 转录: sg docker -c "docker exec videocaptioner python3 /app/temp/transcribe_new.py ..."
# 取回: sg docker -c "docker cp ..."
# 清理: 容器内外都删
```
大视频（10分钟+）约2分钟完成。

## ⑦ 提炼核心观点
转录完不要原文dump。砍废话，只留：关键数据、逻辑链、时间节点、持仓关联度。
**不能凭标题推测内容**，必须读完转录文字再分析。

## ⑧ 贝叶斯更新（五道门）
A(事件/系统/噪声) → B(宏观折价) → C(机械) → D(定性，读position文件) → E(事实/观点分离+共识衰减+查重)。
缺一不调。0pp也必须写理由（例：纯观点/确认框架/无新数据/量级不够）。

## ⑨ 归档
events/当天.md → thinkers/{博主}.md → positions/{资产}.md（如果有持仓影响）→ 23-贝叶斯日志
宏观事件跟踪表（长期主题）也要顺带检查要不要更新。

## ⑩ 更新断点
state JSON自动更新（脚本自带）。同时 `/home/fw/监控处理断点.md` 更新标题+时间。
**不更新断点=明天不记得今天处理过什么。**

## ⑪ 出日报
宏观→中观→微观自上而下。每项：事件链→P值→3选1动作（不动/加/减）。
末尾查长期主题表：≥7天的搜新进展，<7天的跳过。
