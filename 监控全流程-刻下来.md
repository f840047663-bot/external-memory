# 监控全流程（一刀一刀讲清楚）

> 用户2026-05-26要求：刻下来，每一步是干什么，不再问。

---

## ① 读断点
打开state文件（`~/.hermes/{平台}_states/{博主}.json`），取 `last_processed_id` 和 `last_processed_ts`。
这是标记"上次处理到哪了"的唯一依据。

## ② 拉视频列表
API调平台接口，拿最新N条。API跪了→浏览器拼URL直接进主页。
抖音：`/aweme/v1/web/aweme/post/?sec_user_id={id}&count=30` + cookie
B站：`space.bilibili.com/{mid}/video` → 浏览器
知乎：`/api/v4/members/{token}/articles?limit=20` + cookie

## ③ ID锚点比对
列表从最新→最旧遍历，逐条找断点ID。
```
找到断点ID → 它就是边界。它之前的（更新）算新。
找不到 → 降级到时间比对：今天之前全算旧，今天的算新。
置顶视频单独跳过。
```

## ④ 跨平台去重
比对标题，同一博主在多个平台发相同内容→只处理一次。

## ⑤ 标题筛选
每条新视频看标题：
- 持仓沾边（芯片/养殖/黄金/恒科等）→ 进管道
- 宏观/中观沾边（利率/地缘/猪周期/AI等）→ 进管道
- 不沾边（鸡汤/社会/个人生活）→ 跳过，更新断点

## ⑥ 下载+转录
API拿play_url → curl下载 → VideoCaptioner Docker转录（BIJIAN ASR）。

## ⑦ 提炼核心观点
转录完砍废话，只留：关键数据、逻辑链、时间节点、持仓关联度。

## ⑧ 贝叶斯更新（五道门）
A(事件/系统/噪声) → B(宏观折价) → C(机械) → D(定性，读position文件) → E(事实/观点分离+共识衰减+查重)。
缺一不调。0pp也必须写理由。

## ⑨ 归档
events/当天.md → thinkers/{博主}.md → positions/{资产}.md → 23-贝叶斯日志 → 宏观事件跟踪表。

## ⑩ 出日报
宏观→中观→微观自上而下。每项：事件链→P值→3选1（不动/加/减）。末尾查长期主题表。

## ⑪ 更新断点
不管手动处理还是自动跑完，处理完的最新视频ID+时间写回state文件。
**不更新断点 = 明天不记得今天处理过什么。**
